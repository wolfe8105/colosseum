## Agent 01
### renderAfterEffects

`renderAfterEffects` is a synchronous function that accepts two parameters: `breakdown`, typed as `EndOfDebateBreakdown | null`, and `myRole`, a string that is expected to be either `'a'` or `'b'`. It returns a string of HTML markup, or an empty string if there is nothing to render.

The function begins with an early return of `''` if `breakdown` is `null`. It then reads two per-debater data objects from `breakdown`: `myData` is set to `breakdown.debater_a` when `myRole === 'a'`, otherwise `breakdown.debater_b`; `oppData` is the opposite side. From each data object it reads the `adjustments` array, falling back to `[]` via nullish coalescing. It also reads `breakdown.inventory_effects`, again falling back to `[]`. A second early return of `''` fires immediately if all three arrays are empty, meaning no modifiers fired at all.

The function defines two closures inline before building its output. The first, `renderChain`, takes one of the debater data objects and a label string. If that object's `adjustments` array is empty it returns `''`; otherwise it maps each adjustment into a `<span class="ae-step ae-step--positive|negative">` element — the sign prefix and CSS class are determined by whether `adj.delta` is non-negative — with each element joined by an arrow `<span>`. `escapeHTML` from `src/config.ts` is called on `adj.effect_name` to sanitize user content. The resulting step chain is wrapped in a `<div class="ae-row">` that also includes `d.raw_score`, `d.final_score`, and the provided label (also passed through `escapeHTML`). The second closure, `renderInventoryEvent`, accepts a `Record<string, unknown>` and reads the `effect` key to look up a display label from a hardcoded `EFFECT_LABELS` map covering four known keys (`mirror`, `burn_notice`, `parasite`, `chain_reaction`). For unknown keys it falls back to calling `escapeHTML` on the raw effect key string. A `switch` on the effect key builds a `detail` string for each known case: `mirror` reads `ev['copied_effect_id']`; `burn_notice` reads `ev['burned_effect_id']`; `parasite` reads `ev['source']` and `ev['stolen_effect_id']`; `chain_reaction` reads `ev['regenerated_effect']` and `ev['new_powerup_qty']`. All interpolated string values from the event object are cast with `String()` and passed through `escapeHTML` before entering the HTML. The `default` case of the switch serializes the entire event object with `JSON.stringify` and passes the result through `escapeHTML`. Each case returns a `<div class="ae-inv-row">` element.

After defining the closures, the function calls `renderChain` twice — once for `myData` with label `'You'`, once for `oppData` with label `'Opponent'` — and stores the results in `myChain` and `oppChain`. It builds `invSection` by mapping `invEffects` through `renderInventoryEvent` and wrapping the results in a `<div class="ae-inv-section">` if `invEffects` is non-empty, or assigns `''` if it is empty. A third early return of `''` fires if all three assembled strings are falsy. Otherwise the function concatenates `myChain`, `oppChain`, and `invSection` inside a top-level `<div class="arena-after-effects">` wrapper with an `⚡ AFTER EFFECTS` title element and returns the complete HTML string. The function performs no I/O, no DOM reads or writes, no module-level state reads or writes, and no awaited calls.

## Agent 02
### renderAfterEffects

`renderAfterEffects` is a synchronous function that takes two parameters: `breakdown`, typed as `EndOfDebateBreakdown | null`, and `myRole`, a string expected to be either `'a'` or `'b'`. It reads no module-level state and makes no network calls, DOM reads, or browser API calls. It returns a plain HTML string.

The function begins with an early return of `''` if `breakdown` is `null`. It then selects the per-debater data objects `myData` and `oppData` from `breakdown` by comparing `myRole` to `'a'`, assigning `breakdown.debater_a` to `myData` and `breakdown.debater_b` to `oppData` when the role is `'a'`, and reversing the assignment otherwise. It reads `adjustments` from both debater data objects (defaulting to `[]` via nullish coalescing) and reads `breakdown.inventory_effects` the same way into `invEffects`. If all three of those arrays are empty, it returns `''` immediately.

Two inner functions are defined before the rendering logic runs. `renderChain` accepts a debater data object and a label string. If the debater's `adjustments` array is empty it returns `''`. Otherwise it maps each adjustment to a `<span class="ae-step ae-step--positive|negative">` element — the sign prefix and CSS class variant are determined by whether `adj.delta` is non-negative or negative — and joins those spans with `<span class="ae-arrow">→</span>` separators. Effect names are passed through `escapeHTML` (imported from `../config.ts`) before insertion. The function returns a `<div class="ae-row">` containing the label, the raw score, an arrow, the chain of steps, another arrow, and the final score. Raw and final score values are inserted without `escapeHTML` and without `Number()` casting; they are read directly from `d.raw_score` and `d.final_score`.

`renderInventoryEvent` accepts a `Record<string, unknown>` event object. It defines a local lookup table `EFFECT_LABELS` mapping four known effect keys (`mirror`, `burn_notice`, `parasite`, `chain_reaction`) to emoji-prefixed label strings. It reads `ev['effect']` as a string to determine `effectKey`, then resolves the display label either from the table or by falling back to `escapeHTML(effectKey)`. A `switch` on `effectKey` builds a `detail` string for each of the four known cases: `mirror` reports the copied effect ID; `burn_notice` reports the burned effect ID; `parasite` reads `ev['source']` to choose between `'ripped from their ref'` and `'taken from inventory'` and reports the stolen effect ID; `chain_reaction` reports the regenerated effect name and new quantity. All string values read from the event object are cast with `String()` and passed through `escapeHTML` before interpolation. The `new_powerup_qty` value in the `chain_reaction` branch is interpolated directly without casting or escaping. The `default` branch serializes the entire event object with `JSON.stringify` and passes it through `escapeHTML`. The function returns a `<div class="ae-inv-row">` containing the label and detail spans.

After both inner functions are defined, `renderAfterEffects` calls `renderChain(myData, 'You')` and `renderChain(oppData, 'Opponent')` to produce `myChain` and `oppChain`. If `invEffects` is non-empty, it maps each element through `renderInventoryEvent` and wraps the results in a `<div class="ae-inv-section">` block to produce `invSection`; otherwise `invSection` is `''`. A final guard checks whether all three output strings are falsy and returns `''` if so. This guard can only be reached if both chains are empty and `invEffects` was already determined to be non-empty above, making it reachable only if `invEffects` is somehow non-empty but every event produces an empty string — an edge case that appears theoretically unreachable given how `renderInventoryEvent` is written. If at least one of `myChain`, `oppChain`, or `invSection` is truthy, the function returns a `<div class="arena-after-effects">` block containing a title element and the three rendered sections interpolated in order.
