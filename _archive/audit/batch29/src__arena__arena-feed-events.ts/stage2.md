# Stage 2 Outputs — arena-feed-events.ts

## Agent 01

### appendFeedEvent

This function reads the `ev` parameter (a FeedEvent object) and module-level state from `renderedEventIds` (a Set), `currentDebate`, and the DOM (`feed-stream` element). It reads the current profile via `getCurrentProfile()`. The function returns void. It constructs a deduplication key from the event's id or a fallback formula using event_type, side, round, and content. If this key already exists in `renderedEventIds`, the function returns early without further action. Otherwise, it adds the key to `renderedEventIds` and proceeds to look up debater names from the `currentDebate` context—if the current user's role is 'a', it uses the current profile's display_name or username; otherwise it uses the opponent's name. It creates a new div element and dispatches to a type-specific render function via switch statement: `renderSpeechEvent`, `renderPointAwardEvent`, `renderRoundDividerEvent`, `renderReferenceCiteEvent`, `renderReferenceChallengeEvent`, `renderModRulingEvent`, `renderPowerUpEvent`, `renderDefaultEvent`, or `renderDisconnectEvent`, passing the event, element, and names (and debate context where applicable). For sentiment_tip and sentiment_vote events, it calls `applySentimentEvent()` and returns early without appending to the DOM. For all other event types, it appends the element to the feed stream and checks if the stream is near bottom (within 80px); if so, it scrolls the stream to the bottom with smooth behavior.

### addLocalSystem

This function reads the `text` parameter and the DOM (`feed-stream` element), and returns void. It retrieves the feed-stream element and returns early if not found. It creates a new div element, sets its className to `'feed-evt feed-evt-system arena-fade-in'`, sets its textContent to the parameter value, appends the element to the feed stream, and scrolls the stream to the bottom with smooth behavior.

### writeFeedEvent

This async function reads the `eventType`, `content`, `side`, and optional `score` parameters, along with module-level state `currentDebate` and module-level state `round`. It returns a Promise that resolves to void. It reads the debate context and calls `isPlaceholder()` to check if a placeholder is active; if either the debate is missing or a placeholder is detected, it returns early. Otherwise, it enters a try block and awaits a call to `safeRpc()` with the RPC method `'insert_feed_event'` and an object payload containing the debate id, event type, round, side, content, and score (coalescing null score to null). If an error is thrown, the catch block logs a warning to the console via `console.warn()` with the error and returns (silent error handling).

## Agent 02

### appendFeedEvent

The function reads a `FeedEvent` parameter, the DOM element with id "feed-stream", the module-level `renderedEventIds` Set, module-level `round`, the `currentDebate` module-level object, and the result of `getCurrentProfile()`. It writes to `renderedEventIds` by adding a dedup key, creates a DOM element with the rendered event content, appends it to the feed-stream, and may scroll the feed-stream element. The function returns undefined. It first constructs a dedup key using either the event's `id` or a fallback composite key from `event_type:side:round:content`. If this key is already in `renderedEventIds`, it returns early without rendering. Otherwise it adds the key to the set. Next it looks up debater names: if a `currentDebate` exists, `debaterAName` is set to the current profile's display name/username/fallback "You" if the debate role is 'a', otherwise the opponent name; `debaterBName` follows the same logic with role 'b'. If no debate exists, both names default to "Debater A" and "Debater B". The function then creates a new div element and dispatches to render functions via a switch statement on `ev.event_type`: it calls `renderSpeechEvent`, `renderPointAwardEvent`, `renderRoundDividerEvent`, `renderReferenceCiteEvent`, `renderReferenceChallengeEvent`, `renderModRulingEvent`, `renderPowerUpEvent`, or `renderDefaultEvent`, passing the event, element, names, and sometimes debate context. The `sentiment_tip` and `sentiment_vote` cases call `applySentimentEvent` and return early without appending to the DOM. The `disconnect` case calls `renderDisconnectEvent`. After rendering (except in the early return case), the element is appended to feed-stream, and if the feed is scrolled near the bottom (within 80 pixels), the stream auto-scrolls to the bottom smoothly.

### addLocalSystem

The function reads a `text` string parameter and the DOM element with id "feed-stream". It writes a new div element to the DOM with class "feed-evt feed-evt-system arena-fade-in" and the provided text content, appends it to feed-stream, and scrolls feed-stream to the bottom smoothly. The function returns undefined. If feed-stream does not exist, it returns early without modifying the DOM.

### writeFeedEvent

The function is async. It reads the `eventType`, `content`, `side`, and optional `score` parameters, the module-level `currentDebate` object, the module-level `round`, and calls `isPlaceholder()` to check state. It returns a Promise that resolves to void. If `currentDebate` does not exist or `isPlaceholder()` returns true, the function returns early. Otherwise, it awaits a call to `safeRpc` with the method 'insert_feed_event' and an object containing `p_debate_id`, `p_event_type`, `p_round`, `p_side`, `p_content`, and `p_score` (where score is coerced to null if undefined). If the RPC call throws an error, the catch block logs a warning to the console with the error; the error is not rethrown and the function completes normally.

## Agent 03

### appendFeedEvent

This function reads the DOM element with id `feed-stream`, the incoming event parameter `ev`, the module-level state `renderedEventIds` (a Set), and the module-level state `currentDebate`. It also reads the current user profile via `getCurrentProfile()`. The function constructs a deduplication key from either `ev.id` or a fallback composite of `ev.event_type`, `ev.side`, `ev.round`, and `ev.content`, and returns early if that key already exists in `renderedEventIds`; if not, it adds the key to the set. It derives two debater names by checking `currentDebate.role` and consulting `getCurrentProfile()` to distinguish between the current user and the opponent. It then creates a new DOM div element and dispatches to one of nine render helper functions based on `ev.event_type` via a switch statement: `renderSpeechEvent`, `renderPointAwardEvent`, `renderRoundDividerEvent`, `renderReferenceCiteEvent`, `renderReferenceChallengeEvent`, `renderModRulingEvent`, `renderPowerUpEvent`, `renderDisconnectEvent`, or `renderDefaultEvent`, each receiving the event, element, and debater names (and optionally the debate context). For `sentiment_tip` and `sentiment_vote` events, it calls `applySentimentEvent` and returns early without appending to the DOM. After the switch statement, the function appends the created element to the feed stream, then checks if the stream is scrolled near the bottom (within 80 pixels); if so, it auto-scrolls to the bottom with smooth behavior.

### addLocalSystem

This function reads the DOM element with id `feed-stream` and returns early if it does not exist. It creates a new div element, sets its className to `'feed-evt feed-evt-system arena-fade-in'`, sets its textContent to the `text` parameter, appends the element to the feed stream, and scrolls the stream to the bottom with smooth behavior.

### writeFeedEvent

This async function reads the module-level state `currentDebate` and calls `isPlaceholder()` to check the current state; if either check fails, it returns early without performing any action. If both checks pass, it awaits a call to `safeRpc()` with the RPC method name `'insert_feed_event'` and an object containing `p_debate_id` (from `debate.id`), `p_event_type`, `p_round` (from the module-level state `round`), `p_side`, `p_content`, and `p_score` (coalesced to null if undefined). If the RPC call throws an error, the catch block logs a warning to the console but does not re-throw; the function returns normally. The function does not return a value on the happy path (implicitly returns `undefined` when the RPC succeeds).

## Agent 04

### appendFeedEvent

When called, this function receives a FeedEvent object and appends it to the feed stream in the DOM. It first reads the feed-stream element; if the element is absent, it returns early. It derives a deduplication key either from the event's id property or by concatenating the event's event_type, side, round, and content fields. If this key is already in the renderedEventIds set (a module-level state variable), the function returns early without rendering. Otherwise, it adds the key to renderedEventIds. It then reads the currentDebate module-level state and the current user's profile via getCurrentProfile to construct debater names for both sides: if currentDebate exists and the user's role matches a side, it uses the user's display_name or username (falling back to "You"), otherwise it uses the opponent's name from the debate; if currentDebate is absent, it defaults to "Debater A" or "Debater B". It creates a new div element and dispatches the event to a type-specific render function via a switch statement: speech events call renderSpeechEvent, point awards call renderPointAwardEvent, round dividers call renderRoundDividerEvent, reference cites call renderReferenceCiteEvent, reference challenges call renderReferenceChallengeEvent, mod rulings call renderModRulingEvent, power-ups call renderPowerUpEvent, and disconnects call renderDisconnectEvent. Sentiment tip and sentiment vote events take a different path—they call applySentimentEvent and return immediately without appending to the DOM. All other event types call renderDefaultEvent. After rendering, the function appends the element to the feed stream and checks if the stream is scrolled near the bottom (within 80 pixels of scroll end). If it is, the function calls scrollTo with a smooth behavior to scroll to the bottom; if not, no auto-scroll occurs.

### addLocalSystem

When called with a text string, this function creates a local-only system message in the feed stream without persisting it to storage. It retrieves the feed-stream element from the DOM; if it does not exist, the function returns early. It creates a new div element, sets its className to 'feed-evt feed-evt-system arena-fade-in', and sets its textContent to the supplied text. It appends this element to the feed stream and immediately calls scrollTo to scroll to the bottom with smooth behavior.

### writeFeedEvent

When called, this async function attempts to persist a feed event to storage by making an RPC call. It reads the currentDebate module-level state and calls isPlaceholder to check if a placeholder debate is active; if currentDebate is absent or a placeholder is active, the function returns early without making any call. Otherwise, it enters a try block and awaits a call to safeRpc with the function name 'insert_feed_event', passing an object with parameters p_debate_id (from currentDebate.id), p_event_type (the eventType parameter), p_round (from the round module-level state variable), p_side (the side parameter), p_content (the content parameter), and p_score (the score parameter, converted to null if undefined). If the RPC call throws, the function enters the catch block, logs a warning message to the console, and returns silently without rethrowing; the event is assumed to have already been rendered on screen by a separate call to appendFeedEvent in the wiring layer, so the failure only prevents persistence.

## Agent 05

### appendFeedEvent

This function reads the `ev` parameter (a FeedEvent object), the DOM element with id `feed-stream`, the module-level `renderedEventIds` Set, the module-level `currentDebate` object, and calls `getCurrentProfile()` from the auth module. It constructs a deduplication key from the event's id or a fallback combination of event_type, side, round, and content; if this key already exists in `renderedEventIds`, the function returns early without doing anything. Otherwise, it adds the key to `renderedEventIds` and resolves author names (`debaterAName` and `debaterBName`) by examining the current debate's role and calling `getCurrentProfile()` to access the display name or username. It then creates a new div element and routes the event through a switch statement on `ev.event_type`, calling one of nine render functions in order: `renderSpeechEvent`, `renderPointAwardEvent`, `renderRoundDividerEvent`, `renderReferenceCiteEvent`, `renderReferenceChallengeEvent`, `renderModRulingEvent`, `renderPowerUpEvent`, `renderDefaultEvent` (in the default case), `applySentimentEvent`, or `renderDisconnectEvent`, each with contextual arguments like the event, element, names object, and debate. For `sentiment_tip` and `sentiment_vote` events, `applySentimentEvent` is called and the function returns early without appending to the DOM; for all other event types, the constructed element is appended to the feed stream. After appending, the function checks if the stream's scroll position is within 80 pixels of the bottom; if so, it calls `scrollTo` on the stream with smooth behavior to scroll to the bottom.

### addLocalSystem

This function reads the `text` parameter and the DOM element with id `feed-stream`. If the stream element exists, it creates a new div element, sets its className to `'feed-evt feed-evt-system arena-fade-in'`, sets its textContent to the provided text, appends the element to the stream, and then calls `scrollTo` on the stream with smooth behavior to scroll to the bottom. If the stream element does not exist, the function returns early without doing anything.

### writeFeedEvent

This async function reads the `eventType`, `content`, `side`, and `score` parameters, and the module-level `currentDebate` object. It calls `isPlaceholder()` to check the debate state; if `currentDebate` is falsy or if `isPlaceholder()` returns true, the function returns early. Otherwise, it enters a try block and awaits a call to `safeRpc('insert_feed_event', ...)` with an object payload containing the debate id, event type, current module-level `round`, side, content, and score (coalescing null or undefined score to null). If the RPC call rejects, the function catches the error, logs a warning message to the console, and continues without re-throwing; the function then returns or completes.
