# Stage 2 Outputs — home.arsenal-shop-render.ts

## Agent 01

### renderShop

The function reads the `state` parameter (a `ShopState` object containing `catalog`, `productType`, `categoryFilter`, `rarityFilter`, `timingFilter`, `affordableOnly`, and `tokenBalance`) and the `container` parameter (an HTMLElement). It calls `applyFilters(state.catalog, state)` to produce a filtered list of effects based on the current filter state.

The function writes to the `container` element by setting its `innerHTML` to a complete shop UI. The HTML includes: product type toggle buttons (Power-Ups and Modifiers) with active states based on `state.productType`; a token balance pill displaying `state.tokenBalance`; category filter chips generated from the `CATEGORIES` array with active state based on `state.categoryFilter`; rarity filter chips generated from the `RARITIES` array with active state based on `state.rarityFilter`; timing filter chips (In-Debate and Post-Match) with active states based on `state.timingFilter`; an affordability toggle with active state based on `state.affordableOnly`; a result count showing the length of the filtered list with pluralization; and a card grid that either displays an empty message or renders each effect from the filtered list by calling `renderEffectCard(e, {...})` with context-specific flags (`showModButton` and `showPuButton`) determined by `state.productType`. Text values are escaped using `escapeHTML()`. The function returns `undefined`.

After writing the HTML, the function calls `wireShopEvents(container, state, onRerender, onSheetOpen)` with the container, state, and callback parameters. This attaches event listeners and event handling logic to the newly rendered UI.

## Agent 02

### renderShop

This function reads the `state` parameter (a `ShopState` object containing `catalog`, `productType`, `categoryFilter`, `rarityFilter`, `timingFilter`, `affordableOnly`, and `tokenBalance`), calls `applyFilters()` with `state.catalog` and `state` to produce a filtered array of effects, then constructs and writes an HTML string into the `container` element's `innerHTML` property. The HTML includes toggle buttons for product type, a balance display showing `state.tokenBalance`, category filter chips (mapped from the `CATEGORIES` constant using the current `state.categoryFilter`), rarity filter chips and timing chips (mapped from the `RARITIES` constant), an affordability toggle, a result count, and a card grid. The card grid is either a "no effects" message or a mapped list of effect cards generated by calling `renderEffectCard()` for each filtered item, with button visibility controlled by `state.productType`. Text values in the HTML attributes pass through `escapeHTML()`, though `state.tokenBalance` is interpolated directly without a Number() cast (flagged in an inline comment as a landmine). After constructing the HTML, the function calls `wireShopEvents()` synchronously with the container, state, and two callback functions (`onRerender` and `onSheetOpen`) to attach event listeners. The function is synchronous (not async) and returns `void`.

## Agent 04

### renderShop

The function reads the `state` parameter (a ShopState object containing catalog, tokenBalance, productType, categoryFilter, rarityFilter, timingFilter, and affordableOnly properties), the `container` parameter (an HTMLElement), and calls `applyFilters` to filter the state's catalog based on the current filter state. It does not read from module-level state, storage, the DOM, or network responses.

The function writes to the `container` by setting its `innerHTML` to a string of HTML that includes product-type toggle buttons, a token balance display interpolated with `state.tokenBalance`, category and rarity filter chips whose active state depends on the current filter values, a result count, and a card grid populated by either a "no results" message or mapped calls to `renderEffectCard` for each filtered item. The function has no return value.

The function calls `applyFilters(state.catalog, state)` to get the filtered list, then `renderEffectCard(e, {...})` for each filtered item to generate card HTML, and finally `wireShopEvents(container, state, onRerender, onSheetOpen)` to attach event listeners to the newly rendered HTML. The `escapeHTML` function is called on category and rarity values and labels in the filter chip template strings. All calls are synchronous and in sequence: filtering happens first, then innerHTML is set with the rendered HTML, then event wiring is applied. The function is not async.
