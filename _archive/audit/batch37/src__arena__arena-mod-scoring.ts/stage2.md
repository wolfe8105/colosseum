# Stage 2 Outputs — arena-mod-scoring.ts

## Agent 01

### renderModScoring

The function receives a `CurrentDebate` object and an `HTMLElement` container. It reads the debate's `moderatorId` and `moderatorName` fields, and calls `getCurrentProfile()` to read the current user's profile from module state. It returns early (void) if the debate lacks moderator information or if no profile is retrieved.

The function computes two boolean flags by comparing the profile ID against the debate's debater and moderator IDs. If the current user is the moderator themselves, the function returns early. Otherwise, it creates a `div` element with the class name `mod-score-section` and populates it with different HTML based on whether the current user is a debater in the debate or a spectator. For debaters, the HTML contains two buttons labeled "FAIR" and "UNFAIR" with data scores of 25 and 0. For spectators, it contains a range input slider from 1 to 50 with a starting value of 25, a value display field, and a submit button. The moderator's name is escaped via `escapeHTML()` and inserted into the DOM as part of the title. The section is appended to the container, and `injectAdSlot()` is called with the container to add an advertisement slot.

For debater buttons, the function uses `querySelectorAll()` to attach click event listeners to each button. When clicked, a listener parses the button's `data-score` attribute as an integer, disables all buttons and sets their opacity to 0.4, then calls the async `scoreModerator()` function with the debate ID and score. After the promise resolves, the listener queries for the element with ID `mod-scored`, and if a result error exists, it displays the error message (formatted via `friendlyError()` with a red circle emoji) in that element with red accent color; otherwise it displays a green checkmark with "Score submitted". The listener does not re-enable the buttons after submission.

For the spectator slider and submit button, the function queries the slider input and value display element. If both exist, it attaches an input listener to the slider that updates the value display's text content to match the slider's current value. The function then attaches a click listener to the submit button (queried by ID). When clicked, the listener parses the slider's value as an integer (defaulting to 25), changes the button text to an hourglass emoji and sets `disabled` to true. It calls the async `scoreModerator()` function with the debate ID and score. After the promise resolves, the listener checks for errors and displays them (with red accent color) in the `mod-scored` element; if there is no error, it displays the success message and removes the submit button entirely from the DOM. If an error occurred, it restores the button text and re-enables it, allowing the user to retry.
