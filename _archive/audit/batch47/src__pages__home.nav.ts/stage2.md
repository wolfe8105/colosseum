## Agent 01

### navigateTo

When called with a `screenId` string parameter, this function first validates the input by checking if `screenId` exists in the `VALID_SCREENS` array; if not, it defaults `screenId` to `'home'`. It then reads the module-level `state.currentScreen` variable to determine if cleanup is needed. If the current screen is `'arena'` and the new `screenId` is not `'arena'`, it calls `destroyArena()` to clean up arena resources. The function writes `screenId` to `state.currentScreen`.

Next, it manipulates the DOM by querying all elements with class `'screen'` and `'bottom-nav-btn'`, removing the `'active'` class from each. It then queries for the DOM element with id `'screen-' + screenId` and if it exists, adds the `'active'` class to it. Similarly, it queries for a `'.bottom-nav-btn'` element with a `data-screen` attribute matching `screenId` and adds the `'active'` class if found.

After updating the DOM classes, the function branches based on the value of `screenId`. If `screenId === 'home'`, it calls `renderFeed()` and catches any error with a console.error log. If `screenId === 'profile'`, it calls `ModeratorAsync?.renderRivals?()` with the element with id `'rivals-feed'` (using optional chaining), calls `loadFollowCounts()`, queries for an element with id `'profile-debate-archive'`, and if found, calls `loadDebateArchive()` with that element and `true` (the fire-and-forget call is prefixed with `void`). If `screenId === 'arsenal'`, it calls `loadArsenalScreen()` and catches any error. If `screenId === 'invite'`, it queries for an element with id `'invite-content'` and if found, calls `loadInviteScreen()` with that element. The function returns `undefined`.
