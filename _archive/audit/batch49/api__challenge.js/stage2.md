## Agent 01

### handler

`handler` is an async function. It reads three module-level variables: `SUPABASE_ANON_KEY` (from `process.env.SUPABASE_ANON_KEY`), `SUPABASE_URL` (from `process.env.SUPABASE_URL` with a hardcoded fallback), and two imported functions `buildChallengeHtml` and `buildExpiredHtml` from `./challenge.html`. From the request object it reads `req.query.code`. It performs an early return with a 500 response body of the plain string `'Server configuration error'` if `SUPABASE_ANON_KEY` is falsy, logging the condition to `console.error` first. It then reads `req.query.code`, coercing it to a string, trimming whitespace, and uppercasing it. If the resulting `code` is empty or does not match `/^[A-Z0-9]{4,10}$/`, it sets `Content-Type` and `Cache-Control` response headers to `no-cache`, and returns a 404 response whose body is the return value of `buildExpiredHtml()`.

The remainder of the function runs inside a single `try/catch` block. In the try branch, it constructs the URL `${SUPABASE_URL}/rest/v1/rpc/get_challenge_preview` and calls the global `fetch` with a POST request, awaiting the result. The request body is `JSON.stringify({ p_join_code: code })` and the headers carry `SUPABASE_ANON_KEY` as both `apikey` and a Bearer token. If `response.ok` is false, it throws an `Error` carrying the HTTP status code, which immediately falls to the catch branch. Otherwise it awaits `response.json()` to parse the body into `data`. If `data` is falsy or an empty array, it sets `Content-Type` and `Cache-Control` to `no-cache` and returns a 404 response whose body is `buildExpiredHtml()`. If `data[0]` exists, it assigns that object to `preview` and checks `preview.status`: if the status is not `'pending'` or `'lobby'`, it sets `Content-Type` and `Cache-Control` to `no-cache` and returns a 410 response whose body is `buildExpiredHtml()`. If the status check passes, it calls `buildChallengeHtml(preview, code)` and assigns the result to `html`, sets `Content-Type` to `text/html; charset=utf-8` and `Cache-Control` to `public, s-maxage=30, stale-while-revalidate=60`, and returns a 200 response whose body is `html`. In the catch branch, the caught error's `message` is logged via `console.error`, then `Content-Type` and `Cache-Control` are set to `no-cache` and a 500 response whose body is `buildExpiredHtml()` is returned. All calls to `fetch`, `response.json()`, and the two helper functions are synchronous with respect to their callers' awaits; there are no fire-and-forget calls.
