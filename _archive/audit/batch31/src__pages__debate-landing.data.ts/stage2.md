# Stage 2 Outputs — debate-landing.data.ts

## Agent 01

### [module load — no functions]

When this module is imported, it immediately reads the URL search parameters from `window.location.search` and extracts three values: `topic` (defaults to 'mahomes-vs-allen'), `title` (custom debate title, or null if not provided), and `src` (source identifier, or null). If a `topic` slug is provided that doesn't exist in the pre-seeded `DEBATES` object and a custom `title` is supplied, the module injects a new debate entry into `DEBATES` with the decoded title as the topic, default "Yes" and "No" sides, a category looked up from the `cat` URL parameter (defaulting to 'trending', with a predefined map providing icon and label pairs for sports, politics, entertainment, music, and trending), zero votes on both sides, and a single moderator take that credits the source as either "Telegram" or "a shared link" depending on whether `src` is 'telegram' or 'telegram-inline'. The module exports the `DEBATES` object (containing both seed debates and any injected custom topic), a `debate` constant pointing to the matched debate entry (or falling back to 'mahomes-vs-allen'), a `voteKey` string constructed from the topic slug for localStorage tracking, and re-exports `getFingerprint` from the config module. As a DOM side effect, the module sets `document.title` to the current debate's topic with the suffix " — The Moderator".
