# Stage 2 Outputs — arena-feed-wiring-debater.ts

## Agent 01

### wireDebaterControls

This function queries for five DOM elements: a textarea with id `feed-debater-input`, buttons with ids `feed-debater-send-btn`, `feed-finish-turn`, and `feed-concede`, and a cite button with id `feed-cite-btn`, plus a challenge button with id `feed-challenge-btn`. It attaches event listeners to each element. The input textarea listens for `input` events, which trigger a callback that disables the send button if the textarea is empty or disabled, then sets the textarea height to the minimum of its scroll height or 120 pixels. The send button and textarea both listen for click and Enter-key events respectively, both calling `submitDebaterMessage()` with `void` to suppress its promise. The finish button listens for click events and calls `finishCurrentTurn()`. The concede button listens for click events and executes an async IIFE that first prompts the user for confirmation; if declined it returns early, otherwise it reads the `currentDebate` module state and returns early if it is null or already has a `concededBy` property set. If the debate exists and is not already conceded, it sets the `concededBy` property to the debate's `role`, calls `clearFeedTimer()` and `stopTranscription()` and `clearInterimTranscript()`, and if the debate is not in mod view it calls `setDebaterInputEnabled(false)`. It then checks the module-level `feedPaused` state; if true it sets it to false via `set_feedPaused()`, clears the `challengeRulingTimer` interval via `clearInterval()` if it exists, sets the timer to null via `set_challengeRulingTimer()`, clears the active challenge ref id via `set_activeChallengeRefId()`, and removes the challenge overlay element from the DOM. It reads the current profile to get a display name, then awaits `writeFeedEvent()` to persist a speech event saying the debater has conceded, and finally calls `startFinalAdBreak()`. The cite button listens for click events and checks the `feedPaused` state; if paused it returns early, otherwise it calls `showCiteDropdown()`. The challenge button listens for click events, also returning early if `feedPaused` is true, otherwise calling `showChallengeDropdown()`. The function takes a debate parameter but only uses it to pass to `showChallengeDropdown()`; the function reads module-level state to get the debate for most logic. The function does not return a value or modify the debate parameter.

### submitDebaterMessage

This async function first reads the `currentDebate` module state and returns early if it is null. It then queries for the input textarea and sends button DOM elements. If the input element does not exist, is empty after trimming, or is disabled, the function returns early. It reads the trimmed text content from the input, clears the input value and height, and disables the send button. It reads the current profile and constructs an author name from the display name, falling back to username or the literal string `'You'`. It constructs an optimistic feed event object with a randomly generated UUID, the current debate's id, event type `'speech'`, the module-level `round` state, the debate's role as side, the trimmed text as content, the current timestamp, and the author name, then calls `appendFeedEvent()` with this object to update the local feed. It then enters a try-catch block and awaits `writeFeedEvent()` to persist the message via RPC, which broadcasts to other clients via Realtime; if the await throws an exception it catches it, calls `showToast()` with an error message, and logs the error to the console.
