# Stage 2 Outputs — notifications.actions.ts

## Agent 01

### timeAgo
`timeAgo` receives a single parameter `dateStr`, which may be a string, `undefined`, or `null`. If `dateStr` is falsy, the function returns an empty string immediately. Otherwise, it enters a `try` block and computes the elapsed time by subtracting the result of `new Date(dateStr).getTime()` from `Date.now()`, dividing by 1000, and flooring the result to get `diffSec`. The function then walks through a series of exclusive branches in ascending order of magnitude: if `diffSec` is less than 60, it returns `'just now'`; if the derived `diffMin` (floor of `diffSec / 60`) is less than 60, it returns that value suffixed with `'m ago'`; if the derived `diffHr` is less than 24, it returns that value suffixed with `'h ago'`; if the derived `diffDay` is less than 30, it returns that value suffixed with `'d ago'`; otherwise it returns the floor of `diffDay / 30` suffixed with `'mo ago'`. The `catch` block swallows any exception — for example one thrown by an unparseable date string — and returns an empty string. The function is synchronous, reads no module-level or external state, writes nothing, and its only return values are string literals constructed from the arithmetic above.

### markRead
`markRead` receives a single parameter `id`, a string. It calls `markOneRead(id)` from `notifications.state.ts` and branches on the boolean it returns. If `markOneRead` returns `false` — indicating the notification was already read or not found — the function returns without doing anything further. If `markOneRead` returns `true`, the function calls `updateBadge()` from `notifications.panel.ts` and then `renderList()` from the same module, in that order, both synchronously. It then reads external state by calling `getSupabaseClient()` and `getIsPlaceholderMode()` from `auth.ts`. If `getSupabaseClient()` returns a truthy value and `getIsPlaceholderMode()` returns `false`, it calls `safeRpc('mark_notifications_read', { p_notification_ids: [id] })`. The `safeRpc` call returns a promise; the function chains `.then()` on it without `await`, making this a fire-and-forget call. The `.then()` callback destructures `error` from the resolved value and, if `error` is truthy, logs it to the console via `console.error`. The function itself is synchronous from the caller's perspective, returns `void`, and writes no module-level state directly — all state mutation is delegated to `markOneRead`, `updateBadge`, and `renderList`.

### markAllRead
`markAllRead` takes no parameters. It calls `markAllAsRead()` from `notifications.state.ts` unconditionally and without branching on a return value, unlike `markRead`. It then calls `updateBadge()` and `renderList()` from `notifications.panel.ts`, in that order, both synchronously. It then reads `getSupabaseClient()` and `getIsPlaceholderMode()` from `auth.ts`. If the client is truthy and placeholder mode is `false`, it calls `safeRpc('mark_notifications_read', { p_notification_ids: null })`, passing `null` as the ID array — in contrast to `markRead`, which passes a single-element array. As with `markRead`, the returned promise is not awaited; a `.then()` callback inspects the resolved `error` field and, if truthy, logs it via `console.error` with the message `'mark_notifications_read (all) error:'`. The function is synchronous from the caller's perspective, returns `void`, and performs no early-exit branching — all four calls in the body execute unconditionally before the Supabase guard check.
