# Stage 2 Outputs — arena-entrance-render.ts

## Agent 01

### renderTier1

When called, renderTier1 receives nine parameters: a stage HTMLElement and player/opponent information (initials, names, Elo ratings), plus boolean flags for whether the opponent is AI and whether the match is ranked. It returns void. The function writes to external state by setting the `innerHTML` property of the stage element with a template string containing CSS classes and escaped content. For the player avatar, it calls `esc()` (the imported `escapeHTML` function) on the myI parameter. For the opponent avatar, it conditionally outputs a robot emoji if isAI is true, or calls `esc()` on oppI if false. The player name and opponent name are each escaped via `esc()`. Elo ratings are coerced to numbers and inserted as strings; the opponent's Elo displays as "AI" if isAI is true, otherwise as a number followed by "ELO". A ranked badge with sword emoji is prepended to the template only if isRanked is true. No loops, early returns, or branching beyond the conditional emoji and Elo display occur.

### renderTier2

When called, renderTier2 receives ten parameters: a stage HTMLElement, player and opponent player data (initials, names, Elo), a topic string, and boolean flags for AI opponent and ranked match. It returns void. The function writes to external state by setting the `innerHTML` property of the stage element with a template string laid out in three columns (left player, center VS, right opponent). The player's initials and name are escaped via `esc()`, as are the opponent's initials and name if not AI; if isAI is true, a robot emoji replaces the opponent's initials. Both players' Elo ratings are coerced to numbers; the opponent's Elo displays as "AI" if isAI is true, otherwise as a number followed by "ELO". The topic parameter is escaped via `esc()` and rendered below the matchup. A ranked badge with sword emoji is appended at the bottom only if isRanked is true. No loops, early returns, or control flow beyond the conditional emoji and Elo display occur.

### renderTier3

When called, renderTier3 receives twelve parameters: a stage HTMLElement, player data (initials, name, Elo, wins, losses), opponent data (initials, name, Elo), a topic string, and boolean flags for AI opponent and ranked match. It returns void. The function writes to external state by setting the `innerHTML` property of the stage element with a template string containing a background layer, scanline effect, title, left player panel, center divider, right opponent panel, and topic line. The title is conditionally set to a ranked battle message if isRanked is true, otherwise "ENTER THE ARENA". The player's initials and name are escaped via `esc()`, and the player's Elo is coerced to a number; wins and losses are each coerced to numbers and formatted as "W – L". The opponent's avatar is conditionally a robot emoji if isAI is true, or the escaped oppI otherwise. The opponent's Elo is conditionally "AI SPARRING" if isAI is true, otherwise a number followed by "ELO". The opponent's record placeholder is rendered as a non-breaking space. The topic parameter is escaped via `esc()`. No loops, early returns, or control flow beyond the conditional emoji and Elo display occur.

## Agent 02

### renderTier1

The function accepts a DOM element (`stage`), player identity (string `myI`), player name (`myName`), player rating (`myElo`), opponent identity (`oppI`), opponent name (`oppName`), opponent rating (`oppElo`), a boolean indicating AI opponent (`isAI`), and a boolean indicating ranked play (`isRanked`). It reads none of the module-level state. It calls the `escapeHTML` function (aliased as `esc`) twice for the player identity and name, and twice again for the opponent identity and name. It writes to the `innerHTML` property of the `stage` element with a template string that renders the match UI. The template conditionally includes a "RANKED BATTLE" badge at the top if `isRanked` is true. It constructs a matchup display with two debaters: the player's side shows the escaped identity as avatar, escaped name, and numeric ELO; the opponent's side shows a robot emoji if `isAI` is true or the escaped opponent identity otherwise, the escaped opponent name, and either "AI" or the numeric ELO depending on `isAI`. The function returns void and performs no async operations.

### renderTier2

It calls `escapeHTML` (aliased as `esc`) twice for the player identity and name, twice for the opponent identity and name, and once for the topic. It writes to the `innerHTML` property of the `stage` element with a template string that renders a wider matchup display. The outer wrapper contains three columns: a left column for the player with escaped identity as avatar, escaped name, and numeric ELO; a center column displaying "VS"; a right column for the opponent with a robot emoji if `isAI` is true or the escaped opponent identity otherwise, the escaped opponent name, and either "AI SPARRING" or numeric ELO depending on `isAI`. The template includes an escaped topic displayed below the matchup. If `isRanked` is true, the template conditionally includes a "RANKED" badge positioned at the bottom. The function returns void and performs no async operations.

### renderTier3

The function accepts a DOM element (`stage`), player identity (`myI`), player name (`myName`), player rating (`myElo`), wins count (`wins`), losses count (`losses`), opponent identity (`oppI`), opponent name (`oppName`), opponent rating (`oppElo`), AI opponent boolean (`isAI`), topic string (`topic`), and ranked play boolean (`isRanked`). It reads none of the module-level state. It calls `escapeHTML` (aliased as `esc`) twice for the player identity and name, once for the player's record (constructed as numeric wins hyphen numeric losses), twice for the opponent identity and name, and once for the topic. It writes to the `innerHTML` property of the `stage` element with a template string that renders an elaborate arena entrance display. The title conditionally displays "⚔️ RANKED BATTLE" if `isRanked` is true or "ENTER THE ARENA" otherwise. The left panel shows the escaped player identity, escaped player name, numeric ELO, and a record constructed from numeric wins and losses. The center section contains two dividers with "VS" between them. The right panel shows a robot emoji if `isAI` is true or the escaped opponent identity otherwise, the escaped opponent name, the string "AI SPARRING" if `isAI` is true or numeric ELO otherwise, and a non-breaking space (no record). Below the main arena display, the template includes an escaped topic. The template also includes static decorative elements: a background div, a scanline div. The function returns void and performs no async operations.
