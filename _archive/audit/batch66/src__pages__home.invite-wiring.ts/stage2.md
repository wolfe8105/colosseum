## Agent 01

### wireInviteScreen
This function is called with an HTML container element, invite statistics object, and a claim callback. It is synchronous and returns void. The function reads the `stats.invite_url` property and queries the container for three types of buttons: `#invite-copy-btn`, `#invite-native-share`, and `.invite-claim-btn:not([disabled])`. For each button found, it attaches an event listener with click handlers.

When the copy button is clicked, the function awaits `navigator.clipboard.writeText()` with the invite URL. If successful, it synchronously updates the button text to "Copied!", then schedules a timeout callback to revert the text to "Copy" after 2000ms. If the clipboard write fails (caught in try/catch), it calls `showToast()` with an error message. When the native share button is clicked, the function awaits `navigator.share()` with a title and URL; errors from share (user cancellation or unsupported browser) are silently caught with an empty catch block. For each claim button, the function extracts `rewardId` and `rewardType` from the button's dataset attributes; if either is missing, the handler returns early, otherwise it calls the `onClaim` callback synchronously with those values. The function modifies no module state. A comment at lines 41-42 notes a potential hazard: `onClaim` may invoke an async function without error propagation.
