# Stage 2 Outputs ‚Äî settings.load.ts

## Agent 01

### loadSettings

`loadSettings` is a synchronous, void-returning function that takes no parameters. It reads from two external sources ‚Äî `localStorage` and in-memory auth state ‚Äî and writes values into the DOM.

It opens with a `try/catch` block that attempts to parse the string stored under the key `'colosseum_settings'` in `localStorage` into a `Partial<SettingsData>` object, assigning the result to `saved`. If `localStorage.getItem` returns `null`, the expression `|| '{}'` causes `JSON.parse` to produce an empty object. If `JSON.parse` throws (malformed JSON), the `catch` block sets `saved` to `{}` and calls `localStorage.removeItem('colosseum_settings')` to purge the corrupt entry.

With `saved` established, the function enters a DOM-population phase driven entirely by `localStorage` data. It calls `getEl<HTMLInputElement>('set-display-name')`, `getEl<HTMLInputElement>('set-username')`, and `getEl<HTMLTextAreaElement>('set-bio')` in sequence, setting each element's `.value` to the corresponding `saved` field if the element exists, falling back to an empty string via `??`. It then calls `getEl('set-bio-count')` and, if found, sets its `textContent` to the length of `saved.bio` (or `''`) concatenated with the string `'/160'`. It calls `getEl('set-email-display')` and assigns its `textContent` to `saved.email` or the literal `'‚Äî'`. It calls `validateTier(saved.subscription_tier)` to normalize the tier value to a known string, then calls `getEl('set-tier-badge')` and, if found, sets its `textContent` to the corresponding entry in `TIER_LABELS` and assigns its `className` to `'tier-badge '` plus the tier name if the tier is not `'free'`, or an empty string otherwise.

Next, it calls `setChecked` eleven times, one per toggle element in the form. For the notification and audio/privacy flags ‚Äî `set-notif-challenge`, `set-notif-debate`, `set-notif-follow`, `set-notif-reactions`, `set-notif-rivals`, `set-notif-economy`, `set-audio-sfx`, `set-privacy-public`, `set-privacy-online`, and `set-privacy-challenges` ‚Äî it passes `saved.<field> !== false`, so any missing or non-`false` value defaults the checkbox to checked. For `set-audio-mute` it passes `saved.audio_mute === true`, so the checkbox defaults to unchecked. For `set-dark-mode` it reads `document.documentElement.getAttribute('data-theme')` and passes `true` when that attribute is not `'light'`, meaning the checkbox reflects the current live theme attribute rather than anything stored in `saved`. It then calls `getEl<HTMLSelectElement>('set-language')` and, if found, sets its `.value` to `saved.preferred_language` or `'en'`.

After the `localStorage`-based pass, the function calls `getCurrentProfile()` to retrieve the in-memory profile object. If the result is truthy, it enters a second DOM-population pass that overwrites fields with profile data where present, using `saved` values as fallbacks. It casts the profile to `Record<string, unknown>` and re-assigns `nameEl.value`, `userEl.value`, `bioEl.value`, and `bioCount.textContent` using `p.display_name`, `p.bio`, etc., each falling back to the corresponding `saved` field and then to an empty string. It calls `getCurrentUser()`, casts the result, and replaces `emailDisp.textContent` with `user?.email`, falling back through `saved.email` to `'‚Äî'`. It calls `validateTier` again on `p.subscription_tier` and overwrites the badge's `textContent` and `className` using the same logic as the first pass. If `p.preferred_language` is a string, it sets `langSelect.value` to it. If `p.is_private` is a boolean, it calls `setChecked('set-privacy-public', !p.is_private)`, inverting the boolean so that a private profile unchecks the "public" toggle. No element guards are repeated in this second pass ‚Äî the existing `nameEl`, `userEl`, `bioEl`, `bioCount`, `emailDisp`, `badge`, and `langSelect` references from the first pass are reused, and each write is conditioned on the same reference being non-null. If `getCurrentProfile()` returns `null` or `undefined`, the entire second pass is skipped and the `localStorage`-derived values remain in the DOM.

### loadSettings

`loadSettings` is a synchronous, void-returning function that takes no parameters. It reads from two external sources ‚Äî `localStorage` and in-memory module state accessed via imported functions ‚Äî and writes exclusively to the DOM.

The function opens with a `try/catch` block that attempts to parse the string stored at the `'colosseum_settings'` key in `localStorage` into a `Partial<SettingsData>` object assigned to `saved`. With `saved` in hand, the function populates a first wave of DOM fields using `getEl`, a typed wrapper that queries elements by ID. For each of three text inputs ‚Äî `'set-display-name'`, `'set-username'`, and `'set-bio'` ‚Äî it fetches the element and, if present, sets its `.value` to the corresponding property on `saved`, falling back to an empty string via the `??` operator. It then reads the `'set-bio-count'` element and, if present, sets its `textContent` to the length of `saved.bio` (or zero) concatenated with the string `'/160'`. The `'set-email-display'` element receives `saved.email` or the literal `'‚Äî'` as its `textContent`.

Next, it calls `validateTier` with `saved.subscription_tier` to produce a validated `tier` value, then queries the `'set-tier-badge'` element. If that element exists, it sets `textContent` to `TIER_LABELS[tier]` (a lookup into an imported constant map) and sets `className` to `'tier-badge '` followed by the tier string if the tier is not `'free'`, or an empty string if it is.

The function then calls `setChecked` eleven times in sequence to set the checked state of eleven checkbox inputs by ID. Six notification toggles (`'set-notif-challenge'`, `'set-notif-debate'`, `'set-notif-follow'`, `'set-notif-reactions'`, `'set-notif-rivals'`, `'set-notif-economy'`) and `'set-audio-sfx'` and `'set-privacy-public'`, `'set-privacy-online'`, `'set-privacy-challenges'` default to `true` ‚Äî specifically, they are checked when the corresponding `saved` property is `!== false`, meaning a missing or `undefined` value resolves as checked. `'set-audio-mute'` is the single exception: it is only checked when `saved.audio_mute === true`. The language select element `'set-language'` is then set to `saved.preferred_language ?? 'en'`.

The dark-mode checkbox `'set-dark-mode'` is set by reading the live DOM attribute `document.documentElement.getAttribute('data-theme')` and checking whether it is not equal to `'light'`; this is the only value derived from neither `saved` nor the profile.

The function then calls `getCurrentProfile()` (imported from `auth.ts`), which returns an in-memory profile object or null. If a profile is returned, it enters a block that re-applies or overrides the values already written to the DOM. The profile is cast to `Record<string, unknown>` as `p`. The text inputs for display name, username, and bio are overwritten using profile values with fallback to the `saved` values already used. The bio character count is recalculated in the same pattern. `getCurrentUser()` (also imported from `auth.ts`) is then called and cast to `{ email?: string } | null`; the email display element is set to `user.email`, falling back to `saved.email` and then `'‚Äî'`. `validateTier` is called again on `p.subscription_tier` to produce `profileTier`, and the tier badge's text and class are rewritten with the same conditional logic used in the first pass. If `p.preferred_language` is a string, the language select's value is overwritten with it. Finally, if `p.is_private` is a boolean, `setChecked` is called for `'set-privacy-public'` with the boolean negated ‚Äî this overrides the earlier `setChecked` call that used `saved.privacy_public`. If `getCurrentProfile()` returns null, the entire profile block is skipped and the DOM retains the values written from `saved`.
