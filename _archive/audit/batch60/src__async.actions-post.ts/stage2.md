# Stage 2 Outputs — async.actions-post.ts

## Agent 01

### postTake

`postTake` is an async function that returns `Promise<void>`. It reads three sources of module-level state from `async.state.ts`: `state.postingInFlight`, `state.currentFilter`, and `state.hotTakes`. It reads two feature flags/guards from `config.ts` and `auth.ts`: `FEATURES.hotTakes` and `requireAuth`. It reads the current authenticated user and profile via `getCurrentUser()` and `getCurrentProfile()` from `auth.ts`. It reads the DOM directly by calling `document.getElementById('hot-take-input')` cast to `HTMLTextAreaElement | null`, and reads `.value` from that element.

The function opens with three sequential early-return guards before touching any state: it returns immediately if `FEATURES.hotTakes` is falsy, if `requireAuth('post hot takes')` returns falsy (which fires an auth prompt as a side effect), or if `state.postingInFlight` is already `true`. If all three pass, it sets `state.postingInFlight = true` and enters a `try/finally` block that unconditionally resets `state.postingInFlight` to `false` on exit. Two further early returns exist inside the `try`: if the textarea element is not found in the DOM, and if its trimmed value is empty string — both return without clearing `postingInFlight` until `finally` runs.

When content exists, the function constructs a `HotTake` object (`newTake`) with an optimistic temporary `id` of the form `'t_' + Date.now()`, a `user_id` from `getCurrentUser()?.id` (falling back to empty string), a `user` field from `getCurrentProfile()?.username` (falling back to `'YOU'`) uppercased, an `elo` from the profile's `elo_rating` (falling back to `1200`), and a `section` derived from `state.currentFilter` — substituting `'trending'` when the filter is `'all'`. It snapshots the current `state.hotTakes` array via spread into a local `snapshot` variable, prepends `newTake` to `state.hotTakes` via `unshift`, clears `input.value` to empty string, then calls `loadHotTakes(state.currentFilter)` synchronously to re-render the feed with the optimistic entry — this call is not awaited.

The function then checks whether a real Supabase client is available (`getSupabaseClient()` is truthy) and placeholder mode is inactive (`!getIsPlaceholderMode()`). If either condition fails, no network call is made and the optimistic state is left in place. When both conditions are met, it enters an inner `try/catch` and `await`s `safeRpc<CreateHotTakeResult>('create_hot_take', { p_content: text, p_section: section })`. If the RPC response contains an `error`, the function logs it to `console.error`, restores `state.hotTakes` to `snapshot`, calls `loadHotTakes` again to revert the feed, and calls `showToast('Post failed — try again', 'error')`. If the response succeeds and `data` contains an `id`, the optimistic `newTake.id` is mutated in place to the server-assigned ID, and `claimHotTake` from `tokens.ts` is called with that ID — `claimHotTake` is called without `await` (fire-and-forget). The inner `catch` block, which fires on a thrown exception from `safeRpc`, executes the same rollback: restores `snapshot`, re-renders via `loadHotTakes`, and shows the error toast.

## Agent 02

### postTake

`postTake` is an async function that takes no parameters and returns `Promise<void>`. It reads three module-level state variables from `state` (imported from `async.state.ts`): `state.postingInFlight`, `state.hotTakes`, and `state.currentFilter`. It calls four imported functions from `auth.ts` — `requireAuth`, `getCurrentProfile`, `getCurrentUser`, `getSupabaseClient`, and `getIsPlaceholderMode` — and uses the feature flag `FEATURES.hotTakes` from `config.ts`.

The function opens with three consecutive early-return guards: it returns immediately if `FEATURES.hotTakes` is falsy, if `requireAuth('post hot takes')` returns a falsy value (which also triggers an auth prompt as a side effect), or if `state.postingInFlight` is already `true`. Passing all three guards, it sets `state.postingInFlight = true` and enters a `try/finally` block that unconditionally resets `state.postingInFlight = false` on exit regardless of the execution path.

Inside the `try` block, it queries the DOM for an element with id `'hot-take-input'` cast to `HTMLTextAreaElement | null`. If that element is absent it returns early (the `finally` still fires). It reads `input.value`, trims it, and returns early if the trimmed string is empty. It then calls `getCurrentProfile()` and `getCurrentUser()` to assemble a local optimistic `HotTake` object. The `section` field is derived from `state.currentFilter`: if the filter is `'all'`, section is coerced to `'trending'`; otherwise the filter value is used directly. The optimistic take's `id` is a string prefixed with `'t_'` followed by `Date.now()`, and its `user_id` field is the current user's `id` or an empty string if no user is available.

It captures a shallow copy of `state.hotTakes` into a local `snapshot` variable, then prepends the optimistic take to `state.hotTakes` with `unshift`, clears `input.value`, and synchronously calls `loadHotTakes(state.currentFilter)` to re-render the feed with the optimistic entry in place.

If both `getSupabaseClient()` is truthy and `getIsPlaceholderMode()` is falsy, the function enters a nested `try/catch` block and awaits `safeRpc<CreateHotTakeResult>('create_hot_take', { p_content: text, p_section: section })`. On a non-throwing response, if `error` is present it logs the error to the console, reverts `state.hotTakes` to `snapshot`, calls `loadHotTakes` again to re-render the reverted state, and calls `showToast('Post failed — try again', 'error')`. On a successful response where `data` contains an `id`, it overwrites the optimistic take's `id` with the server-assigned id and calls `claimHotTake` from `tokens.ts` with that id as a fire-and-forget side effect (the return value is not awaited or checked). If the `safeRpc` call throws rather than returning an error, the `catch` block performs the same rollback and toast as the error branch. If `getSupabaseClient()` is falsy or `getIsPlaceholderMode()` is truthy, the function skips the RPC entirely, leaving the optimistic take in place as the permanent state.

### postTake

`postTake` is an async function that takes no parameters and returns `Promise<void>`. It reads the feature flag `FEATURES.hotTakes` from `config.ts` as the first guard: if the flag is falsy it returns immediately. It then calls `requireAuth('post hot takes')`, which gates the call on the user being authenticated, returning early if not. It reads `state.postingInFlight` from the `async.state` module-level `state` object, returning immediately if a post is already in progress, then sets `state.postingInFlight = true` before entering a `try/finally` block that resets that flag unconditionally on exit.

Inside the `try`, the function queries the DOM for the element with id `'hot-take-input'`, casting it to `HTMLTextAreaElement | null`. If the element is not found it returns without posting. If `getSupabaseClient()` returns a truthy value and `getIsPlaceholderMode()` returns falsy, the function proceeds to call `safeRpc<CreateHotTakeResult>('create_hot_take', { p_content: text, p_section: section })` and awaits the result inside a nested `try/catch`. On error (the `error` property is truthy), it logs to `console.error`, overwrites `state.hotTakes` with the pre-optimistic `snapshot`, calls `loadHotTakes` again to roll back the rendered list, and calls `showToast('Post failed — try again', 'error')`. If that element is not found, it returns early (the `finally` still fires, clearing the in-flight flag). It then reads `input.value.trim()` into `text`; if `text` is empty, it also returns early. It reads `getCurrentProfile()` into `profile` and derives `section` from `state.currentFilter`: if `currentFilter` is `'all'`, `section` is set to `'trending'`; otherwise `section` takes the value of `currentFilter` directly.
