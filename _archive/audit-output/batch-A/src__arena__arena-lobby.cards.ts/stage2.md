# Stage 2 Outputs — arena-lobby.cards.ts

## Agent 01

### renderArenaFeedCard
`renderArenaFeedCard` accepts two parameters: `d`, an `ArenaFeedItem`, and `_type`, a string that is unused. It derives five boolean/string locals from `d` before building any markup. `isAuto` is set to `true` if `d.source` strictly equals `'auto_debate'`; `isLive` is set to `true` if `d.status` strictly equals `'live'`. `rulesetBadge` is a non-empty span string only when `d.ruleset` equals `'unplugged'`, otherwise an empty string. `roundsBadge` is a non-empty span string only when `d.total_rounds` is truthy and not equal to `4`, injecting `d.total_rounds` directly as the label; otherwise an empty string. `badge` is determined by a three-way priority: if `isLive` is true it is a LIVE badge span; else if `isAuto` is true it is an AI DEBATE badge span; otherwise it is a VERDICT badge span. `votes` is computed as the numeric sum of `d.vote_count_a` (defaulting to `0` if falsy) and `d.vote_count_b` (defaulting to `0` if falsy). `action` is the string `'SPECTATE'` when `isLive` is true, otherwise `'VIEW'`. `cardClass` is `'card-live'` when `isLive` is true, `'card-ai'` when `isAuto` is true, or an empty string otherwise. `cardAttr` is determined by `isLive`: if true, it produces a `data-debate-id` attribute with `d.id` passed through `encodeURIComponent`; if false, it produces a `data-link` attribute whose value is either `'/verdict?id=' + encodeURIComponent(d.id)` when `isAuto` is true, or `'/debate/' + encodeURIComponent(d.id)` otherwise. The function then returns a single HTML string representing a `div.arena-card` with `cardClass` and `cardAttr` injected. Inside, the top row injects `badge`, `rulesetBadge`, `roundsBadge`, and a vote-count meta span using `votes` with correct singular/plural suffix. The topic div renders `d.topic` (falling back to `'Untitled Debate'`) through `escapeHTML`. The VS div renders `d.debater_a_name` (falling back to `'Side A'`) and `d.debater_b_name` (falling back to `'Side B'`) both through `escapeHTML`, and conditionally appends a score span only when `d.score_a` is not `null` and not `undefined`, casting both `d.score_a` and `d.score_b` through `Number()` before injection. The action div renders a button whose text is `action`.

### renderAutoDebateCard
`renderAutoDebateCard` accepts a single parameter `d` of type `AutoDebateItem`. It returns a single HTML string representing a `div.arena-card.card-ai` with a `data-link` attribute set to `'moderator-auto-debate.html?id='` concatenated with `encodeURIComponent(d.id)`. The top row contains a single hardcoded AI DEBATE badge span. The topic div renders `d.topic` through `escapeHTML`. The VS div renders `d.side_a_label` and `d.side_b_label` both through `escapeHTML`, and unconditionally appends a score span rendering `d.score_a` and `d.score_b` each cast through `Number()`. The action div contains a button with the hardcoded label `'VIEW'`. No conditional logic is present; all fields on `d` are treated as required and rendered unconditionally.

### renderPlaceholderCards
`renderPlaceholderCards` accepts one parameter `type`, a string. If `type` strictly equals `'live'`, the function immediately returns a single hardcoded `div.arena-empty` string containing a temple emoji and the message `'No live debates yet — be the first to enter the arena'`, with no further processing. For all other values of `type`, the function defines a local array `placeholders` containing three hardcoded objects, each with properties `topic` (string), `a` (string), `b` (string), `sa` (number), and `sb` (number). It maps over this array, and for each element `p` produces an HTML string for a `div.arena-card` containing: a top row with a hardcoded VERDICT badge span; a topic div injecting `p.topic` directly without escaping; a VS div injecting `p.a`, `p.b`, and a score span with `p.sa` and `p.sb` each cast through `Number()`. None of the placeholder string values pass through `escapeHTML` since they are hardcoded literals, not user-supplied data. The mapped array of strings is joined with an empty string and returned as a single HTML string.

## Agent 02

### renderArenaFeedCard
Takes an `ArenaFeedItem` object `d` and a `_type` string (unused). Computes five boolean/string locals from `d`'s fields before building the HTML string. `isAuto` is true when `d.source === 'auto_debate'`. `isLive` is true when `d.status === 'live'`. `rulesetBadge` is a span with class `unplugged` when `d.ruleset === 'unplugged'`, otherwise an empty string. `roundsBadge` is a span showing the round count when `d.total_rounds` is truthy and not equal to 4, otherwise an empty string. `badge` is one of three spans: a LIVE badge when `isLive` is true, an AI DEBATE badge when `isAuto` is true, or a VERDICT badge otherwise. `votes` is the numeric sum of `d.vote_count_a` and `d.vote_count_b`, defaulting each to 0 when nullish. `action` is the string `'SPECTATE'` when `isLive` is true, otherwise `'VIEW'`. `cardClass` is `'card-live'` when `isLive` is true, `'card-ai'` when `isAuto` is true, or an empty string otherwise. `cardAttr` is a `data-debate-id` attribute with the URI-encoded debate ID when `isLive` is true; otherwise it is a `data-link` attribute pointing to `/verdict?id=` + URI-encoded ID for auto debates, or `/debate/` + URI-encoded ID for all other debates. Returns a single HTML string: a div with class `arena-card` plus `cardClass` and the chosen attribute, containing four child divs. The top div concatenates `badge`, `rulesetBadge`, `roundsBadge`, and a meta span showing the vote count with correct singular/plural suffix. The topic div renders `d.topic` through `escapeHTML`, falling back to `'Untitled Debate'`. The VS div renders `d.debater_a_name` and `d.debater_b_name` through `escapeHTML` with fallbacks `'Side A'` and `'Side B'`; when `d.score_a` is not null, appends a score span showing `Number(d.score_a)` and `Number(d.score_b)` separated by an en-dash. The action div holds a button with the `action` label.

### renderAutoDebateCard

The action div holds a button with the fixed label `'VIEW'`. No conditional branches exist; all fields are rendered directly with no fallback values.

### renderPlaceholderCards

For all other values of `type`, defines a hardcoded array of three plain objects, each carrying `topic`, `a`, `b`, `sa`, and `sb` fields with literal string and number values. Maps over that array, producing one HTML string per object: a div with class `arena-card` containing three child divs. The top div holds a VERDICT badge span. The topic div renders the literal `p.topic` string directly into the template without escaping. The VS div renders the literal `p.a` and `p.b` label strings directly without escaping, separated by a VS span, followed by a score span showing `Number(p.sa)` and `Number(p.sb)` separated by an en-dash. Joins all three strings with empty delimiter and returns the concatenated result. No user-supplied data enters the output; all content is static.

### renderArenaFeedCard

Inside, the top row renders the three badge strings and a vote count span with pluralized label. The topic div renders `d.topic` through `escapeHTML`, falling back to `'Untitled Debate'`. The VS row renders `d.debater_a_name` and `d.debater_b_name` through `escapeHTML` with fallbacks, and conditionally appends a score span when `d.score_a` is not null, casting both `d.score_a` and `d.score_b` through `Number()` before insertion. ### renderPlaceholderCards
