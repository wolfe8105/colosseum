# Stage 2 Outputs — arena-feed-disconnect.ts

## Agent 01

### handleParticipantGone
`handleParticipantGone` takes a `role` string parameter. It first reads the module-level `disconnectHandled` boolean and returns immediately if it is `true`. If not, it calls `set_disconnectHandled(true)` to set that flag. It reads `currentDebate` into the local `debate` variable and returns immediately if `debate` is falsy. It then returns immediately if any of the following are truthy: `debate.concededBy`, `debate._nulled`, or if the module-level `phase` equals the string `'finished'`. Past those guards, it calls `clearFeedTimer()`, `stopTranscription()`, `clearInterimTranscript()`, and `stopHeartbeat()` with no arguments. It then checks whether both `debate.modView` and `debate.spectatorView` are falsy; if so, it calls `setDebaterInputEnabled(false)`. Next it branches on `role`. If `role === 'mod'`, it calls `handleModDisconnect(debate)` as a void async call. If `role === 'a'` or `role === 'b'`, it checks `debate.modView` first; if truthy, it calls `handleDebaterDisconnectAsViewer(debate, role)`. Otherwise it checks `debate.spectatorView`; if truthy, it also calls `handleDebaterDisconnectAsViewer(debate, role)`. If neither view flag is set, it calls `handleDebaterDisconnect(debate, role)` as a void async call.

### handleDebaterDisconnect
`handleDebaterDisconnect` is an async function taking `debate` (a `CurrentDebate`) and `disconnectedSide` (`'a'` or `'b'`). It resolves `disconnectorName` by checking `disconnectedSide`: if `'a'`, it checks whether `debate.role === 'a'`; if so, it reads `getCurrentProfile()?.display_name`, falling back to the string `'You'`; otherwise it uses `debate.opponentName`. If `disconnectedSide` is `'b'`, the same logic applies with `debate.role === 'b'`. It calls `writeFeedEvent('disconnect', \`${disconnectorName} disconnected.\`, 'mod')` as a void async call, then calls `addLocalSystem(\`${disconnectorName} disconnected.\`)`. It reads `scoreA` and `scoreB` from module scope into `disconnectorScore` (the score for `disconnectedSide`) and `opponentScore` (the opposite score). It then branches: if `disconnectorScore > opponentScore`, it sets `debate._nulled = true` and `debate._nullReason` to a string containing `disconnectorName` followed by `' disconnected while ahead — debate nulled'`, then awaits `safeRpc('update_arena_debate', { p_debate_id: debate.id, p_status: 'cancelled', p_current_round: round })`, catching any error with a `console.warn` call prefixed `'[FeedRoom] cancel debate failed:'`. In the else branch (disconnector score is not greater than opponent score), it sets `winnerSide` to `'b'` if `disconnectedSide === 'a'`, else `'a'`, then awaits `safeRpc('update_arena_debate', { p_debate_id: debate.id, p_status: 'complete', p_current_round: round, p_winner: winnerSide, p_score_a: scoreA, p_score_b: scoreB })`, catching errors with a `console.warn` prefixed `'[FeedRoom] finalize debate failed:'`. After both branches, it calls `setTimeout` with a callback that calls `endCurrentDebate()` as a void async call, with a delay of `1500` milliseconds.

### handleDebaterDisconnectAsViewer
`handleDebaterDisconnectAsViewer` is a synchronous function taking `debate` (a `CurrentDebate`) and `disconnectedSide` (`'a'` or `'b'`). It resolves `disconnectorName`: if `disconnectedSide === 'a'`, it uses `debate.debaterAName` falling back to the string `'Side A'`; otherwise it uses `debate.debaterBName` falling back to `'Side B'`. It calls `addLocalSystem(\`${disconnectorName} disconnected.\`)` and then calls `showDisconnectBanner(\`${disconnectorName} disconnected — debate ending\`)`. It then checks `debate.spectatorView`; if truthy, it calls `setTimeout` with a delay of `5000` milliseconds. The callback removes the element with id `'feed-disconnect-banner'` from the DOM if it exists, calls `cleanupFeedRoom()`, then calls `import('./arena-lobby.ts')` and, on resolution, calls `m.renderLobby()`. If `debate.spectatorView` is falsy, the function returns without scheduling the timeout.

### handleModDisconnect
`handleModDisconnect` is an async function taking `debate` (a `CurrentDebate`). It reads `debate.moderatorName`, falling back to the string `'Moderator'`, into `modName`. It calls `writeFeedEvent('disconnect', \`${modName} disconnected.\`, 'mod')` as a void async call. It calls `addLocalSystem(\`${modName} disconnected — debate nulled.\`)`. It calls `showDisconnectBanner('Moderator disconnected — debate nulled')`. It sets `debate._nulled = true` and sets `debate._nullReason` to the string `'Moderator disconnected — debate nulled'`. It awaits `safeRpc('record_mod_dropout', { p_debate_id: debate.id })`, catching any error with a `console.warn` prefixed `'[FeedRoom] record_mod_dropout failed:'`. After the await, it calls `setTimeout` with a callback that calls `endCurrentDebate()` as a void async call, with a delay of `1500` milliseconds.

### modNullDebate
`modNullDebate` is an exported async function taking `reason`, which is one of the string literals `'eject_a'`, `'eject_b'`, or `'null'`. It reads the module-level `disconnectHandled` boolean and returns immediately if `true`. Otherwise it calls `set_disconnectHandled(true)`. It reads `currentDebate` into `debate` and returns immediately if falsy. It calls `clearFeedTimer()`, `stopTranscription()`, `clearInterimTranscript()`, and `stopHeartbeat()` with no arguments. It then resolves `msg`: if `reason === 'null'`, `msg` is set to `'Moderator nulled the debate.'`. Otherwise, it reads `debate.debaterAName` (falling back to `'Side A'`) if `reason === 'eject_a'`, or `debate.debaterBName` (falling back to `'Side B'`) if `reason === 'eject_b'`, storing the result in `ejectedName`, and sets `msg` to `\`${ejectedName} ejected by moderator.\``. It calls `writeFeedEvent('disconnect', msg, 'mod')` as a void async call and calls `addLocalSystem(msg)`. It sets `debate._nulled = true` and `debate._nullReason = msg`. It awaits `safeRpc('mod_null_debate', { p_debate_id: debate.id, p_reason: reason })`, catching any error with a `console.warn` prefixed `'[FeedRoom] mod_null_debate failed:'`. After the await, it calls `setTimeout` with a callback that calls `endCurrentDebate()` as a void async call, with a delay of `1500` milliseconds.

### showDisconnectBanner
`showDisconnectBanner` is an exported synchronous function taking a `message` string. It calls `document.getElementById('feed-disconnect-banner')` and, if the element exists, calls `.remove()` on it. It creates a new `div` element via `document.createElement('div')`, sets its `id` to `'feed-disconnect-banner'`, sets its `className` to `'feed-disconnect-banner'`, and sets its `textContent` to `message`. It calls `document.querySelector('.feed-room')` into `room`. If `room` is truthy, it calls `room.prepend(banner)`, inserting the new banner as the first child of the `.feed-room` element. If no `.feed-room` element exists, the banner is created but not inserted into the DOM.

`handleModDisconnect` is a private async function accepting `debate: CurrentDebate`, returning `Promise<void>`. It reads `debate.moderatorName` with fallback `'Moderator'` into `modName`. It calls `writeFeedEvent('disconnect', \`${modName} disconnected.\`, 'mod')` as a void-discarded promise. It calls `addLocalSystem` with `\`${modName} disconnected — debate nulled.\``. It calls `showDisconnectBanner('Moderator disconnected — debate nulled')`. It then sets `debate._nulled = true` and `debate._nullReason = 'Moderator disconnected — debate nulled'` directly on the `debate` object. It `await`s `safeRpc('record_mod_dropout', { p_debate_id: debate.id })`, catching any rejection with `console.warn`. After the await it schedules `endCurrentDebate()` via `setTimeout` at 1500 ms, discarding the returned promise with `void`.

### modNullDebate
`modNullDebate` is an exported async function accepting `reason: 'eject_a' | 'eject_b' | 'null'`, returning `Promise<void>`. It reads `disconnectHandled`; if truthy it returns immediately. Otherwise it calls `set_disconnectHandled(true)`. It reads `currentDebate` into `debate`; if falsy it returns. It then calls `clearFeedTimer()`, `stopTranscription()`, `clearInterimTranscript()`, and `stopHeartbeat()` in sequence. It declares `msg: string` and branches on `reason`: if `'null'`, assigns `msg = 'Moderator nulled the debate.'`; otherwise computes `ejectedName` by reading `debate.debaterAName` with fallback `'Side A'` when `reason === 'eject_a'`, or `debate.debaterBName` with fallback `'Side B'` otherwise, and assigns `msg = \`${ejectedName} ejected by moderator.\``. It calls `writeFeedEvent('disconnect', msg, 'mod')` as a void-discarded promise and calls `addLocalSystem(msg)`. It sets `debate._nulled = true` and `debate._nullReason = msg`. It `await`s `safeRpc('mod_null_debate', { p_debate_id: debate.id, p_reason: reason })`, catching any rejection with `console.warn`. After the await it schedules `endCurrentDebate()` via `setTimeout` at 1500 ms, voiding the promise.

### showDisconnectBanner
`showDisconnectBanner` is an exported synchronous function accepting `message: string`, returning `void`. It first calls `document.getElementById('feed-disconnect-banner')?.remove()`, removing any existing element with that id from the DOM. It then calls `document.createElement('div')` and assigns the result to `banner`. It sets `banner.id = 'feed-disconnect-banner'`, `banner.className = 'feed-disconnect-banner'`, and `banner.textContent = message`. It then calls `document.querySelector('.feed-room')` into `room`; if `room` is truthy it calls `room.prepend(banner)`, inserting `banner` as the first child of the `.feed-room` element. If `room` is falsy the banner element is created but never inserted into the document.
