# Stage 2 Outputs — arena-room-setup.ts

## Agent 01

### showPreDebate

`showPreDebate` is async and accepts a `CurrentDebate` object as its sole parameter. It begins by writing module-level state: it calls `set_view('room')`, calls `pushArenaState('preDebate')`, and calls `set_currentDebate(debateData)`. It clears `screenEl.innerHTML` if `screenEl` is non-null. It resets power-up state by calling `activatedPowerUps.clear()`, `set_shieldActive(false)`, and `set_equippedForDebate([])`. If the module-level `silenceTimer` is non-null, it calls `clearInterval` on it and then `set_silenceTimer(null)`. It reads `getCurrentProfile()` from auth to extract `display_name`, `username`, `token_balance`, and `elo_rating`.

It then builds a `div.arena-pre-debate` element by direct `innerHTML` assignment. Every user-supplied string passed into that HTML goes through `escapeHTML`. The rendered HTML includes a rank/ruleset badge, a VS bar showing both debaters' names and ELO, and five empty container `div`s (`pre-debate-presets`, `pre-debate-staking`, `pre-debate-loadout`, `pre-debate-refs`, `pre-debate-bounty`), a share button, and an "ENTER BATTLE" button. The element is appended to `screenEl`. After appending, `injectAdSlot` is called on the element — the source of `injectAdSlot` is not visible in this file, so it is unclear what module provides it. Three network-dependent sections follow, each in its own `try/catch` that swallows errors with a `console.warn` on failure. First, it awaits `getPool(debateData.id)` and, if `#pre-debate-staking` is found in the DOM, sets its `innerHTML` to the result of `renderStakingPanel(...)` then calls `wireStakingPanel(debateData.id)`. Second, it awaits `getMyPowerUps(debateData.id)` and, if `#pre-debate-loadout` is found, sets its `innerHTML` to the result of `renderLoadout(...)` then calls `wireLoadout(debateData.id, callback)` where the callback is a fire-and-forget call to `showPreDebateLoadout`. Third, if `#pre-debate-refs` exists and `debateData.mode` is not `'ai'`, it awaits `renderLoadoutPicker(refsEl, debateData.id)`. After that, if `#pre-debate-presets` exists and `debateData.mode` is not `'ai'`, it fire-and-forgets `renderPresetBar(...)` with both the refs and loadout container elements. If `#pre-debate-bounty` exists and the debate is ranked, has a non-null `opponentId`, and is not AI mode, it calls `resetBountyClaim()` and fire-and-forgets `renderBountyClaimDropdown(...)`.

Two event listeners are then attached. The enter button listener disables the button and changes its text to `'ENTERING...'`, then awaits `getMyPowerUps(debateData.id)` and calls `set_equippedForDebate` with the result (or with `[]` on error), then calls `enterRoom(debateData)` — synchronously, not awaited. The share button listener constructs a spectator URL using `window.location.origin` and `debateData.id`, then calls `navigator.clipboard.writeText`. On success it shows and hides `#pre-debate-share-confirm` via `setTimeout`. On failure it falls back to a `textarea`-based `document.execCommand('copy')` approach and then performs the same show/hide behavior.

### showPreDebateLoadout

`showPreDebateLoadout` is async and accepts a `CurrentDebate` and an `HTMLElement` container. It returns immediately if `container` is falsy. Inside a `try` block with a silent `catch`, it awaits `getMyPowerUps(debateData.id)`, sets `container.innerHTML` to the result of `renderLoadout(...)` using inventory, equipped, and `questions_answered` from the returned data (defaulting each to empty array or `0` if absent), then calls `wireLoadout(debateData.id, callback)` where the callback is a recursive fire-and-forget call to `showPreDebateLoadout` with the same arguments. There is no error logging — the `catch` block is empty.

### enterRoom

`enterRoom` is synchronous and accepts a `CurrentDebate`. It immediately calls `set_view('room')`. It then dynamically imports `./arena-sounds.ts` as a fire-and-forget promise, calling `stopIntroMusic()` on resolution and silently swallowing any rejection. If `debate.mode === 'live'`, it calls `nudge('enter_debate', ...)` to show a toast, then — if `isPlaceholder()` returns false and `debate.id` does not start with `'placeholder-'` — it fire-and-forgets `safeRpc('update_arena_debate', { p_debate_id: debate.id, p_status: 'live' })`, logging a warning on rejection. It then calls `enterFeedRoom(debate)` and returns early, skipping all remaining code. For all other modes, it dynamically imports `./arena-entrance.ts` and calls `playEntranceSequence(debate)`. Whether that promise resolves or rejects (both paths are covered by `.finally`), it calls `_renderRoom(debate)` synchronously as the `.finally` callback. The dynamic import rejection is silently swallowed by the `.catch(() => {})` before `.finally`.

### _renderRoom

`_renderRoom` is synchronous and accepts a `CurrentDebate`. It calls `pushArenaState('room')`, `set_currentDebate(debate)`, and clears `screenEl.innerHTML` if non-null. It calls `nudge('enter_debate', ...)` to display a toast. If `debate.mode === 'ai'` and `debate.id` does not start with `'ai-local-'` or `'placeholder-'` and `isPlaceholder()` is false, it fire-and-forgets `safeRpc('update_arena_debate', { p_debate_id: debate.id, p_status: 'live' })`, logging a warning on rejection.

It reads `getCurrentProfile()` and derives several display values from it. `isModView` is set to `debate.modView === true`. `myName` is the profile's `display_name` or `username` if not mod view, or `debate.debaterAName` (falling back to `'Debater A'`) if mod view. `oppName` is `debate.opponentName` normally or `debate.debaterBName` (falling back to `'Debater B'`) if mod view. Boolean flags `isAI` and `isUnplugged` are derived from `debate.mode` and `debate.ruleset`. A `div.arena-room` element is built via `innerHTML` with the room header (topic, round label, and a timer element only if `debate.mode === 'live'`), a conditional AI badge, a VS bar, a power-up loadout container (hidden via `style` if `isUnplugged`), a conditional spectator bar (omitted if `isUnplugged`), an empty messages div, and an input area (hidden via `style` if `isModView`). All user-supplied strings go through `escapeHTML`. The element is appended to `screenEl`.

After the DOM is in place, `addSystemMessage` is called with a round announcement string. If the module-level `selectedModerator` is truthy and `debate.id` is truthy, `assignSelectedMod(debate.id)` is called as a fire-and-forget void, and three properties are written directly onto the `debate` object (`moderatorType`, `moderatorId`, `moderatorName`). A moderator bar `div` is then inserted into the DOM immediately after `.arena-vs-bar` using `insertBefore`.

If not `isUnplugged` and `debate.id` is non-empty and does not start with `'placeholder-'`, an immediately-invoked async arrow function awaits `getMyPowerUps(debate.id)` and, if both `#powerup-loadout-container` and the returned data are truthy, sets its `innerHTML` via `renderLoadout(...)` and calls `wireLoadout(debate.id, refreshCallback)` where the refresh callback re-awaits `getMyPowerUps` and re-renders. Errors are caught with a `console.warn`.

`renderInputControls(debate.mode)` is called synchronously, followed by `addReferenceButton()`. If not `isUnplugged` and `equippedForDebate.length > 0`, `renderActivationBar(equippedForDebate)` is called; if it returns a non-empty string, a container is created, the HTML is set on it, and its first child element is inserted after `.arena-messages` using `insertBefore`. `wireActivationBar(debate.id, handlers)` is then called with three named handlers: `onSilence` calls `addSystemMessage`, `set_silenceTimer(renderSilenceOverlay(...))`, and `activatedPowerUps.add('silence')`; `onShield` calls `set_shieldActive(true)`, `renderShieldIndicator()`, `addSystemMessage`, and `activatedPowerUps.add('shield')`; `onReveal` is async — it calls `addSystemMessage`, awaits `getOpponentPowerUps(debate.id)`, and either calls `renderRevealPopup(oppData.equipped)` on success or calls `addSystemMessage` with an error string, then adds `'reveal'` to `activatedPowerUps`. After wiring, if `hasMultiplier(equippedForDebate)` returns true, `addSystemMessage` is called with a multiplier notice.

Finally, three conditional polling calls are made. If `selectedModerator` or `debate.modView` is truthy and `debate.id` is non-empty and not a local/placeholder ID, `startReferencePoll(debate.id)` is called. If `debate.modView` is falsy, `debate.mode` is not `'ai'`, `debate.id` is not a local/placeholder ID, and `isPlaceholder()` returns false, `startModStatusPoll(debate.id)` is called. If `debate.mode === 'live'`, `startLiveRoundTimer()` is called synchronously and `initLiveAudio()` is called as a fire-and-forget void.
