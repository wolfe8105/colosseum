# Stage 2 Outputs — arena-feed-wiring.ts

## Agent 01

### renderControls
Reads the `debate` parameter and `isModView` boolean. Calls `document.getElementById('feed-controls')` and returns immediately if the element is null. Branches on `isModView`: if true, sets `controlsEl.innerHTML` to a string of moderator-control HTML, interpolating `FEED_SCORE_BUDGET[1..5]` values and `escapeHTML(debate.debaterAName || 'A')` / `escapeHTML(debate.debaterBName || 'B')` into the template, then calls `wireModControls()`. If false, branches on `debate.spectatorView`: if true, builds a spectator tip-strip HTML using `escapeHTML(debate.debaterAName || 'Side A')` and `escapeHTML(debate.debaterBName || 'Side B')`, then fires `void wireSpectatorTipButtons(debate)` (fire-and-forget). If neither, builds debater control HTML interpolating `challengesRemaining` into a button label, then calls `wireDebaterControls(debate)`. Returns void.

### wireDebaterControls
Reads the `debate` parameter. Queries the DOM for four elements by ID: `feed-debater-input` (textarea), `feed-debater-send-btn`, `feed-finish-turn`, `feed-concede`. Attaches an `input` listener to the textarea that updates send button disabled state based on text length and textarea disabled state, and auto-resizes the textarea height (min auto, max 120px). Attaches a `click` listener to the send button that fires `void submitDebaterMessage()`. Attaches a `keydown` listener to the textarea that fires `void submitDebaterMessage()` on Enter (non-shifted). Attaches a `click` listener to the finish button that calls `finishCurrentTurn()`. Attaches a `click` listener to the concede button: on click, shows a `confirm()` dialog; if confirmed, reads `currentDebate`; if null or already conceded returns; otherwise sets `d.concededBy = d.role`, calls `clearFeedTimer()`, `stopTranscription()`, `clearInterimTranscript()`, conditionally calls `setDebaterInputEnabled(false)` if not mod view, and if `feedPaused` is true: calls `set_feedPaused(false)`, clears `challengeRulingTimer` via `clearInterval`, calls `set_challengeRulingTimer(null)` and `set_activeChallengeRefId(null)`, removes the `feed-challenge-overlay` element from the DOM. Then reads `getCurrentProfile()` for display name, calls `void writeFeedEvent('speech', '${name} has conceded.', 'mod')` (fire-and-forget), and calls `startFinalAdBreak(d)`. Queries `feed-cite-btn` and attaches a click listener that returns if `feedPaused` is true, otherwise calls `showCiteDropdown(debate)`. Queries `feed-challenge-btn` and attaches a click listener that returns if `feedPaused` is true, otherwise calls `showChallengeDropdown(debate)`. Returns void.

### wireSpectatorTipButtons
Async. Reads the `debate` parameter. Queries the DOM for `feed-tip-status` element and all `.feed-tip-btn` elements (cast to HTMLButtonElement[]). Initializes `tier = 'Unranked'`. Awaits `safeRpc('get_user_watch_tier', {})`: if no error and data is an array with a first element, reads `.tier` from it; if no error and data is an object with a `tier` key, reads `.tier`. On exception, logs a warning. If tier is `'Unranked'`, sets statusEl text to a message and returns (buttons remain disabled). Otherwise enables all tip buttons, sets statusEl text to `tier + ' · Tap to tip'`. Attaches a click listener to each tip button that fires `void handleTip(btn, debate, statusEl)`. Returns void (Promise<void>).

### handleTip
Async. Reads `btn.dataset.side` (cast to `'a' | 'b'`) and `Number(btn.dataset.amount)`. Returns immediately if either is falsy. Queries all `.feed-tip-btn` elements and disables them all. Awaits `safeRpc('cast_sentiment_tip', { p_debate_id, p_side, p_amount })`. On error or null data, logs warning, shows 'Tip failed' error toast, re-enables all buttons, returns. On success, reads `result.error`: if `'insufficient_tokens'` shows toast and re-enables; if `'unranked_blocked'` shows toast and re-enables; if `'debate_not_live'` shows toast and re-enables; otherwise shows generic error toast and re-enables. Returns after re-enabling buttons. If no `result.error`: if side is `'a'` calls `set_pendingSentimentA(pendingSentimentA + amount)`, else calls `set_pendingSentimentB(pendingSentimentB + amount)`. Calls `applySentimentUpdate()`. Sets statusEl text to `+${amount} → ${side.toUpperCase()} ✓`. Shows `+${amount} tokens tipped` success toast. After 800ms timeout, re-enables all buttons and resets statusEl text to 'Tap to tip'. On exception, logs warning, shows error toast, re-enables all buttons.

### wireModControls
Reads no parameters. Queries the DOM for `feed-mod-input` textarea and `feed-mod-send-btn` button. Attaches input listener to textarea that updates send button disabled state and auto-resizes (max 80px). Attaches click listener to send button that fires `void submitModComment()`. Attaches keydown listener to textarea that fires `void submitModComment()` on Enter (non-shifted). Queries `feed-stream` and attaches a delegated click listener: on click, checks if target has class `feed-pin-btn` — if so, stops propagation and fires `void handlePinClick(target)`, then returns. Checks if target has a `.feed-cite-claim` ancestor — if so, stops propagation, calls `showReferencePopup(citeEl)`, returns. Otherwise, finds closest `.feed-evt-a` or `.feed-evt-b` ancestor; if none, returns. Removes `feed-evt-selected` class from any previously selected element, adds it to the found target, shows the score row by setting its display to `'flex'`, sets the score prompt text based on side (A or B), and calls `updateBudgetDisplay()`. Attaches click listeners to each `.feed-score-btn`: reads pts, checks budget limit from `FEED_SCORE_BUDGET[pts]` and `scoreUsed[pts]`; if over budget, shows error toast and returns. Gets selected element and its `data-event-id`; if none selected returns; if no eventId shows error toast and returns. Reads `currentDebate`; if null returns. Disables the clicked button. Fires `void safeRpc('score_debate_comment', {...}).then(...)`: on error, logs warning, shows error toast, re-enables button. Removes `feed-evt-selected` class from selected element and hides score row. Attaches click listener to `feed-score-cancel`: removes selected class and hides score row. Attaches click listeners to `feed-mod-eject-a`, `feed-mod-eject-b`, and `feed-mod-null` buttons: each shows a confirm dialog; if confirmed, fires `void modNullDebate('eject_a' | 'eject_b' | 'null')`. Returns void.

### submitDebaterMessage
Async. Reads `currentDebate`; returns if null. Queries `feed-debater-input` textarea; returns if null, value is empty/whitespace, or disabled. Captures trimmed text, clears textarea value and height. Disables send button if found. Reads `getCurrentProfile()` for author name. Calls `appendFeedEvent({...})` with a locally-generated UUID, the debate id, event_type 'speech', current `round`, `debate.role` as side, the text, current timestamp, and authorName. Awaits `writeFeedEvent('speech', text, debate.role)`.

### submitModComment
Async. Reads `currentDebate`; returns if null. Queries `feed-mod-input` textarea; returns if null or empty. Captures trimmed text, clears textarea value and height. Disables send button if found. Calls `appendFeedEvent({...})` with UUID, debate id, 'speech', current `round`, 'mod' as side, text, timestamp, and `debate.moderatorName || 'Moderator'`. Awaits `writeFeedEvent('speech', text, 'mod')`.

### handlePinClick
Async. Reads `currentDebate`; returns if null. Reads `btn.dataset.eid`. If eid is absent or contains a hyphen (indicating a temporary UUID, not yet confirmed), shows 'Cannot pin — waiting for confirmation' error toast and returns. Sets `btn.style.pointerEvents = 'none'` to block double-tap. In a try block, awaits `safeRpc('pin_feed_event', { p_debate_id: debate.id, p_feed_event_id: Number(eid) })`. On error, logs warning, shows 'Pin failed' toast, returns. On success, checks `pinnedEventIds.has(eid)`: if was pinned, calls `pinnedEventIds.delete(eid)`, removes `pinned` class from btn, removes `feed-evt-pinned` class from closest `.feed-evt` ancestor. If not pinned, calls `pinnedEventIds.add(eid)`, adds `pinned` class to btn, adds `feed-evt-pinned` class to ancestor. In finally block, resets `btn.style.pointerEvents = ''`.

## Agent 02

### renderControls
Reads `debate` (CurrentDebate) and `isModView` (boolean). Retrieves the `feed-controls` DOM element by ID; returns without action if absent. Branches into three paths based on `isModView` and `debate.spectatorView`. When `isModView` is true: constructs a moderator-control HTML string using `escapeHTML` on debater names (defaulting to 'A'/'B') and inlining `FEED_SCORE_BUDGET` values for scores 1-5; assigns to `controlsEl.innerHTML`; calls `wireModControls()` synchronously. When `isModView` is false but `debate.spectatorView` is truthy: constructs a spectator tip-strip HTML using `escapeHTML` on debater names (defaulting to 'Side A'/'Side B'); assigns to `controlsEl.innerHTML`; calls `wireSpectatorTipButtons(debate)` without awaiting (fire-and-forget via `void`). Otherwise (debater view): constructs debater-control HTML interpolating `challengesRemaining` into the challenge button label; assigns to `controlsEl.innerHTML`; calls `wireDebaterControls(debate)` synchronously.

### wireDebaterControls
Reads `debate` (CurrentDebate). Retrieves four DOM elements: `feed-debater-input` textarea, `feed-debater-send-btn` button, `feed-finish-turn` button, `feed-concede` button (each may be null). Registers an `input` event listener on the textarea that reads the textarea's current value length, updates the send button's disabled state based on zero-length or the textarea's own disabled state, and resizes the textarea height (clamped to 120px). Registers a `click` listener on the send button that fires `submitDebaterMessage()` as fire-and-forget. Registers a `keydown` listener on the textarea that fires `submitDebaterMessage()` when Enter is pressed without Shift. Registers a `click` listener on the finish button that calls `finishCurrentTurn()`. Registers a `click` listener on the concede button: prompts user with `confirm()`; on cancel, returns; reads `currentDebate` module variable; if null or already has `concededBy` set, returns; sets `currentDebate.concededBy = currentDebate.role`, calls `clearFeedTimer()`, `stopTranscription()`, `clearInterimTranscript()`, conditionally calls `setDebaterInputEnabled(false)` if `currentDebate.modView` is falsy, and if `feedPaused` module variable is true: calls `set_feedPaused(false)`, `clearInterval(challengeRulingTimer)`, `set_challengeRulingTimer(null)`, `set_activeChallengeRefId(null)`, removes `feed-challenge-overlay` element from DOM; reads `getCurrentProfile()?.display_name`; fires `writeFeedEvent('speech', '${name} has conceded.', 'mod')` without awaiting; calls `startFinalAdBreak(d)`. Attaches `click` listener to `feed-cite-btn`: if `feedPaused`, no-op; else calls `showCiteDropdown(debate)`. Attaches `click` listener to `feed-challenge-btn`: if `feedPaused`, no-op; else calls `showChallengeDropdown(debate)`.

### wireSpectatorTipButtons
Async function. Reads `debate` (CurrentDebate). Queries DOM for `feed-tip-status` element and collects all `.feed-tip-btn` button elements. Sets `tier = 'Unranked'`. Awaits `safeRpc('get_user_watch_tier', {})` inside a try-catch: on success, attempts to extract `tier` from the returned data (checks array-first or object-with-tier-key). On exception, logs a warning. If `tier === 'Unranked'`, updates statusEl text with a prompt to watch a full debate and returns (buttons remain disabled). Otherwise enables all tip buttons, updates statusEl to show tier and 'Tap to tip'. Registers `click` listener on each tip button to fire `handleTip(btn, debate, statusEl)` without awaiting.

### handleTip
Async function. Reads `btn.dataset.side` and `btn.dataset.amount` (Number-cast). If side or amount is falsy, returns early. Disables all `.feed-tip-btn` elements. Awaits `safeRpc('cast_sentiment_tip', { p_debate_id, p_side, p_amount })`. On RPC failure or null data: logs warning, shows error toast, re-enables all tip buttons, returns. On success, checks `result.error` string: maps `'insufficient_tokens'`, `'unranked_blocked'`, `'debate_not_live'` to specific toasts; any other error value gets a generic toast. In all error cases, re-enables buttons and returns. On success (no `result.error`): if side is 'a', calls `set_pendingSentimentA(pendingSentimentA + amount)`; otherwise calls `set_pendingSentimentB(pendingSentimentB + amount)`. Calls `applySentimentUpdate()`. Updates statusEl to show confirmation text. Shows success toast. Schedules a 800ms `setTimeout` to re-enable all buttons and reset statusEl. On exception: logs warning, shows error toast, re-enables all buttons.

### wireModControls
Reads no parameters. Queries DOM for `feed-mod-input` textarea and `feed-mod-send-btn` button. Attaches `input` listener to textarea that updates send button disabled state (non-zero length) and auto-resizes (max 80px). Attaches `click` listener to send button and `keydown` listener to textarea (Enter without Shift) — both fire `submitModComment()` without awaiting. Queries `feed-stream`; registers a delegated `click` listener that checks three cases in order: (1) if click target has class `feed-pin-btn`, calls `handlePinClick(target)` without awaiting and stops propagation; (2) if click target is inside a `.feed-cite-claim` element, calls `showReferencePopup(citeEl)` and stops propagation; (3) otherwise, finds the closest `.feed-evt-a` or `.feed-evt-b` ancestor, removes `feed-evt-selected` from all currently selected items, marks the found target as selected, makes the score row visible, updates the score-prompt text for side A or B, and calls `updateBudgetDisplay()`. Registers `click` listener on each `.feed-score-btn`: reads pts value, checks against `FEED_SCORE_BUDGET[pts]` limit and `scoreUsed[pts]` count; if over limit shows error toast and returns; reads `data-event-id` from selected element; if no selection or no event ID shows error; reads `currentDebate`; if null returns; disables the button; fires `safeRpc('score_debate_comment', {...}).then(...)` as fire-and-forget; on RPC error re-enables button. Clears selection and hides score row. Attaches `click` to `feed-score-cancel` to clear selection and hide score row. Attaches `click` to `feed-mod-eject-a`, `feed-mod-eject-b`, `feed-mod-null` — each confirms via dialog and fires `modNullDebate(reason)` without awaiting.

### submitDebaterMessage

Clears the textarea value and resets height to 'auto'. Disables the send button if found. Reads `getCurrentProfile()` for display name, falling back through `.username` to 'You'. Calls `appendFeedEvent({...})` with a new `crypto.randomUUID()` as id, the current debate id, type 'speech', module-level `round`, `debate.role` as side, the text, current ISO timestamp, and the author name (optimistic local render). Awaits `writeFeedEvent('speech', text, debate.role)`.

### submitModComment
Async. Reads `currentDebate` module variable; returns if null. Reads `feed-mod-input` textarea; returns if null or value is empty. Trims the text. Clears textarea and resets height. Disables send button if found. Calls `appendFeedEvent({...})` with new UUID, debate id, 'speech', `round`, 'mod' as side, text, current timestamp, and `debate.moderatorName || 'Moderator'`. ### handlePinClick
Async. Reads `currentDebate`; returns if null. Reads `btn.dataset.eid`. If eid is absent or contains a '-' character (UUID not yet server-confirmed), shows an error toast and returns. Sets `btn.style.pointerEvents = 'none'`. In finally block: resets `btn.style.pointerEvents = ''`.
