## Agent 01

### renderLobby

`renderLobby` is synchronous. It begins by resetting a collection of module-level state variables via their setter functions: `set_view` is called with `'lobby'`, `set_selectedMode` with `null`, `set_selectedModerator` with `null`, `set_selectedRanked` with `false`, `set_selectedRuleset` with `'amplified'`, `set_selectedCategory` with `null`, and `set_selectedWantMod` with `false`. It then checks the module-level `privateLobbyPollTimer`; if it is truthy, it calls `clearInterval` on it and calls `set_privateLobbyPollTimer(null)`. `set_privateLobbyDebateId(null)` follows unconditionally. It calls `stopReferencePoll()`, clears the module-level `activatedPowerUps` set via `.clear()`, and calls `set_shieldActive(false)` and `set_equippedForDebate([])`. It checks `silenceTimer` and, if truthy, calls `clearInterval` on it and calls `set_silenceTimer(null)`. It then calls `removeShieldIndicator()` and removes two DOM elements by id (`powerup-silence-overlay`, `powerup-reveal-popup`) if they exist. It calls `history.replaceState` to write `{ arenaView: 'lobby' }` into the browser history. If the module-level `screenEl` is non-null, it sets `screenEl.innerHTML` to the empty string.

After clearing state, it calls `getCurrentProfile()` and reads `profile?.login_streak`, casting it through `Number()` with a `|| 0` fallback. It creates a `div.arena-lobby.arena-fade-in` element, assigns a large HTML template string to its `innerHTML` — embedding the `loginStreak` value directly, and conditionally including a mod-queue button if `profile?.is_moderator` is truthy, and a moderator-recruitment banner if `getCurrentUser()` returns a value and `profile?.is_moderator` is falsy — then appends the element to `screenEl` if `screenEl` is non-null. It attaches click listeners to four buttons by id: `arena-enter-btn` → `showRankedPicker`, `arena-powerup-shop-btn` → `showPowerUpShop`, `arena-private-btn` → `showPrivateLobbyPicker`, `arena-mod-queue-btn` → `showModQueue`. It attaches an async click listener to `arena-become-mod-btn` that awaits `toggleModerator(true)`: if the result has no `.error`, it calls `showToast` with a success message and calls `renderLobby()` recursively; otherwise it calls `showToast` with an error message. It wires the join-code GO button and the input's `keydown` event: both read from the `arena-join-code-input` field, uppercase and trim the value, and if the code is exactly 6 characters call `joinWithCode(code)` as a fire-and-forget void; the button also shows a toast if the length is not 6. It attaches a click listener to `arena-challenge-cta` that calls `navigateTo('home')`. Finally it calls `loadLobbyFeed()` as fire-and-forget (`void`), conditionally calls `loadPendingChallenges()` as fire-and-forget if `isPlaceholder()` returns false, and attaches a delegated click listener to the `lobby` element that checks each click target: if `target.closest` finds a `.arena-card.card-live[data-debate-id]` element, it calls `enterFeedRoomAsSpectator` with the element's `data-debate-id` and returns; otherwise if it finds a `.arena-card[data-link]` element, it sets `window.location.href` to the element's `data-link`.

### loadLobbyFeed

`loadLobbyFeed` is async and returns `Promise<void>`. It reads three DOM elements by id: `arena-live-feed`, `arena-verdicts-feed`, and `arena-unplugged-feed`. If either `liveFeed` or `verdictsFeed` is null, it returns early without doing anything. It then calls `isPlaceholder()`: if that returns true, it sets `liveFeed.innerHTML` to the result of `renderPlaceholderCards('live')` and `verdictsFeed.innerHTML` to the result of `renderPlaceholderCards('verdict')`, then returns.

If not a placeholder, the function enters a try block. It calls `getSupabaseClient()` and awaits `safeRpc<ArenaFeedItem[]>('get_arena_feed', { p_limit: 20 })`. If the result has an error, `data` is nullish, or `data` is an empty array, it falls into an error branch: it awaits a Supabase query on `auto_debates`, selecting specific columns, filtered by `status === 'active'`, ordered by `created_at` descending, limited to 10. If `autoData` is non-empty, it sets `liveFeed.innerHTML` to a static "No live debates" message and sets `verdictsFeed.innerHTML` to the array of `autoData` items mapped through `renderAutoDebateCard`, joined as a string; otherwise it sets both feeds to placeholder card HTML via `renderPlaceholderCards`. Either way it returns after the fallback branch. If the RPC succeeds with data, it splits `feedData` into two arrays by `ruleset`: `unplugged` (where `ruleset === 'unplugged'`) and `amplified` (all others). It further splits `amplified` into `live` (status `'live'` or `'pending'`) and `complete` (status `'complete'`, `'voting'`, or `source === 'auto_debate'`). It sets `liveFeed.innerHTML` to either the `live` items mapped through `renderArenaFeedCard(d, 'live')` joined, or a static empty-state string if `live` is empty. It sets `verdictsFeed.innerHTML` to either the `complete` items mapped through `renderArenaFeedCard(d, 'verdict')` joined, or a static empty-state string. If `unpluggedFeed` is non-null, it sets its `innerHTML` to either the `unplugged` items mapped through `renderArenaFeedCard` (using `'live'` or `'verdict'` based on each item's `status`) joined, or a static empty-state string. The catch block logs the error to `console.error` and sets both `liveFeed.innerHTML` and `verdictsFeed.innerHTML` to placeholder card HTML via `renderPlaceholderCards`.

### showPowerUpShop

`showPowerUpShop` is async and returns `Promise<void>`. It calls `getCurrentUser()` and `isPlaceholder()`: if `getCurrentUser()` is falsy and `isPlaceholder()` is also false, it sets `window.location.href` to `'moderator-plinko.html'` and returns immediately. Otherwise it proceeds: it calls `set_view('powerUpShop')` and `pushArenaState('powerUpShop')`. It reads `getCurrentProfile()?.token_balance`, casts it with `Number()` with a `|| 0` fallback, and awaits `renderShop(tokenBalance)` from `../powerups.ts`, storing the resulting HTML string. If the module-level `screenEl` is non-null, it replaces its `innerHTML` with a wrapper div containing a back button and the shop HTML. It then attaches a click listener to `powerup-shop-back` that calls `renderLobby()`, which re-renders the lobby and discards the shop view. It queries all `.powerup-buy-btn` elements and attaches an async click listener to each: the listener reads `data-id` and `data-cost` from the button element, disables the button and sets its text to `'...'`, then awaits `buyPowerUp(id, 1, cost)` from `../powerups.ts`. If `result.success` is true, it calls `showToast` with a success message and calls `showPowerUpShop()` recursively as a fire-and-forget re-render to reflect the updated balance. If `result.success` is false, it calls `showToast` with `result.error` or `'Purchase failed'`, re-enables the button, and restores its text to the cost and token icon.

It first calls `getCurrentUser()` and `isPlaceholder()`; if there is no current user and the app is not in placeholder mode, it sets `window.location.href` to `'moderator-plinko.html'` and returns immediately. Otherwise it calls `set_view('powerUpShop')` and `pushArenaState('powerUpShop')` from `arena-core.ts`. It reads `getCurrentProfile()` and extracts `token_balance` as a number, defaulting to `0`. It then awaits `renderShop(tokenBalance)` from `powerups.ts`, which returns an HTML string. If the module-level `screenEl` is non-null, it sets `screenEl.innerHTML` to a wrapper div containing a back button and the shop HTML. A click listener is attached to `powerup-shop-back` which calls `renderLobby()`, replacing the shop view with the lobby. The function then queries all elements matching `.powerup-buy-btn` and attaches an async click listener to each. Each listener reads `data-id` and `data-cost` from the button element, disables the button and sets its text to `'...'`, then awaits `buyPowerUp(id, 1, cost)` from `powerups.ts`. If `result.success` is true it calls `showToast` with a success message and then calls `showPowerUpShop()` recursively to re-render the shop with the updated balance; the recursive call is not awaited (it is a fire-and-forget void call at this point). If the purchase fails, it calls `showToast` with the error message or a fallback string and re-enables the button, restoring its text to the cost and token icon.

### renderLobby

`renderLobby` is synchronous. It begins by writing a series of arena-state variables via setter functions imported from `arena-state.ts`: `set_view` to `'lobby'`, `set_selectedMode` and `set_selectedModerator` to `null`, `set_selectedRanked` to `false`, `set_selectedRuleset` to `'amplified'`, `set_selectedCategory` and `set_selectedWantMod` to their null/false defaults. It then inspects the module-level `privateLobbyPollTimer` variable; if that interval is set, it calls `clearInterval` on it and calls `set_privateLobbyPollTimer(null)`. It calls `set_privateLobbyDebateId(null)`, then calls `stopReferencePoll()` from `arena-mod-refs.ts`. It calls `activatedPowerUps.clear()` on the module-level `activatedPowerUps` Set, calls `set_shieldActive(false)` and `set_equippedForDebate([])`. It inspects the module-level `silenceTimer`; if set, clears it with `clearInterval` and calls `set_silenceTimer(null)`. It then calls `removeShieldIndicator()` from `powerups.ts` and removes two DOM elements by id (`powerup-silence-overlay`, `powerup-reveal-popup`) using optional chaining. It calls `history.replaceState` to set `{ arenaView: 'lobby' }` on the current history entry. If the module-level `screenEl` is non-null, it sets its `innerHTML` to an empty string.

After clearing state, the function reads from `getCurrentProfile()` to retrieve the current user profile and extracts `login_streak` as a number (defaulting to `0`). It creates a `<div>` with class `arena-lobby arena-fade-in` and sets its `innerHTML` to a large template string containing the hero section, stat row, main buttons, an optional moderator queue button (rendered only when `profile?.is_moderator` is truthy), an optional moderator recruitment banner (rendered only when `getCurrentUser()` returns a user and `profile?.is_moderator` is falsy), a join-code input row, and four section containers for pending challenges, live debates, open challenges, unplugged debates, and recent verdicts. The constructed `lobby` div is appended to `screenEl`. The function then wires click listeners: `arena-enter-btn` calls `showRankedPicker`, `arena-powerup-shop-btn` calls `showPowerUpShop`, `arena-private-btn` calls `showPrivateLobbyPicker`, and `arena-mod-queue-btn` calls `showModQueue`. An async click listener on `arena-become-mod-btn` awaits `toggleModerator(true)` and on success calls `showToast` and recursively calls `renderLobby()`; on error it calls `showToast` with an error message. The join-code button listener reads the input value, uppercases it, and if it is exactly 6 characters fires `joinWithCode(code)` as a fire-and-forget void call; otherwise calls `showToast`. A `keydown` listener on the input does the same on `Enter`. A click listener on `arena-challenge-cta` calls `navigateTo('home')`. Finally, `loadLobbyFeed()` is called as a fire-and-forget void, `loadPendingChallenges()` is conditionally called as fire-and-forget only when `isPlaceholder()` returns false, and a delegated click listener on the `lobby` element intercepts clicks on `.arena-card.card-live[data-debate-id]` elements to call `enterFeedRoomAsSpectator` with the debate id, and clicks on `.arena-card[data-link]` elements to set `window.location.href` to the card's `data-link` attribute.

### loadLobbyFeed

`loadLobbyFeed` is async. It reads three DOM elements by id: `arena-live-feed`, `arena-verdicts-feed`, and `arena-unplugged-feed`. If either `liveFeed` or `verdictsFeed` is absent it returns immediately with no further action. It then calls `isPlaceholder()` from `arena-core.ts`; if that returns true it sets `liveFeed.innerHTML` to the output of `renderPlaceholderCards('live')` and `verdictsFeed.innerHTML` to `renderPlaceholderCards('verdict')` and returns.

In the non-placeholder path the function enters a try block. It calls `getSupabaseClient()` to obtain the Supabase client and awaits `safeRpc<ArenaFeedItem[]>('get_arena_feed', { p_limit: 20 })`. If the RPC returns an error, no data, or an empty array, the function falls back: it awaits a PostgREST query on the `auto_debates` table selecting specific columns where `status = 'active'`, ordered by `created_at` descending, limited to 10 rows. If `autoData` is non-empty, `liveFeed.innerHTML` is set to a static empty-state string and `verdictsFeed.innerHTML` is set to the result of mapping each row through `renderAutoDebateCard` and joining the strings. If `autoData` is also empty or absent, both feeds are set to `renderPlaceholderCards` output. The function then returns. On the happy path with valid `data`, the feed items are split: items with `ruleset === 'unplugged'` go into `unplugged`; all others go into `amplified`. From `amplified`, items with `status` of `'live'` or `'pending'` go into `live`; items with `status` of `'complete'` or `'voting'` or `source === 'auto_debate'` go into `complete`. `liveFeed.innerHTML` is set to the joined output of `renderArenaFeedCard(d, 'live')` for each live item, or an empty-state string if none. `verdictsFeed.innerHTML` is set similarly from complete items. If `unpluggedFeed` is present, its `innerHTML` is set from unplugged items, using `'live'` or `'verdict'` as the second argument to `renderArenaFeedCard` depending on the item's status. The catch block logs the error to `console.error` and sets both `liveFeed.innerHTML` and `verdictsFeed.innerHTML` to `renderPlaceholderCards` output; `unpluggedFeed` is not written in the catch path.
