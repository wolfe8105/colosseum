# Stage 2 Outputs — spectate.render.ts

## Agent 01

### showError

`showError` accepts a single string parameter `msg`. It reads two properties from the module-level `state` object: `state.loading` and `state.app`, both of which are DOM element references (or null/undefined). If `state.loading` is truthy, the function sets its `style.display` to `'none'`, hiding it. If `state.app` is truthy, the function sets its `innerHTML` to a string that wraps the result of calling `escHtml(msg)` inside a `<div class="error-state">` element, followed by a hardcoded anchor tag pointing to `/` with the text "Back to Home". The function returns nothing. There is no try/catch, no branching beyond the two null guards, and the function is synchronous.

### renderMessages

`renderMessages` accepts a `DebateMessage[]` array and a `SpectateDebate` object `d`. It is synchronous and returns a string of HTML. It reads no module-level state except `state.lastMessageTime`, which it may write. It iterates over the `messages` array with a `for...of` loop, maintaining a local `lastRound` variable initialized to `0`. For each message, if `m.round` is truthy and differs from `lastRound`, it appends a round-divider `<div>` containing the round number cast through `Number()`, then updates `lastRound`. It then determines the message side (defaulting to `'a'` if `m.side` is falsy) and the display name: if `m.is_ai` is truthy, the name is a robot emoji followed by "AI"; otherwise it uses `d.debater_a_name` or `d.debater_b_name` depending on side, falling back to `'Side A'` or `'Side B'`. It appends a `.msg` div containing the name, content, and round number, passing the side through `escHtml()` and the name and content through `escHtml()` as well; the round number is cast with `Number()`. After appending the message HTML, it checks whether `m.created_at` is truthy and whether it is greater than `state.lastMessageTime` (or `state.lastMessageTime` is falsy); if both conditions are met, it writes `m.created_at` to `state.lastMessageTime`. The accumulated HTML string is returned after all messages are processed.

### formatPointBadge

`formatPointBadge` is a private synchronous function that accepts a single `ReplayPointAward` parameter `pa` and returns a formatted string. It reads `pa.metadata` and, from it, the fields `base_score`, `in_debate_multiplier`, `in_debate_flat`, and `final_contribution`, falling back to `pa.base_score` for the base and `0`/`1.0` defaults for the others, each cast through `Number()`. It computes two boolean flags: `hasMult` is true when `mult !== 1.0`, and `hasFlat` is true when `flat !== 0`. It then branches across four mutually exclusive return paths: if neither flag is set, it returns `'+<base> pts'`; if only `hasMult` is set, it returns a string with the multiplication shown as `'+<base> × <mult> = <final> pts'`; if only `hasFlat` is set, it returns `'+<final> pts'` with no multiplication shown; if both flags are set, it returns `'+<base> × <mult> + <flat> = <final> pts'`. The function calls no other functions and writes no state.

### renderTimeline

`renderTimeline` accepts a `DebateMessage[]` array and a `SpectateDebate` object `d`, and returns a string of HTML. It is synchronous. It begins by reading `state.replayData` into a local variable `rd`.

It builds a `Map<string, ReplayPointAward>` called `pointAwardMap` and an `orphanAwards` array. If `rd?.point_awards` is non-empty, it iterates over each award: awards whose `metadata.scored_event_id` field is truthy are inserted into the map keyed by that ID (converted to string); awards without it are pushed into `orphanAwards`. It then computes `hasSpeechEvents` as true if `rd?.speech_events` has at least one entry, and `hasEnrichment` as true if `rd` exists and any of `power_ups`, `references`, `speech_events`, or `point_awards` are non-empty. If `hasEnrichment` is false and `messages.length` is greater than zero, the function returns early by calling `renderMessages(messages, d)` and returning its result directly without further processing.

Otherwise, it builds a `TimelineEntry[]` array called `entries`. If `hasSpeechEvents` is true, it iterates over `rd!.speech_events` and pushes a `'speech'`-typed entry for each, then pushes a `'score'`-typed entry for each entry in `orphanAwards`. If `hasSpeechEvents` is false, it iterates over `messages` and pushes a `'message'`-typed entry for each, using `'1970-01-01T00:00:00Z'` as the timestamp when `m.created_at` is absent, then pushes `'score'`-typed entries for all items in both `orphanAwards` and the values of `pointAwardMap`. In both branches, if `rd?.power_ups` is non-empty, a `'power_up'` entry is pushed for each item; if `rd?.references` is non-empty, a `'reference'` entry is pushed for each. The entries array is then sorted in ascending chronological order by parsing each timestamp with `new Date().getTime()`.

The function then iterates over the sorted entries with a `for...of` loop, maintaining a local `lastRound` variable. For `'message'` entries, it renders a round-divider if the round changed, appends a `.msg` div with name, content, and round number (using `escHtml()` on user-supplied values and `Number()` on numerics), and updates `state.lastMessageTime` if the message's `created_at` is more recent. For `'speech'` entries, it does the same structural rendering using `se.debater_name` and `se.content`, then looks up the speech event's `id` in `pointAwardMap`; if a matching award is found, it appends a `.msg-score-badge` div whose text comes from calling `formatPointBadge(award)`, with a CSS class of `award-side-a` or `award-side-b` depending on `award.side`. It then updates `state.lastMessageTime` the same way as for message entries. For `'score'` entries, it appends a `.timeline-event.score-event` div containing the result of `formatPointBadge(pa)` and the debater name resolved from `pa.side`. For `'power_up'` entries, it appends a `.timeline-event.power-up-event` div with the power-up icon and name, both passed through `escHtml()`. For `'reference'` entries, it appends a `.timeline-event.reference-event` div containing the submitter name, and conditionally the description and URL fields if truthy, then a ruling line whose icon is `✅`, `❌`, or `⏳` based on `ref.ruling`, with a ruling reason appended if `ref.ruling_reason` is truthy. After all entries are processed, the accumulated HTML string is returned.

### renderSpectateView

`renderSpectateView` accepts a `SpectateDebate` object `d` and a `DebateMessage[]` array `messages`. It is synchronous, returns nothing, and produces the complete spectator page by assembling a single large HTML string and writing it to the DOM.

It begins by hiding `state.loading` if it is present. It computes `isLive` as true when `d.status` is one of `'live'`, `'pending'`, `'round_break'`, or `'voting'`. It then builds the HTML string section by section. It calls `statusBadge(d.status)` and `modeLabel(d.mode)` from `spectate.utils` and inlines their return values. It calls `escHtml()` on all user-supplied string fields throughout. It calls `renderAvatar()` twice for the VS bar, once per debater. For the moderator bar, it checks `d.moderator_type`; if it is neither absent nor `'none'`, it appends the bar, using either the literal string `'AI Moderator'` or `escHtml(d.moderator_name)` depending on whether the type is `'ai'`. For the audience pulse gauge, it reads `d.vote_count_a` and `d.vote_count_b` cast through `Number()`; if their sum is greater than zero, it renders percentage-width fill divs with computed values, otherwise it renders 50/50 placeholder divs and an empty-state message. If `isLive` is true, it appends a live-indicator div. For the message stream, it reads `state.replayData?.speech_events?.length`; if `messages.length` is zero and that count is also zero, it renders an empty-state placeholder; otherwise it calls `renderTimeline(messages, d)` and inlines the result. For the spectator chat section, it reads `state.chatMessages`, `state.chatOpen`, and `state.isLoggedIn` from module-level state. If `state.chatMessages` is non-empty it calls `renderChatMessages(state.chatMessages)` from `spectate.chat`. If `state.isLoggedIn` is true it renders a text input and send button; otherwise it renders a login prompt.

For the final scoreboard section, it checks `d.status` for `'complete'` or `'completed'` and `d.score_a != null`; if both are true, it renders score values cast through `Number()` and applies a `'winner'` class based on `d.winner`. For the moderator rating section, it additionally checks `d.moderator_type === 'human'`, `d.moderator_name`, and `state.replayData?.mod_scores.length`; if the combined condition is met, it filters `state.replayData.mod_scores` into `debaterScores` and `spectatorScores` by `scorer_role`. For debater scores it renders a verdict of "FAIR" or "UNFAIR" based on `s.score >= 25`. For spectator scores it computes an average, renders a bar whose fill width is `avgScore * 2` percent, and displays the average as `N/50`. For the AI scorecard, it checks `d.ai_scorecard`; if present, it reads `sc.side_a` and `sc.side_b` for the four criteria (`logic`, `evidence`, `delivery`, `rebuttal`), sums each side's scores, renders score-bar rows with widths at `score * 10` percent, and if `sc.verdict` is truthy appends a verdict div. The vote section, share bar, CTA banner, and footer are appended unconditionally. After the full string is assembled, it writes it to `state.app.innerHTML` and sets `state.lastRenderedMessageCount` to `messages.length`. It then calls `wireVoteButtons(d)`, `wireShareButtons(d)`, and `wireChatUI(d)` in that order to attach event listeners to the newly written DOM nodes. Finally, if the sum of `d.vote_count_a` and `d.vote_count_b` is greater than zero, it calls `updateVoteBar(d.vote_count_a, d.vote_count_b)` to synchronize the vote bar display. None of these post-write calls are awaited, and the function has no try/catch block.

The accumulated `html` string is returned at the end.

### renderSpectateView

`renderSpectateView` takes a `SpectateDebate` object `d` and a `DebateMessage[]` array. It is synchronous and returns nothing. It reads `state.loading`, `state.app`, `state.replayData`, `state.chatMessages`, `state.chatOpen`, and `state.isLoggedIn` from module-level state.

It first hides `state.loading` if present, then computes `isLive` as true when `d.status` is one of `'live'`, `'pending'`, `'round_break'`, or `'voting'`. It then assembles a large HTML string in sections:

A status/mode badge row calls `statusBadge(d.status)` and `modeLabel(d.mode)`. A topic card emits `d.topic` through `escHtml`. A VS bar calls `renderAvatar` twice (once per debater) and emits both debater names and ELO values through `escHtml` and `Number()`. If `d.moderator_type` is present and not `'none'`, a moderator bar is appended; the label is `'AI Moderator'` for `'ai'` type or `escHtml(d.moderator_name || 'Human Moderator')` otherwise. An info bar emits `d.spectator_count` and, if both `d.current_round` and `d.total_rounds` are present, the round fraction. An audience pulse gauge section computes `va` and `vb` from `d.vote_count_a` and `d.vote_count_b`; if the total is greater than zero, it renders fill bars using percentage widths; otherwise it renders 50/50 placeholder bars and an empty-state label. If `isLive` is true, a live-indicator div is appended.

For the message stream, it checks `state.replayData?.speech_events?.length` to set `hasSpeechReplay`. If `messages.length === 0` and `hasSpeechReplay` is false, an empty-state `<div>` is rendered. Otherwise `renderTimeline(messages, d)` is called and its return value is injected.

The spectator chat section reads `state.chatMessages`, `state.chatOpen`, and `state.isLoggedIn`. The chat header shows a count of messages if `state.chatMessages.length > 0`. The chat body open/closed state comes from `state.chatOpen`. If `state.chatMessages.length === 0` an empty-state line is shown; otherwise `renderChatMessages(state.chatMessages)` is called and its return value is injected. The chat input row is rendered only when `state.isLoggedIn` is true; when false a login link is rendered instead.

If `d.status` is `'complete'` or `'completed'` and `d.score_a` is not null, a final scoreboard is appended, applying a `'winner'` CSS class to the side that matches `d.winner`. If the same completion condition holds and additionally `d.moderator_type === 'human'`, `d.moderator_name` is present, and `state.replayData.mod_scores` is non-empty, a moderator rating section is appended: debater scores are filtered by `scorer_role === 'debater'` and rendered with a fair/unfair verdict based on `s.score >= 25`; spectator scores are filtered by `scorer_role === 'spectator'`, their average is computed with `Math.round`, and a progress bar is rendered at `avgScore * 2` percent width. If `d.status` is complete and `d.ai_scorecard` is present, an AI scorecard section is appended: totals for each side are computed by summing the four criteria scores inline, a header row shows both totals, a fixed four-element criteria array is iterated rendering score bars, and if `sc.verdict` is present it is appended. A vote section, a share bar with four buttons, a CTA banner, and a footer are always appended unconditionally.

After the full HTML string is assembled, the function sets `state.app.innerHTML` to it (if `state.app` is non-null) and sets `state.lastRenderedMessageCount` to `messages.length`. It then calls `wireVoteButtons(d)`, `wireShareButtons(d)`, and `wireChatUI(d)` in that order. Finally, if the sum of `d.vote_count_a` and `d.vote_count_b` is greater than zero, it calls `updateVoteBar(d.vote_count_a || 0, d.vote_count_b || 0)`.

### formatPointBadge

`formatPointBadge` accepts a single `ReplayPointAward` parameter `pa` and returns a plain string label. It is synchronous, reads no external or module-level state, and writes nothing outside its own local scope.

It extracts four numeric values from the award: `base` from `pa.metadata?.base_score` falling back to `pa.base_score`, defaulting to `0`; `mult` from `pa.metadata?.in_debate_multiplier`, defaulting to `1.0`; `flat` from `pa.metadata?.in_debate_flat`, defaulting to `0`; and `final` from `pa.metadata?.final_contribution`, falling back to `base`. ### renderTimeline

`renderTimeline` accepts a `DebateMessage[]` array and a `SpectateDebate` object `d`. It reads the module-level `state.replayData` object and may write to `state.lastMessageTime` during iteration. It returns a string of HTML and touches no DOM directly. It is synchronous.

The accumulated HTML string is returned after the loop completes.

### renderMessages

It initializes a local accumulator `html` and a local integer `lastRound` at zero, then iterates over every message in `messages` in order. For each message, if the message has a `round` value that differs from `lastRound`, a round divider `<div>` is appended to `html` and `lastRound` is updated to `m.round`. The `side` is read from `m.side`, defaulting to `'a'` if absent. The display name is derived from `m.is_ai`: if true the name is the literal robot emoji string; otherwise, if `side === 'a'`, the name is `d.debater_a_name` (falling back to `'Side A'`), and if `side` is anything else, it is `d.debater_b_name` (falling back to `'Side B'`). A message `<div>` is then built with sub-divs for name, content, and round number, all passing user-derived strings through `escHtml()` and numeric values through `Number()`. After appending the message HTML, the function checks whether `m.created_at` is set and either greater than `state.lastMessageTime` or `state.lastMessageTime` is falsy; if so, it writes `m.created_at` to `state.lastMessageTime`, advancing the module-level watermark. The completed `html` string is returned; nothing is written to the DOM.

### formatPointBadge

`formatPointBadge` accepts a single `ReplayPointAward` parameter `pa` and returns a formatted string. It is synchronous and private (not exported). It reads from `pa.metadata` (which may be absent) to extract `base_score`, `in_debate_multiplier`, `in_debate_flat`, and `final_contribution`, falling back to `pa.base_score` for the base and sensible numeric defaults for the others, all cast with `Number()`. It then derives two boolean flags: `hasMult` (true when the multiplier is not exactly `1.0`) and `hasFlat` (true when the flat bonus is not exactly `0`). The return value is one of four string forms depending on those flags: if neither is set, `'+N pts'` using the base; if only `hasMult`, `'+base × mult = final pts'`; if only `hasFlat`, `'+final pts'` (the multiplier is suppressed); if both flags are set, `'+base × mult + flat = final pts'`. There are no loops, no DOM reads or writes, and no calls to other functions on the anchor list.

### renderTimeline

`renderTimeline` accepts a `DebateMessage[]` array and a `SpectateDebate` object `d`, and returns a string of HTML. It is synchronous. It reads `state.replayData` from module-level state, which holds replay data (power-ups, references, speech events, point awards) or is null/undefined. From `state.replayData` it reads `point_awards`, `speech_events`, `power_ups`, and `references`.

The function begins by building a `Map<string, ReplayPointAward>` called `pointAwardMap` and a separate array `orphanAwards`. If `rd?.point_awards` is non-empty, each award is examined: if `pa.metadata?.scored_event_id` is present the award is placed into `pointAwardMap` keyed by that ID; otherwise it is pushed into `orphanAwards`. Next, `hasSpeechEvents` is derived as true if `rd?.speech_events` has any entries. `hasEnrichment` is computed: it is true if `rd` is present and at least one of `power_ups`, `references`, `speech_events`, or `point_awards` is non-empty. If `hasEnrichment` is false and `messages` is non-empty, the function returns early by calling `renderMessages(messages, d)` and returning its result.

If the early return is not taken, a `TimelineEntry[]` array called `entries` is populated. When `hasSpeechEvents` is true, each speech event becomes a `'speech'` entry and each orphan award becomes a `'score'` entry. When false, each message becomes a `'message'` entry and all awards (orphans and pointAwardMap values) become `'score'` entries. Regardless of path, power-ups become `'power_up'` entries and references become `'reference'` entries. All entries are sorted chronologically.

The function iterates over sorted entries dispatching on `entry.type`: `'message'` emits round-dividers and message blocks as in `renderMessages`, updating `state.lastMessageTime`; `'speech'` does the same plus looks up `se.id` in `pointAwardMap` to append a score badge via `formatPointBadge(award)`; `'score'` emits standalone score-event div with `formatPointBadge(pa)`; `'power_up'` emits power-up-event div with `escHtml()`; `'reference'` emits reference-event div with ruling icon selection and optional sub-fields. The accumulated HTML string is returned.

### renderSpectateView

`renderSpectateView` accepts a `SpectateDebate` object `d` and a `DebateMessage[]` array `messages`. It is synchronous and returns `void`. It reads `state.loading`, `state.app`, `state.replayData`, `state.chatMessages`, `state.chatOpen`, `state.isLoggedIn`, and `state.lastRenderedMessageCount` from module-level state.

It hides `state.loading` if truthy, derives `isLive` from `d.status`, then assembles the page HTML by concatenating sections: status/mode badges (using `statusBadge` and `modeLabel`), topic card, VS bar (two `renderAvatar` calls), optional moderator bar, info bar, audience pulse gauge (branching on `totalVotes > 0`), optional live indicator, messages container (empty-state or `renderTimeline(messages, d)`), spectator chat (with optional `renderChatMessages`), conditional scoreboard, conditional moderator rating (filtering `mod_scores` by `scorer_role`, rendering fair/unfair per `s.score >= 25` for debaters and average bar for spectators), conditional AI scorecard (summing four criteria per side, rendering score bars, optional verdict), vote section, share bar, CTA banner, footer.

After assembly, sets `state.app.innerHTML`, writes `state.lastRenderedMessageCount`, then calls `wireVoteButtons(d)`, `wireShareButtons(d)`, `wireChatUI(d)`, and conditionally `updateVoteBar` if vote count > 0.

### formatPointBadge

`formatPointBadge` takes a single `ReplayPointAward` parameter `pa`. It is synchronous and returns a string. It reads `pa.metadata` and, from it, extracts `base_score`, `in_debate_multiplier`, `in_debate_flat`, and `final_contribution`, each cast with `Number()` and with fallbacks: `base` falls back to `pa.base_score ?? 0`, `mult` falls back to `1.0`, `flat` falls back to `0`, and `final` falls back to `base`. It then computes two booleans: `hasMult` is true when `mult !== 1.0`, and `hasFlat` is true when `flat !== 0`. The function branches on these two booleans across four exclusive cases. If neither is true, it returns `'+N pts'` using `base`. If only `hasMult` is true, it returns `'+base × mult = final pts'` using the Unicode multiplication sign `×`. ### renderTimeline

`renderTimeline` accepts a `DebateMessage[]` array and a `SpectateDebate` object `d`, and returns a string of HTML. It is synchronous. It reads `state.replayData` from module-level state, which holds enrichment data or is absent. From `state.replayData` it reads `point_awards`, `speech_events`, `power_ups`, and `references`.

The function builds a `Map<string, ReplayPointAward>` called `pointAwardMap` and a separate array `orphanAwards`. If `rd?.point_awards` is non-empty, each award is examined: if `pa.metadata?.scored_event_id` is present it is placed into `pointAwardMap` keyed by that ID; otherwise pushed into `orphanAwards`. `hasSpeechEvents` is true if `rd?.speech_events` has any entries. `hasEnrichment` is true if `rd` is set and any of the four data buckets is non-empty. If `hasEnrichment` is false and `messages` is non-empty, returns early via `renderMessages(messages, d)`.

Otherwise builds a unified `TimelineEntry[]`. When `hasSpeechEvents` is true, speech events become `'speech'` entries and orphan awards become `'score'` entries. When false, messages become `'message'` entries and all awards become `'score'` entries. Regardless, power-ups become `'power_up'` entries and references become `'reference'` entries. All entries are sorted chronologically by `timestamp.getTime()`.

Iteration dispatches on `entry.type`: `'message'` mirrors `renderMessages` behavior including `state.lastMessageTime` advancement; `'speech'` does the same plus checks `pointAwardMap` for `se.id` and appends a score badge with `formatPointBadge(award)` if found, also updating `state.lastMessageTime`; `'score'` appends standalone score-event div with `formatPointBadge(pa)` and debater name; `'power_up'` appends power-up-event div with all strings through `escHtml()`; `'reference'` selects ruling icon (✅/❌/⏳) and text, conditionally appends description, URL, ruling_reason, and wraps in reference-event div. All user-derived strings pass through `escHtml()`. Returns accumulated HTML.

### renderSpectateView

`renderSpectateView` accepts a `SpectateDebate` object `d` and a `DebateMessage[]` array, and returns `void`. It is synchronous. It reads `state.loading`, `state.app`, `state.replayData`, `state.chatMessages`, `state.chatOpen`, `state.isLoggedIn`, and `state.lastRenderedMessageCount` from module-level state. It writes to `state.lastRenderedMessageCount` at the end.

Begins by hiding `state.loading` if truthy via `style.display = 'none'`. Derives `isLive` from `d.status` matching `'live'`, `'pending'`, `'round_break'`, or `'voting'`. Builds large HTML string in sections: status/mode badges (`statusBadge(d.status)` and `modeLabel(d.mode)`); topic card (`escHtml(d.topic)`); VS bar with two `renderAvatar()` calls and `Number()` on ELO values; optional moderator bar when `d.moderator_type` is set and not `'none'` (label: `'AI Moderator'` for `'ai'` type or `escHtml(d.moderator_name)` fallback); info bar with spectator count and round fraction; audience pulse gauge branching on `(va + vb) > 0` for percentage fills or 50/50 placeholders; optional live indicator if `isLive`.

Messages block: if `messages.length === 0` and `state.replayData?.speech_events` empty, inserts placeholder; otherwise calls `renderTimeline(messages, d)`.

Chat block: reads `state.chatMessages.length` for count badge, `state.chatOpen` for CSS toggling, calls `renderChatMessages(state.chatMessages)` if non-empty, renders input row if `state.isLoggedIn` else login prompt.

Conditional sections when `d.status` is `'complete'`/`'completed'`: scoreboard if `d.score_a != null`; moderator rating if `d.moderator_type === 'human'` and `state.replayData.mod_scores` non-empty (debater scores with fair/unfair per `s.score >= 25`, spectator scores averaged with `Math.round` and bar at `avgScore * 2` percent); AI scorecard if `d.ai_scorecard` set (totals summed, four criteria score bars at `score * 10` percent, optional verdict).

Always appends: vote section, share bar (four buttons), CTA banner, footer.

After assembly: sets `state.app.innerHTML` if truthy, writes `state.lastRenderedMessageCount = messages.length`, calls `wireVoteButtons(d)`, `wireShareButtons(d)`, `wireChatUI(d)`, then conditionally `updateVoteBar` if `(d.vote_count_a || 0) + (d.vote_count_b || 0) > 0`. No try/catch, no async.
