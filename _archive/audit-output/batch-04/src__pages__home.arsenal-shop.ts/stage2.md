# Stage 2 Outputs — home.arsenal-shop.ts

## Agent 01

### loadShopScreen

`loadShopScreen` is async and takes a single `container: HTMLElement` parameter. On entry it writes `container` into the module-level `_container` variable. If `_sheetCleanup` is non-null it calls the stored cleanup function and sets `_sheetCleanup` to null. It then sets `container.innerHTML` to a loading placeholder string. It fires `getModifierCatalog()` and `getUserInventory()` concurrently via `Promise.all`, awaiting both. After both settle, it writes the resolved catalog array into `_state.catalog`. The `_state.tokenBalance` assignment that follows is structurally unusual: both branches of the ternary — whether `inventory?.powerup_stock != null` is truthy or falsy — call `_readTokenBalance()` and assign its return value, so the condition has no observable effect and `_readTokenBalance()` is always what sets `_state.tokenBalance`. Neither branch inspects the `inventory` object's contents for the balance value. If `getModifierCatalog()` or `getUserInventory()` throws, `Promise.all` will reject and the rejection propagates out of `loadShopScreen` uncaught — no try/catch exists in this function. After writing state, it calls `render(container)`.

### _readTokenBalance

`_readTokenBalance` takes no parameters. It queries the live DOM for the first element matching the attribute selector `[data-token-balance]`. If such an element exists, it reads `el.textContent`, falls back to `'0'` if that is null, parses it as a base-10 integer via `parseInt`, and returns `0` if the parse produces `NaN` (the `|| 0` guard). If no matching element exists in the DOM, it returns `0` immediately. It does not touch `_state` and has no async behavior. The comment in the source states the balance element is kept current by `home.ts`; this function does not verify that assumption.

### render

`render` takes a `container: HTMLElement` parameter and is synchronous. It immediately calls `_readTokenBalance()` and overwrites `_state.tokenBalance` with the result. It then calls `applyFilters(_state.catalog)`, capturing the returned array as `filtered`. It constructs a complete HTML string using `_state.productType`, `_state.tokenBalance`, `_state.categoryFilter`, `_state.rarityFilter`, `_state.timingFilter`, `_state.affordableOnly`, and the `CATEGORIES` and `RARITIES` module-level constant arrays. Category chip values and labels are passed through `escapeHTML`. Rarity chip values and labels are also passed through `escapeHTML`. If `filtered` is empty it renders a no-results message; otherwise it maps each element through `renderEffectCard` (imported from `modifiers.ts`), passing a display-options object whose `showModButton` and `showPuButton` flags are set based on `_state.productType`. The entire constructed string is assigned to `container.innerHTML`, replacing all previous content. After the DOM is written, it calls `wireEvents(container)`. `render` returns nothing.

### applyFilters

`applyFilters` takes a `catalog: ModifierEffect[]` parameter and returns a new array that is a filtered subset of that input. It reads `_state.categoryFilter`, `_state.rarityFilter`, `_state.timingFilter`, `_state.affordableOnly`, `_state.productType`, and `_state.tokenBalance` from module-level state. For each element it applies four sequential guard conditions. First, if `categoryFilter` is not `'all'` and the element's `category` field does not match, the element is excluded. Second, if `rarityFilter` is not `'all'` and the element's `tier_gate` field does not match, the element is excluded. Third, if `timingFilter` is not `'all'` and the element's `timing` field does not match, the element is excluded. Fourth, if `affordableOnly` is true, it reads either `e.mod_cost` or `e.pu_cost` depending on `_state.productType`, and excludes the element if that cost exceeds `_state.tokenBalance`. Elements that pass all checks are included. The function does not write to `_state` or the DOM and has no async behavior.

### wireEvents

`wireEvents` takes a `container: HTMLElement` parameter and is synchronous. It attaches click listeners to five categories of elements found within `container`.

For `[data-product]` buttons, each click listener writes `btn.dataset.product` cast to `ProductType` into `_state.productType` and then calls `render(container)`. For `[data-cat]` buttons, each click writes `btn.dataset.cat` cast to `ModifierCategory | 'all'` into `_state.categoryFilter` and calls `render(container)`. For `[data-rarity]` buttons, each click writes `btn.dataset.rarity` cast to `RarityTier | 'all'` into `_state.rarityFilter` and calls `render(container)`. For `[data-timing]` buttons, each click reads `btn.dataset.timing` and compares it to the current `_state.timingFilter`. If they are equal the filter resets to `'all'`; otherwise the filter is set to the button's timing value. Either way `render(container)` is called. For the single `[data-afford]` button, if the element exists, its click listener toggles `_state.affordableOnly` to its boolean opposite and calls `render(container)`.

For `.mod-effect-card` elements, each click listener checks whether the click target or any of its ancestors matches `.mod-buy-btn`; if so, the handler returns early without doing anything. Otherwise it reads `card.dataset.effectId`, returns early if that is falsy, searches `_state.catalog` for the matching effect by `id`, and if found calls `openBottomSheet(effect)`.

For `.mod-buy-btn` buttons, each async click listener calls `e.stopPropagation()`, reads `btn.dataset.effectId`, returns early if falsy, searches `_state.catalog` for the matching effect, and if found calls `openBottomSheet(effect)`. Despite the handler being declared `async`, it contains no `await` expressions; the `async` keyword has no observable runtime effect here.

### openBottomSheet

`openBottomSheet` takes an `effect: ModifierEffect` parameter and is synchronous as a function (the confirm click handler inside it is async). On entry, if `_sheetCleanup` is non-null it calls the stored function and sets `_sheetCleanup` to null, tearing down any pre-existing sheet. It reads `_state.productType` to derive `cost` (either `effect.mod_cost` or `effect.pu_cost`) and `productLabel`. It builds `socketNote` as an HTML string: for `'modifier'` product type it includes tier-gate information using `tierLabel(effect.tier_gate)`; for `'powerup'` it includes consumable-use text. It computes `canAfford` by comparing `_state.tokenBalance` to `cost`. It creates a new `div` element, sets its `className` to `'bottom-sheet-overlay'`, and assigns a constructed HTML string to its `innerHTML`. The HTML includes `escapeHTML(effect.name)`, `escapeHTML(effect.description)`, `escapeHTML(effect.id)`, `rarityClass(effect.tier_gate)` (the local function), `tierLabel(effect.tier_gate)`, and `categoryLabel(effect.category)` from imports. The `cost` value interpolated into `innerHTML` is a number from the effect object, not user-supplied string data. The confirm button is rendered `disabled` when `canAfford` is false. The completed `overlay` element is appended to `document.body`.

A `close` function is defined that calls `overlay.remove()` and sets `_sheetCleanup` to null. A click listener on the overlay itself calls `close()` if the click target is the overlay element directly (i.e., clicking the backdrop). The `#sheet-cancel` button's click listener also calls `close()`.

The `#sheet-confirm` button is queried; if it exists and `canAfford` is true, an async click listener is attached. When fired, it sets `confirmBtn.disabled = true` and its `textContent` to `'Purchasing…'`. It reads `confirmBtn.dataset.product` as `ProductType`. If `'modifier'`, it awaits `handleBuyModifier(effect.id, effect.name)`; otherwise it awaits `handleBuyPowerup(effect.id, effect.name)`. Both return a boolean into `ok`. After the await, it calls `close()`. If `ok` is true and `_container` is non-null, it calls `render(_container)` to refresh the grid with an updated token balance. If `canAfford` is false, no listener is attached to `confirmBtn` and the button remains permanently disabled.

Finally, `_sheetCleanup` is set to the `close` function.

### rarityClass

`rarityClass` takes a `tier: RarityTier` parameter and returns a `string`. It constructs a local `Record<RarityTier, string>` map with five entries: `common → 'common'`, `uncommon → 'uncommon'`, `rare → 'rare'`, `legendary → 'legendary'`, `mythic → 'mythic'`. It looks up `tier` in the map and returns the result, or `'common'` if the key is not found (via the `?? 'common'` fallback). Because the map's values are identical to the keys, the function effectively returns the input string unchanged for all known tier values, and returns `'common'` for any unknown value. It does not touch module-level state or the DOM.

### cleanupShopScreen

`cleanupShopScreen` takes no parameters and is synchronous. If `_sheetCleanup` is non-null, it calls the stored function (which removes the bottom-sheet overlay from the DOM and nulls `_sheetCleanup`), then explicitly sets `_sheetCleanup` to null. It then sets the module-level `_container` to null. It does not reset `_state`, so filter selections and the cached catalog remain in place for the next time `loadShopScreen` is called. It returns nothing.

`applyFilters` is a synchronous function that takes `catalog: ModifierEffect[]` and returns a filtered `ModifierEffect[]`. It calls `Array.prototype.filter` on the input array, running each element through a predicate that reads `_state.categoryFilter`, `_state.rarityFilter`, `_state.timingFilter`, `_state.affordableOnly`, `_state.productType`, and `_state.tokenBalance`. The predicate applies up to four successive rejection conditions in order: if `categoryFilter` is not `'all'` and the element's `category` does not match, it returns false; if `rarityFilter` is not `'all'` and the element's `tier_gate` does not match, it returns false; if `timingFilter` is not `'all'` and the element's `timing` does not match, it returns false; if `affordableOnly` is true, it reads `e.mod_cost` or `e.pu_cost` depending on `_state.productType` and returns false when that cost exceeds `_state.tokenBalance`. If all conditions pass, the predicate returns true. There are no early returns at the function level, no error paths, and no async behavior.

### wireEvents

`wireEvents` is a synchronous function that takes `container: HTMLElement`. It does not return a value. It attaches click listeners to all DOM elements within `container` that match specific `data-*` selectors.

For `[data-product]` buttons, each click writes the button's `dataset.product` cast to `ProductType` into `_state.productType` and calls `render(container)`. For `[data-cat]` buttons, each click writes `dataset.cat` cast to `ModifierCategory | 'all'` into `_state.categoryFilter` and calls `render(container)`. For `[data-rarity]` buttons, each click writes `dataset.rarity` into `_state.rarityFilter` and calls `render(container)`. For `[data-timing]` buttons, each click reads the button's `dataset.timing`, compares it to the current `_state.timingFilter`, sets `_state.timingFilter` to `'all'` if they match (toggle-off) or to the timing value if they do not (toggle-on), then calls `render(container)`. For the single `[data-afford]` button (queried with `querySelector`, not `querySelectorAll`), a click toggles `_state.affordableOnly` to its boolean inverse and calls `render(container)`. If no `[data-afford]` element is found, the block is skipped entirely.

For `.mod-effect-card` elements, each click checks whether the event target or any ancestor matches `.mod-buy-btn` via `e.target.closest()`; if so, it returns early and does nothing. Otherwise it reads `card.dataset.effectId`, returns early if falsy, searches `_state.catalog` for the matching element by `id`, returns early if not found, then calls `openBottomSheet(effect)`. For `.mod-buy-btn` buttons, each click (marked `async`) calls `e.stopPropagation()`, reads `btn.dataset.effectId`, returns early if falsy, searches `_state.catalog` for the matching element, returns early if not found, then calls `openBottomSheet(effect)`. The click handler on `.mod-buy-btn` is declared `async` but contains no `await` expressions; `openBottomSheet` is called synchronously within it.

### openBottomSheet

`openBottomSheet` is a synchronous function that takes `effect: ModifierEffect`. It first checks `_sheetCleanup`; if set, it calls it and sets it to null, closing any previously open sheet. It reads `_state.productType` to determine the cost field (`effect.mod_cost` vs `effect.pu_cost`) and the product label string. It constructs a `socketNote` string whose content depends on whether `productType` is `'modifier'` (showing a tier-gate note via `tierLabel(effect.tier_gate)`) or `'powerup'` (showing a consumable note). It reads `_state.tokenBalance` to compute `canAfford` as a boolean.

It creates a new `div` element, assigns it the class `bottom-sheet-overlay`, and sets its `innerHTML` to a template that includes: `escapeHTML(effect.name)`, `rarityClass(effect.tier_gate)` and `tierLabel(effect.tier_gate)` for the rarity badge, `effect.timing` for timing badge styling, `categoryLabel(effect.category)`, `escapeHTML(effect.description)`, `socketNote`, the cost value, a conditional broke-note paragraph when `!canAfford`, and a confirm button whose `disabled` attribute and CSS class are set when `!canAfford`. The confirm button's `data-effect-id` attribute is set via `escapeHTML(effect.id)`. The overlay element is appended to `document.body`.

A `close` closure is defined that removes the overlay from the DOM and sets `_sheetCleanup` to null. The function attaches a click listener on the overlay itself that calls `close()` only when `e.target === overlay` (a click directly on the backdrop). It attaches a click listener to `#sheet-cancel` that calls `close()`. It queries `#sheet-confirm`; if found and `canAfford` is true, it attaches an `async` click listener that sets `confirmBtn.disabled = true`, sets `confirmBtn.textContent` to `'Purchasing…'`, reads `confirmBtn.dataset.product` to determine the `productType`, then awaits either `handleBuyModifier(effect.id, effect.name)` or `handleBuyPowerup(effect.id, effect.name)` depending on that value, assigning the boolean result to `ok`. After the await resolves, it calls `close()` unconditionally, then checks whether `ok` is true and `_container` is non-null; if both hold, it calls `render(_container)`. If `confirmBtn` is not found or `canAfford` is false, no confirm listener is attached and the button remains inert. After all listeners are wired, `_sheetCleanup` is set to the `close` closure.

### rarityClass

`rarityClass` is a synchronous function that takes `tier: RarityTier` and returns a string. It defines a local `Record<RarityTier, string>` map with five entries, each mapping a rarity tier name to an identical string (`'common'` → `'common'`, etc.). It looks up `tier` in the map and returns the result, falling back to `'common'` via the `??` nullish-coalescing operator if the lookup returns `undefined`. The map values are identity mappings, so the function effectively validates and normalizes the tier name, substituting `'common'` for any unrecognized input.

### cleanupShopScreen

`cleanupShopScreen` is a synchronous exported function with no parameters and no return value. It checks `_sheetCleanup`; if it is set, it calls it and sets `_sheetCleanup` to null, which removes the bottom sheet overlay from the DOM and clears that reference. It then sets `_container` to null. It does not reset `_state` and does not clear any DOM content from the container itself beyond what the `_sheetCleanup` call does.

## Agent 05

### loadShopScreen

`loadShopScreen` is async and takes a single `container: HTMLElement` parameter. It writes `container` into the module-level `_container` variable. If `_sheetCleanup` is non-null, it calls that function and then sets `_sheetCleanup` to null, dismissing any open bottom sheet before reinitializing. It then sets `container.innerHTML` to a loading placeholder string. It fires `getModifierCatalog()` and `getUserInventory()` in parallel via `Promise.all`, awaiting both. Once both settle, it writes the resolved catalog array into `_state.catalog`. The assignment to `_state.tokenBalance` always calls `_readTokenBalance()` regardless of whether `inventory?.powerup_stock` is null or not — both branches of the ternary call `_readTokenBalance()`, so the condition on `inventory?.powerup_stock` has no observable effect on what value is stored. There is no catch block; if either network call rejects, the rejection propagates to the caller and `render` is never called. On the happy path, `render(container)` is called after state is written.

### _readTokenBalance

`_readTokenBalance` is synchronous and takes no parameters. It queries the live DOM for the first element matching the attribute selector `[data-token-balance]`. If such an element exists, it reads `el.textContent`, substitutes an empty string if `textContent` is null, parses the result as a base-10 integer, and returns that integer, falling back to `0` if `parseInt` yields `NaN`. If no matching element is found, it returns `0`. The function reads no module-level state and writes nothing; it always returns a number.

### render

`render` is synchronous and takes `container: HTMLElement`. It immediately calls `_readTokenBalance()` and writes the result back into `_state.tokenBalance`, refreshing the balance before building HTML. It then calls `applyFilters(_state.catalog)` to produce a filtered array. It builds a full HTML string using `_state.productType`, `_state.categoryFilter`, `_state.rarityFilter`, `_state.timingFilter`, `_state.affordableOnly`, and `_state.tokenBalance` — all read from module-level state. Category chip buttons are rendered by mapping over the module-level `CATEGORIES` constant, and rarity chip buttons by mapping over `RARITIES`; both apply `escapeHTML` to value and label strings. If `filtered.length` is zero, the card grid contains a single "No effects match" empty-state div. Otherwise it maps over `filtered`, calling `renderEffectCard(e, { showModButton, showPuButton })` for each item, where the two flags are derived from `_state.productType`. The completed string is written to `container.innerHTML`, replacing all previous content. Finally, `wireEvents(container)` is called to attach listeners to the newly inserted DOM. `render` returns nothing.

### applyFilters

`applyFilters` is synchronous and takes `catalog: ModifierEffect[]`. It returns a new array produced by `Array.prototype.filter` over the input. For each element `e`, four sequential rejection checks run, each guarded by an early `return false` if the condition fails. First, if `_state.categoryFilter` is not `'all'` and `e.category` does not equal `_state.categoryFilter`, the item is excluded. Second, if `_state.rarityFilter` is not `'all'` and `e.tier_gate` does not equal `_state.rarityFilter`, the item is excluded. Third, if `_state.timingFilter` is not `'all'` and `e.timing` does not equal `_state.timingFilter`, the item is excluded. Fourth, if `_state.affordableOnly` is true, the cost is selected as `e.mod_cost` when `_state.productType` is `'modifier'` and `e.pu_cost` otherwise; if that cost exceeds `_state.tokenBalance`, the item is excluded. Items that pass all checks are included. The function reads `_state` for all filter values but writes nothing; it does not mutate the input array.

### wireEvents

`wireEvents` is synchronous and takes `container: HTMLElement`. It attaches `click` listeners to six distinct groups of elements within `container`, relying entirely on the DOM having been written by the preceding `render` call.

For `[data-product]` buttons, each click writes `btn.dataset.product` cast to `ProductType` into `_state.productType` and then calls `render(container)`. For `[data-cat]` buttons, each click writes `btn.dataset.cat` cast to `ModifierCategory | 'all'` into `_state.categoryFilter` and calls `render(container)`. For `[data-rarity]` buttons, each click writes `btn.dataset.rarity` into `_state.rarityFilter` and calls `render(container)`. For `[data-timing]` buttons, each click reads the button's `data-timing` value; if `_state.timingFilter` already equals that value the filter resets to `'all'`, otherwise it is set to the clicked value; then `render(container)` is called. For the single `[data-afford]` button, if found, the click toggles `_state.affordableOnly` and calls `render(container)`. All five of those groups replace the entire DOM and re-wire on every interaction.

For `.mod-effect-card` elements, each click checks whether the click target or any of its ancestors matches `.mod-buy-btn` via `closest`; if so, the handler returns without opening a sheet. Otherwise, `card.dataset.effectId` is read and the item is looked up by `id` in `_state.catalog`; if the effect is found, `openBottomSheet(effect)` is called. For `.mod-buy-btn` buttons, each click calls `e.stopPropagation()`, reads `btn.dataset.effectId`, looks up the effect in `_state.catalog`, and if found calls `openBottomSheet(effect)`. Both card-click and buy-button handlers ultimately invoke `openBottomSheet` — there is no path where a buy-button tap skips the sheet and goes directly to a purchase. The handler is declared `async` but the body awaits nothing; `openBottomSheet` is not async, so the async declaration has no observable effect.

### openBottomSheet

`openBottomSheet` is synchronous and takes `effect: ModifierEffect`. If `_sheetCleanup` is non-null, it calls it and sets `_sheetCleanup` to null before proceeding, dismissing any already-open sheet. It reads `_state.productType` and `_state.tokenBalance` from module-level state to derive `cost`, `productLabel`, `socketNote`, and the `canAfford` boolean. It creates a new `div` element, sets its class to `'bottom-sheet-overlay'`, and writes an HTML string into its `innerHTML` using `escapeHTML` for `effect.name`, `effect.description`, and `effect.id`; `rarityClass` and `tierLabel` are called for badge classes and labels; `categoryLabel` is called for the category badge. If `canAfford` is false, the confirm button receives the `disabled` attribute and the `'disabled'` class, and an additional "not enough tokens" paragraph is rendered. The overlay is appended to `document.body`.

A `close` closure is defined that calls `overlay.remove()` and nulls `_sheetCleanup`. A click listener on the overlay itself calls `close()` when `e.target === overlay` (clicking the backdrop). The `#sheet-cancel` button's click calls `close()`. If `canAfford` is true and `#sheet-confirm` is found, an async click listener is attached: it disables the button, sets its text to `'Purchasing…'`, reads `confirmBtn.dataset.product` as `ProductType`, then awaits either `handleBuyModifier(effect.id, effect.name)` or `handleBuyPowerup(effect.id, effect.name)` depending on the product type, each returning a boolean assigned to `ok`. After the await, `close()` is called unconditionally. If `ok` is true and `_container` is non-null, `render(_container)` is called to refresh the grid with an updated token balance. If `canAfford` is false, no click listener is attached to the confirm button; the button is disabled in the DOM as well. After all listener setup, `_sheetCleanup` is set to the `close` closure.

### rarityClass

`rarityClass` is synchronous and takes `tier: RarityTier`. It defines a local `Record<RarityTier, string>` mapping each of the five rarity values (`common`, `uncommon`, `rare`, `legendary`, `mythic`) to the same string as the key. It returns the mapped string for the given `tier`, or `'common'` if the key is not found via the nullish coalescing fallback. No module-level state is read or written; the function has no side effects and always returns a string.

### cleanupShopScreen

`cleanupShopScreen` is synchronous and exported. It takes no parameters. If `_sheetCleanup` is non-null, it calls `_sheetCleanup()` — which removes the bottom sheet overlay from the DOM and nulls `_sheetCleanup` internally — then explicitly sets `_sheetCleanup` to null. It then sets `_container` to null. No other state is reset: `_state` retains whatever values it held at the time of the call, including catalog, filters, and token balance.
