# Stage 2 Outputs — async.fetch.ts

## Agent 01

### fetchTakes

`fetchTakes` is an async function that accepts an optional `section` string parameter. On entry it reads `FEATURES.hotTakes` from `config.ts` and returns immediately if that flag is falsy. It then calls `getSupabaseClient()` and `getIsPlaceholderMode()` from `auth.ts`; if the client is absent or placeholder mode is active it returns without doing anything. Inside a `try` block it constructs a PostgREST query against the `hot_takes` table, selecting `id`, `content`, `section`, `created_at`, `user_id`, `reaction_count`, `challenge_count`, and a joined `profiles` sub-select (`username`, `display_name`, `elo_rating`, `verified_gladiator`), ordered by `created_at` descending, limited to 30 rows. If `section` is provided and is not `'all'`, it appends an `.eq('section', section)` filter to the query before awaiting it. The resolved `{ data, error }` is destructured; if `error` is truthy it is thrown, jumping to the outer `catch`. If `data` is non-null and non-empty, `state.hotTakes` (module-level state from `async.state.ts`) is replaced wholesale with a mapped array of plain objects, each populated from the row and its joined `profiles` record, with `userReacted` initialized to `false` and `time` produced by calling `_timeAgo` (from `async.utils.ts`) on `created_at`. After writing `state.hotTakes`, the function calls `getCurrentUser()` from `auth.ts` to read the current user's `id`. If an `id` is present, it opens a second `try` block and awaits a query against `hot_take_reactions` selecting `hot_take_id`, filtered to the current user and filtered to the set of IDs now in `state.hotTakes`. If that query resolves with a non-null `data` array, it builds a `Set` of reacted IDs and iterates over `state.hotTakes` in place, setting each element's `userReacted` to `true` where the ID is in the set. Any error in this inner `try` is silently swallowed with a comment marking it non-critical. Any error from the outer `try` — including a thrown PostgREST error — is caught and logged to `console.error`. The function returns `void` in all paths; it writes only to `state.hotTakes` and to each element's `userReacted` property.

### fetchPredictions

`fetchPredictions` is an async function that takes no parameters. It reads `FEATURES.predictions` from `config.ts` and returns immediately if that flag is falsy. It then calls `getSupabaseClient()` and `getIsPlaceholderMode()`; if either check fails it returns. Inside a `try` block it awaits `safeRpc<Record<string, unknown>[]>('get_hot_predictions', { p_limit: 10 })` from `auth.ts`. The destructured `{ data, error }` is checked: if `error` is truthy it is thrown to the `catch` block. If `data` is non-null and non-empty, `state.predictions` (module-level state) is replaced with a mapped array of plain objects. Each object is built from the raw RPC row: `debate_id`, `topic`, `status` are cast directly; `p1` and `p2` are derived from `p1_username`/`p1_display_name` and `p2_username`/`p2_display_name` with a fallback of `'Side A'` and `'Side B'`; `p1_elo` and `p2_elo` fall back to `1200`; `total` falls back to `0`; `pct_a` and `pct_b` are computed as rounded percentages from `picks_a`/`picks_b` over `prediction_count`, defaulting to `50` each when `prediction_count` is zero; `user_pick` is always set to `null`. Any thrown error is caught and logged to `console.error`. The function returns `void`; the only external write is the replacement of `state.predictions`.

### fetchStandaloneQuestions

`fetchStandaloneQuestions` is an async function that accepts an optional `category` string parameter. It has no feature-flag guard at entry. It calls `getSupabaseClient()` and `getIsPlaceholderMode()`; if the client is absent or placeholder mode is active it returns immediately. Inside a `try` block it awaits `safeRpc<StandaloneQuestion[]>('get_prediction_questions', { p_limit: 20, p_category: category || null })`. The `category` parameter is forwarded as `p_category`, coerced to `null` when `category` is falsy. The destructured `{ data, error }` is checked: if `error` is truthy it is thrown. If `data` is non-null, is an array, and has at least one element, `state.standaloneQuestions` (module-level state) is replaced with `data` cast to `StandaloneQuestion[]` — no field-level mapping is performed; the RPC response is assigned directly. Any thrown error is caught and logged to `console.error`. The function returns `void`; the only external write is the replacement of `state.standaloneQuestions`.

### fetchTakes

`fetchTakes` is async and accepts an optional `section` string parameter. At entry it reads `FEATURES.hotTakes` from the `config` module and returns immediately if that flag is falsy. It then calls `getSupabaseClient()` and `getIsPlaceholderMode()` from `auth`; if the client is absent or placeholder mode is active it returns immediately. Any error in the reaction sub-query is silently caught. Any error in the outer `try` block is caught and logged via `console.error`. `fetchTakes` returns `void` and has no return value.

### fetchStandaloneQuestions

### fetchTakes

`fetchTakes` is async and accepts an optional `section` string parameter. It returns `Promise<void>`. On entry, it reads `FEATURES.hotTakes` from the imported `config.ts` module-level constant and returns immediately if that flag is falsy. It then calls `getSupabaseClient()` and `getIsPlaceholderMode()`, both imported from `auth.ts`, and returns without doing anything if the client is absent or placeholder mode is active. Inside a `try` block, it constructs a PostgREST query against the `hot_takes` table, selecting `id`, `content`, `section`, `created_at`, `user_id`, `reaction_count`, `challenge_count`, and a joined `profiles` sub-select for `username`, `display_name`, `elo_rating`, and `verified_gladiator`. The query is ordered by `created_at` descending and limited to 30 rows. If `section` is provided and is not `'all'`, an `.eq('section', section)` filter is appended to the query before awaiting it. If the resolved `error` is truthy, it is thrown. If `data` is non-null and non-empty, `state.hotTakes` — the `hotTakes` array on the imported `state` object from `async.state.ts` — is fully replaced with a mapped array. Each element is a plain object whose `time` field is computed by calling `_timeAgo` (from `async.utils.ts`) on the `created_at` string; `userReacted` is initialized to `false` for every element. After mapping, `getCurrentUser()` is called to read the current authenticated user's `id`. If an `id` is present, a second awaited query is issued against `hot_take_reactions`, selecting `hot_take_id`, filtered by `.eq('user_id', userId)` and `.in('hot_take_id', ...)` using the IDs from the freshly written `state.hotTakes`. If that query resolves with `reacts` data, the IDs are loaded into a `Set` and each entry in `state.hotTakes` has its `userReacted` property mutated to `true` if its `id` is in the set. Any error from the reactions sub-query is caught silently by an inner `try/catch` with no action. Any error from the outer block is caught by the outer `catch` and logged via `console.error`. There is no return value beyond `void`; the only external write is `state.hotTakes`.

If that query returns data, the IDs are placed into a `Set` and `state.hotTakes` is mutated in place, setting `userReacted = true` on each entry whose `id` appears in the set. The function returns `void` in all paths; `state.hotTakes` is the sole write destination.
