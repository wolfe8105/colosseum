# Stage 2 Outputs — src/intro-music.ts

## Agent 01

### _injectCSS

`_injectCSS` reads the module-level boolean `_cssInjected`, which is initialized to `false`. If `_cssInjected` is already `true`, the function returns immediately without doing anything. Otherwise it sets `_cssInjected` to `true`, creates a `<style>` element via `document.createElement('style')`, assigns a multi-rule CSS string to `s.textContent`, and appends the element to `document.head`. The CSS string defines all visual rules for the picker: `.im-backdrop`, `.im-sheet`, `.im-handle`, `.im-title`, `.im-subtitle`, `.im-grid`, `.im-track-btn` and its hover/selected/after variants, `.im-track-icon`, `.im-track-info`, `.im-track-label`, `.im-track-desc`, `.im-preview-btn`, `.im-tier2-section`, `.im-tier2-label`, `.im-tier2-locked`, `.im-upload-btn`, `.im-upload-desc`, `.im-save-btn`, and two keyframe animations (`imFadeIn`, `imSlideUp`). The function is synchronous and returns nothing. After the first call, every subsequent call is a no-op due to the guard.

### openIntroMusicPicker

`openIntroMusicPicker` is synchronous and exported. It first calls `_injectCSS`, then reads the current user profile by calling `getCurrentProfile()`. From the returned profile object it derives four local variables: `currentId` (the `intro_music_id` field, defaulting to `'gladiator'` if absent), `currentUrl` (the `custom_intro_url` field, defaulting to `null`), `depthPct` (the `profile_depth_pct` field, defaulting to `0`), and the boolean `tier2Unlocked` (`depthPct >= 35`). It then removes any pre-existing element with id `im-backdrop` from the DOM. It creates a `div.im-backdrop` and a `div.im-sheet`, builds a `tier2Html` string conditionally: if `tier2Unlocked` is true the string contains an upload button and a hidden file input; otherwise it contains a locked message showing the current depth percentage. The sheet's `innerHTML` is set to a template that includes the drag handle, title, subtitle, a track grid built by mapping `INTRO_TRACKS` (imported from `arena-sounds.ts`) into buttons with `data-id` attributes and embedded preview buttons, the `tier2Html` block, and a save button. The sheet is appended to the backdrop, and the backdrop is appended to `document.body`.

After building the DOM, the function initializes three closure-scoped variables: `selectedId` (starts at `currentId`), `pendingFile` (starts at `null`), and `pendingUrl` (starts at `currentUrl`). It then attaches four categories of event listeners. Track buttons (`.im-track-btn[data-id]`) each get a click listener that early-returns if the click target is a `.im-preview-btn`; otherwise it sets `selectedId` to the button's `data-id`, sets `pendingFile` to `null`, and calls `_refreshSelected(sheet, selectedId)`. Preview buttons (`.im-preview-btn`) each get a click listener that calls `e.stopPropagation()` then calls `playIntroMusic` with the button's `data-preview` value. The upload button, if present, gets a click listener that triggers `fileInput.click()`. The file input gets a `change` listener that reads `fileInput.files[0]`; if no file is present it returns early; if the file exceeds 5 MB it calls `showToast('File too large — max 5MB', 'error')` and returns; otherwise it sets `pendingFile` to the file, sets `selectedId` to `'custom'`, updates the upload button's text content to the escaped filename (truncated to 28 characters), calls `_refreshSelected(sheet, 'custom')`, creates an object URL with `URL.createObjectURL(file)`, and calls `playIntroMusic('custom', localUrl)`. The save button gets an async click listener that disables the button, sets its text to `'SAVING…'`, then awaits `_saveIntroMusic(selectedId, pendingFile, pendingUrl)` inside a try/catch. On success it calls `showToast('Intro music saved ✓', 'success')` then `_close(backdrop)`. On failure it calls `showToast` with the error message or `'Save failed'`, re-enables the button, and resets its text to `'SAVE'`. Finally, the backdrop gets a click listener that calls `_close(backdrop)` only when `e.target === backdrop` (i.e., a click on the overlay itself, not the sheet).

### _refreshSelected

`_refreshSelected` takes two parameters: `sheet` (an `HTMLElement`) and `id` (a string). It queries all elements matching `.im-track-btn` within `sheet` and for each one calls `classList.toggle('selected', ...)`, passing `true` if the element's `dataset.id` equals `id` and `false` otherwise — this clears `selected` from all non-matching buttons and sets it on the matching one in a single pass. It then queries for `#im-upload-btn` within `sheet`; if that element exists, it calls `classList.toggle('selected', id === 'custom')`, adding `selected` when `id` is `'custom'` and removing it otherwise. The function is synchronous, reads no module-level state, and returns nothing.

### _close

`_close` takes one parameter: `backdrop`, an `HTMLElement`. It sets `backdrop.style.opacity` to `'0'` and `backdrop.style.transition` to `'opacity 0.2s'`, which causes the element to fade out via CSS transition. It then calls `setTimeout` with a callback that calls `backdrop.remove()` and a delay of 220 milliseconds. The 220 ms delay is slightly longer than the 0.2s (200 ms) transition so the element is removed after the fade-out animation completes. The function is synchronous, writes no module-level state, and returns nothing.

### _saveIntroMusic

`_saveIntroMusic` is async and takes three parameters: `trackId` (a string), `file` (a `File` or `null`), and `existingUrl` (a string, `null`, or `undefined`). It declares `uploadedUrl` as `string | undefined`, initially undefined. The function enters a branch on whether `trackId === 'custom'`. Inside that branch it further branches on whether `file` is truthy. If `file` is present, it calls `getSupabaseClient()` and throws `new Error('Not connected')` if the result is falsy. It calls `getCurrentProfile()` and throws `new Error('Not signed in')` if the result is falsy. It derives the file extension from `file.name` (defaulting to `'mp3'`) and constructs a storage path of the form `{profile.id}/intro.{ext}`. It then awaits `client.storage.from('intro-music').upload(path, file, { upsert: true, contentType: file.type })`; if `upErr` is truthy it throws `new Error(upErr.message)`. It then awaits `client.storage.from('intro-music').createSignedUrl(path, 31536000)` (one year in seconds); if `signErr` is truthy or `signedData.signedUrl` is absent it throws `new Error('Could not get URL')`; otherwise it sets `uploadedUrl` to `signedData.signedUrl`. If `file` is falsy but `existingUrl` is truthy, `uploadedUrl` is set to `existingUrl`. If both are falsy, the function throws `new Error('No file selected')`. If `trackId` is not `'custom'`, the entire upload block is skipped and `uploadedUrl` remains undefined.

After the conditional upload block, the function awaits `safeRpc('save_intro_music', { p_track_id: trackId, p_custom_url: uploadedUrl ?? null })`. If the returned `error` is truthy, it throws `new Error(error.message ?? 'Save failed')`. It then casts `data` to `{ error?: string } | null` and throws `new Error(result.error)` if `result?.error` is present. On the success path, it calls `getCurrentProfile()` again and, if the profile exists, directly mutates two properties on it via a `Record<string, unknown>` cast: `intro_music_id` is set to `trackId` and `custom_intro_url` is set to `uploadedUrl ?? null`. The function returns `Promise<void>` and propagates all errors as thrown exceptions rather than catching them, leaving error display to the caller in `openIntroMusicPicker`.

On success, it assigns the signed URL to `uploadedUrl`. If `file` is absent but `existingUrl` is truthy, `uploadedUrl` is set to `existingUrl` without any network call. If both are absent, it throws `new Error('No file selected')`. When `trackId` is not `'custom'`, the entire block is skipped and `uploadedUrl` remains `undefined`.

After the conditional block, the function always awaits `safeRpc('save_intro_music', { p_track_id: trackId, p_custom_url: uploadedUrl ?? null })`. If the RPC response contains a top-level `error`, it throws `new Error(error.message ?? 'Save failed')`. It then casts `data` to `{ error?: string } | null` and, if `data.error` is set, throws `new Error(result.error)`. On the happy path it reads `getCurrentProfile()` again; if a profile object is present it directly mutates its `intro_music_id` and `custom_intro_url` properties to reflect the newly saved values, casting the profile to `Record<string, unknown>` to bypass TypeScript's type constraints. The function returns `Promise<void>` and propagates all thrown errors to the caller — `openIntroMusicPicker`'s save handler catches them and displays a toast.

## Agent 05

### _injectCSS

`_injectCSS` reads the module-level boolean `_cssInjected`, which is initialized to `false`. If `_cssInjected` is already `true`, the function returns immediately without doing anything. Otherwise, it sets `_cssInjected` to `true`, creates a `<style>` element via `document.createElement('style')`, assigns a multi-rule CSS string to its `textContent`, and appends the element to `document.head`. The CSS defines all styles for the bottom-sheet UI: backdrop and sheet layout, slide-up and fade-in keyframe animations, track button grid, preview button, tier-2 upload section, and the save button. The function is synchronous and returns nothing. It is idempotent after the first call.

### openIntroMusicPicker

`openIntroMusicPicker` is synchronous and returns nothing. It begins by calling `_injectCSS`, which injects the stylesheet on first call and is a no-op thereafter. It then calls `getCurrentProfile()` to read the current user's profile from the auth module's cache, extracting `intro_music_id` (defaulting to `'gladiator'`), `custom_intro_url` (defaulting to `null`), and `profile_depth_pct` (defaulting to `0`). It derives the boolean `tier2Unlocked` by checking whether `depthPct >= 35`. Any existing element with id `im-backdrop` is removed from the DOM before a new one is created. The function then constructs a `div.im-backdrop` containing a `div.im-sheet` whose `innerHTML` is built from the `INTRO_TRACKS` array (imported from `arena-sounds.ts`), calling `escapeHTML` on each track's `label` and `description`. If `tier2Unlocked` is true, the sheet also includes a file upload button and a hidden `<input type="file">`; otherwise it renders a locked placeholder showing the user's current depth percentage.

After injecting the backdrop into `document.body`, the function establishes three pieces of closure-local state: `selectedId` (initialized to `currentId`), `pendingFile` (initialized to `null`), and `pendingUrl` (initialized to `currentUrl`). It then registers four categories of event listeners. Track buttons (`[data-id]`) check on click whether the click target is a `.im-preview-btn` descendant and return early if so; otherwise they update `selectedId`, clear `pendingFile`, and call `_refreshSelected(sheet, selectedId)` to update CSS classes. Preview buttons stop click propagation and call `playIntroMusic` with the track id. The upload button (if present) triggers `fileInput.click()`; when the file input fires `change`, it reads `fileInput.files[0]`, returns early if absent, calls `showToast` with an error and returns if the file exceeds 5 MB, otherwise sets `pendingFile` and `selectedId` to `'custom'`, updates the upload button's `textContent` with the filename (truncated to 28 characters, passed through `escapeHTML`), calls `_refreshSelected(sheet, 'custom')`, creates a local object URL with `URL.createObjectURL`, and calls `playIntroMusic('custom', localUrl)` for an in-browser preview. The save button's async click handler disables the button and sets its text to `'SAVING…'`, then awaits `_saveIntroMusic(selectedId, pendingFile, pendingUrl)`. On success it calls `showToast` with a success message and calls `_close(backdrop)`. On error it calls `showToast` with the error message, re-enables the button, and resets its text to `'SAVE'`. A click on the backdrop element itself (checked via `e.target === backdrop`) calls `_close(backdrop)`.

### _refreshSelected

`_refreshSelected` accepts a `sheet` element and a track `id` string. It queries all `.im-track-btn` elements within `sheet` and for each one calls `classList.toggle('selected', ...)` with a boolean derived by comparing `(b as HTMLElement).dataset.id` to `id`. This adds the `selected` class to the matching button and removes it from all others in a single pass. It then queries for `#im-upload-btn` within `sheet` and, if found, calls `classList.toggle('selected', id === 'custom')`, adding or removing `selected` on the upload button depending on whether `id` equals `'custom'`. The function is synchronous, writes only to the DOM, and returns nothing.

### _close

`_close` accepts a `backdrop` element. It sets `backdrop.style.opacity` to `'0'` and `backdrop.style.transition` to `'opacity 0.2s'`, triggering a CSS fade-out. It then calls `setTimeout` with a callback that calls `backdrop.remove()` and a delay of 220 milliseconds, which is 20 ms longer than the transition duration to allow it to finish before the node is removed from the DOM. The function is synchronous, has no branches or error paths, and returns nothing. The `remove()` call is fire-and-forget inside the timeout callback.

### _saveIntroMusic

`_saveIntroMusic` is async and accepts three parameters: `trackId` (a string), `file` (a `File` or `null`), and `existingUrl` (a string, null, or undefined). It declares a local variable `uploadedUrl` of type `string | undefined`, initially undefined. If `trackId` equals `'custom'`, it enters a conditional block. If `file` is truthy, it calls `getSupabaseClient()` and throws `'Not connected'` if the result is falsy; it calls `getCurrentProfile()` and throws `'Not signed in'` if the result is falsy. It derives a storage path as `${profile.id}/intro.${ext}` where `ext` is taken from the file's name extension, defaulting to `'mp3'`. It then awaits `client.storage.from('intro-music').upload(path, file, { upsert: true, contentType: file.type })`; if the upload returns an error, it throws that error's message. It next awaits `client.storage.from('intro-music').createSignedUrl(path, 31536000)` (one year in seconds); if that returns an error or no `signedUrl`, it throws `'Could not get URL'`. Otherwise it assigns the signed URL to `uploadedUrl`. If `file` is falsy but `existingUrl` is truthy, it assigns `existingUrl` to `uploadedUrl`. If both are falsy, it throws `'No file selected'`. If `trackId` is not `'custom'`, the entire conditional block is skipped and `uploadedUrl` remains undefined.

After the conditional block, the function awaits `safeRpc('save_intro_music', { p_track_id: trackId, p_custom_url: uploadedUrl ?? null })`. If the RPC call returns an error, it throws that error's message. It then casts `data` to `{ error?: string } | null` and throws `result.error` if that field is present. On the success path, it calls `getCurrentProfile()` again and, if a profile is returned, directly mutates its `intro_music_id` and `custom_intro_url` fields on the cached object (cast to `Record<string, unknown>`) to reflect the newly saved values. The function returns `void` on success and propagates thrown errors to the caller.
