# Stage 2 Outputs — async.render.takes.ts

## Agent 01

### _setWireTakes
Reads its single parameter `fn` of type `WireFn` (a function taking `HTMLElement` and returning `void`). Writes to the module-level variable `_wireTakes`, assigning `fn` to it. Returns `void`. Makes no other calls, has no branches, no loops, no error paths, and is synchronous. No DOM, storage, network, or browser API access.

### loadHotTakes
Synchronous function reading its `category` parameter (defaulting to `'all'`), the module-level `state` object (specifically `state.currentFilter`, `state.wiredContainers`, and `state.hotTakes`), and the DOM via `document.getElementById('hot-takes-feed')`. It also reads the module-level `_wireTakes` callback and calls `getCurrentUser()` and `getCurrentProfile()` imported from `./auth.ts` for auth/profile lookups. It writes `state.currentFilter = category`, may add the container element to the `state.wiredContainers` Set, and writes to the container's `innerHTML` (either an empty-state block or the joined rendered cards). Returns `void`.

Control flow: assigns `state.currentFilter`, looks up the container, and returns early if the container is missing. If the container is not already in `state.wiredContainers`, it invokes `_wireTakes?.(container)` (optional-chained — no-op when the callback is unset) and adds the container to the Set. It then branches on `category`: when `'all'`, it uses `state.hotTakes` directly; otherwise it filters by `t.section === category`. If the resulting `takes` array is empty, it writes the "No takes here yet" empty-state markup to `container.innerHTML` and returns. Otherwise it maps each take through `_renderTake(t)` into a `rendered` string array, reads `user` and `profile`, and — if `profile?.is_moderator` is falsy and `rendered.length >= 2` — calls `_renderModeratorCard(!user)` and splices the result into index 2 of `rendered`. Finally it joins `rendered` with the empty string and assigns it to `container.innerHTML`. No `try/catch`, no `await`, no network or storage access here; all calls are synchronous.

### _renderTake
Synchronous pure-ish function reading its `t: HotTake` parameter (fields `user_id`, `user`, `text`, `id`, `username`, `verified_gladiator`, `time`, `userReacted`, `reactions`, `challenges`) and calling `getCurrentUser()` from `./auth.ts` once to compare `t.user_id` against the current user's id. It reads no module state directly and does not touch the DOM, storage, or network. It returns a single template-literal HTML string (the card markup); it writes nothing else.

Control flow is linear with no branches beyond inline ternaries: it computes `userClickable` (truthy when `t.user_id` exists and differs from the current user's id), then escapes `user`, the first character of `user` (falling back to `'?'` if empty, then `?? ''` for the character), `text`, `id`, `user_id`, and `username ?? ''` via the local `esc` alias for `escapeHTML`. It builds `profileAttr` as either a clickable data-attribute block or an empty string based on `userClickable`. Inside the returned template it calls `vgBadge(t.verified_gladiator)` and `bountyDot(t.user_id)` to inline their returned markup, casts `t.reactions` and `t.challenges` through `Number()` before interpolation, and uses ternaries on `t.text.length > 150` to toggle line-clamp styling and emit an optional "tap to read more" row, and on `t.userReacted` to pick the react-button background/border/color. No loops, no early returns, no `try/catch`, no awaits.

### _renderModeratorCard
Synchronous function reading its single optional `isGuest` parameter (default `false`). It reads no module state, no DOM, no storage, no network, and no browser APIs. It makes no function calls. It writes nothing outside its return value — a single template-literal HTML string containing the "MODERATORS WANTED" card markup.

Control flow: two parallel ternaries compute `btnLabel` (`'SIGN UP TO MODERATE'` when `isGuest`, else `'BECOME A MODERATOR'`) and `btnAction` (`'mod-signup'` vs `'become-mod'`), both interpolated into the returned string. No branches beyond those ternaries, no loops, no early returns, no error paths, no awaits. It is unclear from this file whether the `data-action` values `mod-signup` and `become-mod` are bound by the wiring installed via `_setWireTakes` or elsewhere.

### loadHotTakes

### _renderTake
Reads the single `HotTake` parameter `t` (fields `user_id`, `user`, `text`, `id`, `username`, `time`, `verified_gladiator`, `reactions`, `challenges`, `userReacted`) and reads the current user's id via `getCurrentUser()?.id` to compute `userClickable` (true when `t.user_id` exists and differs from the current user's id). Writes nothing to module state or the DOM; the return value is a single HTML template-literal string representing one `.hot-take-card` element. Synchronous.

Control flow is straight-line with no branches, loops, or try/catch. It calls the module-local `esc` alias (which is `escapeHTML` from `./config.ts`) on `t.user`, the first character of `t.user` (falling back to `'?'` when empty, with a further `?? ''` guard), `t.text` (twice — once for body, once for the share button's `data-text` attribute), `t.id`, `t.user_id`, `t.username ?? ''`, and `t.time`. It also calls `vgBadge(t.verified_gladiator)` from `./badge.ts` and `bountyDot(t.user_id)` from `./bounties.ts`, inlining both returned HTML fragments next to the username. The `profileAttr` variable is either a `data-action="profile"` attribute block with the escaped user id and username plus a cursor style, or an empty string when `userClickable` is false. Inline ternaries on `userClickable` also toggle `cursor:pointer` on the avatar and username. A `t.text.length > 150` check conditionally applies a 3-line webkit line-clamp style and appends a separate "tap to read more" div with `data-action="expand"`. The react button's background, border color, and text color are switched on `t.userReacted`. Numeric values `t.reactions` and `t.challenges` are cast through `Number()` before interpolation (the codebase's required pattern for numeric innerHTML). No async, no error handling.

### _renderModeratorCard
Reads the single boolean parameter `isGuest` (defaulting to `false`). Writes nothing; returns an HTML string for a cyan-accented `.hot-take-card` element promoting moderator recruitment. Synchronous, no external calls, no I/O, no module state access.

Control flow is two parallel ternaries with no branches or loops: `btnLabel` becomes `'SIGN UP TO MODERATE'` when `isGuest` is true else `'BECOME A MODERATOR'`, and `btnAction` becomes `'mod-signup'` when `isGuest` is true else `'become-mod'`. Neither value passes through `escapeHTML`; they are hardcoded literals chosen by the ternaries, so no user-supplied data reaches the returned markup. No try/catch, no error paths.
