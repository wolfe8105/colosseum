# Stage 2 Outputs — arena-feed-machine-pause.ts

## Agent 01

### pauseFeed
Reads the module-level `feedPaused` flag and returns immediately if it is already truthy. Otherwise writes `true` into `feedPaused` via `set_feedPaused`, then copies the current `timeLeft` (imported from `arena-feed-state`) into `feedPauseTimeLeft` via `set_feedPauseTimeLeft`. Calls `setDebaterInputEnabled(false)` from the feed UI module. Queries the DOM for `#feed-finish-turn`, cast to `HTMLButtonElement | null`, and if present sets its `disabled` to `true`. Invokes `updateCiteButtonState()` and `updateChallengeButtonState()`. Calls `updateTurnLabel` with the string `"⚔️ CHALLENGE IN PROGRESS"` (the two characters encoded as `\u2694\uFE0F`). Then branches on `debate.modView`: if falsy, the function ends; if truthy, it calls `showChallengeRulingPanel(debate)` (described in its own entry) and sets up a countdown. The countdown initializes a local `countdown` variable to `FEED_CHALLENGE_RULING_SEC`, reads `document.getElementById('feed-timer')` into `timerEl`, and if `challengeRulingTimer` is already set clears it with `clearInterval`. It then calls `setInterval` with a 1000 ms tick and stores the returned handle via `set_challengeRulingTimer`.

Each tick decrements `countdown`, and if `timerEl` exists writes `` `⚔️ ${countdown}s` `` into its `textContent`. When `countdown` reaches zero or below, the tick clears `challengeRulingTimer` (if still set) via `clearInterval`, writes `null` to it through `set_challengeRulingTimer`, reads `activeChallengeRefId` into a local `refId`, and calls `safeRpc('insert_feed_event', ...)` with `p_debate_id: debate.id`, `p_event_type: 'mod_ruling'`, `p_round: round`, `p_side: 'mod'`, `p_content: 'Auto-accepted (moderator timeout)'`, `p_reference_id: refId`, and `p_metadata: { ruling: 'upheld' }`. The returned promise is chained: the first `.then` checks `activeChallengeId` and, if truthy, returns `safeRpc('rule_on_reference', { p_challenge_id: activeChallengeId, p_ruling: 'upheld' })`; if falsy, the chain produces `undefined`. The next `.then` calls `unpauseFeed()` (its own entry). A `.catch` receives any error `e` and logs `'[Arena] Auto-accept ruling failed:'` plus `e` through `console.warn`. The outer `pauseFeed` function itself is not `async` and returns `void`; the promise chain fires inside the interval tick without being awaited by the caller.

### unpauseFeed
Reads `feedPaused` and returns immediately if it is falsy (the comment labels this idempotent, safe under both local and Realtime paths). Otherwise writes `false` to `feedPaused` via `set_feedPaused`. If `challengeRulingTimer` is truthy, calls `clearInterval` on it, then writes `null` into `challengeRulingTimer`, `activeChallengeRefId`, and `activeChallengeId` via their setters. Queries the DOM for `#feed-challenge-overlay` and calls `.remove()` on the result using optional chaining (no-op if the element is absent). Writes `feedPauseTimeLeft` into `timeLeft` via `set_timeLeft`, then writes `0` into `feedPauseTimeLeft` via `set_feedPauseTimeLeft`, then calls `updateTimerDisplay()`. Reads `currentDebate` into a local `debate`. If `debate` is truthy and `debate.modView` is falsy, it computes `isMyTurn` as true when either `phase === 'speaker_a'` and `debate.role === 'a'`, or `phase === 'speaker_b'` and `debate.role === 'b'`. It calls `setDebaterInputEnabled(isMyTurn)`, queries `#feed-finish-turn` (cast to `HTMLButtonElement | null`), and if present sets its `disabled` to `!isMyTurn`. It then calls `updateCiteButtonState()` and `updateChallengeButtonState()`. Finally, regardless of the `debate`/`modView` branch, calls `updateTurnLabel` with `"Side A's turn"` when `phase === 'speaker_a'`, `"Side B's turn"` when `phase === 'speaker_b'`, and the empty string otherwise. The function is synchronous and returns `void`.

### showChallengeRulingPanel
Queries `#feed-challenge-overlay` and calls `.remove()` on it via optional chaining (no-op if absent). Creates a `div` through `document.createElement`, sets its `className` to `'feed-challenge-overlay'` and its `id` to `'feed-challenge-overlay'`, and assigns an `innerHTML` literal containing a `.feed-ruling-panel` wrapper with a title (`"⚖️ RULING NEEDED"`), a subline, a `<textarea id="feed-ruling-reason">` with `maxlength="500"` and `rows="2"`, and two buttons `#feed-ruling-accept` (`"✅ ACCEPT (Upheld)"`) and `#feed-ruling-reject` (`"❌ REJECT"`). The overlay is appended to `document.body`. A local `busy` flag is initialized to `false`. An async arrow `handleRuling` is defined taking a `ruling` argument of `'upheld' | 'rejected'`: if `busy` is true it returns immediately, otherwise it sets `busy = true`, reads `#feed-ruling-reason`'s `.value`, trims it, and falls back to `''` if any link in the chain is nullish (the textarea is queried each call via `getElementById` with optional chaining on `.value`). It computes `label` as `'UPHELD'` or `'REJECTED'` based on `ruling`.

Inside a `try`, it awaits `safeRpc('insert_feed_event', { p_debate_id: debate.id, p_event_type: 'mod_ruling', p_round: round, p_side: 'mod', p_content: `${label}${reason ? ': ' + reason : ''}`, p_reference_id: activeChallengeRefId, p_metadata: { ruling } })`. If `activeChallengeId` is truthy, it calls `safeRpc('rule_on_reference', { p_challenge_id: activeChallengeId, p_ruling: ruling })` without awaiting; a `.catch` on that call logs `'[Arena] rule_on_reference failed:'` and the error through `console.warn`. If the awaited `insert_feed_event` call throws, the `catch` branch calls `showToast('Ruling failed: ' + (e instanceof Error ? e.message : 'unknown'), 'error')`, sets `busy = false`, and returns before removing the overlay. On the success path (after the try block), `overlay.remove()` is called; a trailing comment notes that unpause arrives via the Realtime `mod_ruling` event rather than being invoked here. Two click listeners are then attached: `overlay.querySelector('#feed-ruling-accept')` gets a listener that invokes `handleRuling('upheld')` through `void`, and `overlay.querySelector('#feed-ruling-reject')` gets one invoking `handleRuling('rejected')` through `void`; both use optional chaining on `querySelector`. The outer function itself is synchronous and returns `void`; only the inner `handleRuling` is `async`.

The function is not declared `async` and returns `void`; there is no awaiting. If `debate.modView` is falsy, the function returns after the turn label update.

When `debate.modView` is truthy, it calls `showChallengeRulingPanel(debate)` — see that entry for the panel's runtime behavior. It then initializes a local `countdown` variable to `FEED_CHALLENGE_RULING_SEC` imported from `arena-types-feed-room.ts`, grabs the DOM element with id `feed-timer`, and if the module-level `challengeRulingTimer` is truthy it calls `clearInterval` on it. It then calls `set_challengeRulingTimer` with the return of `setInterval(..., 1000)`. Inside the interval callback, each tick decrements `countdown`, writes `"⚔️ {countdown}s"` into `timerEl.textContent` if the element existed, and when `countdown <= 0` it re-clears `challengeRulingTimer`, calls `set_challengeRulingTimer(null)`, captures the module-level `activeChallengeRefId` into a local `refId`, and fires a promise chain starting with `safeRpc('insert_feed_event', {...})` carrying `p_debate_id: debate.id`, `p_event_type: 'mod_ruling'`, `p_round: round` (the imported module-level value), `p_side: 'mod'`, `p_content: 'Auto-accepted (moderator timeout)'`, `p_reference_id: refId`, and `p_metadata: { ruling: 'upheld' }`. The `.then` branch reads `activeChallengeId` and, if truthy, returns `safeRpc('rule_on_reference', { p_challenge_id: activeChallengeId, p_ruling: 'upheld' })`; if falsy it returns `undefined`, skipping the second RPC. The next `.then` calls `unpauseFeed()` — see that entry. A trailing `.catch` logs the error via `console.warn('[Arena] Auto-accept ruling failed:', e)` and swallows it. The promise chain is fire-and-forget; pauseFeed does not await it.

### unpauseFeed
This function takes no parameters and reads the module-level `feedPaused` flag; if it is false, it returns immediately, making the call idempotent. Otherwise it calls `set_feedPaused(false)`. It reads `challengeRulingTimer`, calls `clearInterval` on it if truthy, then calls `set_challengeRulingTimer(null)`, `set_activeChallengeRefId(null)`, and `set_activeChallengeId(null)`. It queries the DOM for `feed-challenge-overlay` and calls `.remove()` on it via optional chaining, which is a no-op if absent. It reads the module-level `feedPauseTimeLeft`, passes it to `set_timeLeft` (writing the state in `arena-feed-state.ts`), then calls `set_feedPauseTimeLeft(0)`, and calls `updateTimerDisplay()`.

It then reads the module-level `currentDebate` into a local `debate`. If `debate` is truthy and `debate.modView` is falsy, it computes `isMyTurn` as true when `phase === 'speaker_a' && debate.role === 'a'` or `phase === 'speaker_b' && debate.role === 'b'`, reading the imported `phase` value. It calls `setDebaterInputEnabled(isMyTurn)`, queries the DOM for `feed-finish-turn`, casts to `HTMLButtonElement | null`, and if found sets `disabled = !isMyTurn`. It calls `updateCiteButtonState()` and `updateChallengeButtonState()`. Regardless of the modView branch (this final call runs whether `debate` was null, modView was true, or not), it calls `updateTurnLabel` with `"Side A's turn"` when `phase === 'speaker_a'`, `"Side B's turn"` when `phase === 'speaker_b'`, and the empty string otherwise. The function is not async and returns `void`.

### showChallengeRulingPanel
This function takes a `debate: CurrentDebate` parameter. It queries the DOM for `feed-challenge-overlay` and calls `.remove()` on it via optional chaining, removing any prior overlay. It creates a new `div` via `document.createElement('div')`, sets its `className` to `'feed-challenge-overlay'`, its `id` to `'feed-challenge-overlay'`, and assigns an `innerHTML` string containing a `.feed-ruling-panel` with a title, a subtitle, a `<textarea id="feed-ruling-reason">` with `maxlength="500"` and `rows="2"`, and two buttons `#feed-ruling-accept` and `#feed-ruling-reject`. Note that neither `debate` nor any other value is interpolated into the innerHTML — the template is static. It appends the overlay to `document.body`.

It declares a local `busy` flag initialized to `false` and defines an async arrow `handleRuling(ruling: 'upheld' | 'rejected')`. Inside `handleRuling`, if `busy` is true it returns immediately; otherwise it sets `busy = true`, reads the textarea with id `feed-ruling-reason`, extracts `.value?.trim()` and falls back to an empty string if any step is null/undefined, and sets a local `label` to `'UPHELD'` for `'upheld'` and `'REJECTED'` for `'rejected'`. In a try block it awaits `safeRpc('insert_feed_event', {...})` with `p_debate_id: debate.id`, `p_event_type: 'mod_ruling'`, `p_round: round` (module-level), `p_side: 'mod'`, `p_content: "{label}" + (reason ? ': ' + reason : '')`, `p_reference_id: activeChallengeRefId` (module-level), and `p_metadata: { ruling }`. After the await, if `activeChallengeId` is truthy it calls `safeRpc('rule_on_reference', { p_challenge_id: activeChallengeId, p_ruling: ruling })` without awaiting and attaches a `.catch` that logs `'[Arena] rule_on_reference failed:'` via `console.warn`. The catch block of the outer try calls `showToast('Ruling failed: ' + (e instanceof Error ? e.message : 'unknown'), 'error')`, resets `busy = false`, and returns — leaving the overlay in the DOM for retry. On the success path (after any fire-and-forget rule_on_reference is dispatched) it calls `overlay.remove()`; per the inline comment, unpause is not triggered here and is left to a Realtime-received mod_ruling event. Finally, it queries the overlay for `#feed-ruling-accept` and `#feed-ruling-reject` and attaches `click` listeners that invoke `handleRuling('upheld')` and `handleRuling('rejected')` respectively via `void` to discard the returned promise. The function itself is synchronous and returns `void`; only the inner `handleRuling` is async.
