# Stage 2 Outputs — powerups.loadout.ts

## Agent 01

### renderLoadout
Computes `maxSlots` by calling `getPowerUpSlots(questionsAnswered || 0)` and `tier` via `getTier(questionsAnswered || 0)`; the `_debateId` parameter is unused. On the locked branch where `maxSlots === 0`, it calls `getNextTier(questionsAnswered || 0)`, derives `remaining` from `next.questionsNeeded` or `0` when `next` is null, and returns an inline-styled locked card that interpolates `remaining` into the prompt and exits early. Hardcoded hex literals `#1a1a2e`, `#2a2a3e`, `#0f0f1a` are emitted directly (landmine LM-PU-002); the `remaining` number is interpolated into HTML without `Number()` casting or `escapeHTML`.

On the unlocked path it builds a local `equippedMap: Record<number, EquippedItem>` keyed by `slot_number`, using optional-chained `equipped?.length` before `forEach`. It loops `i` from `1` through `maxSlots`, pushing one of two HTML fragments into a local `slots` array: a filled slot reading `eq.icon ?? cat?.icon ?? '?'` and `eq.name ?? cat?.name ?? ''` through `escapeHTML` (with `cat` looked up in `CATALOG` by `eq.power_up_id`), or an empty slot carrying `data-slot="${i}"`. It then builds `invItems` by filtering `inventory || []` for `item.quantity > 0` and mapping each through `escapeHTML` on icon/name while interpolating `item.power_up_id` and `item.quantity` raw into the markup. Finally it returns a template literal joining the slot row, a hidden `#powerup-inventory-picker` that either lists `invItems` or shows a "Buy some" link, and a hidden `#powerup-equip-error` div; it interpolates `tier.icon`, `tier.name`, and `maxSlots` into the header without escaping. No try/catch, no awaits, synchronous return.

### wireLoadout
Declares a closure-scoped `selectedSlot: number | null` initialized to `null`. It queries `document.querySelectorAll('.powerup-slot.empty')` globally (landmine LM-PU-003 — collides when multiple loadout panels coexist) and, for each, attaches a click listener that parses the clicked element's `dataset.slot` via `Number.parseInt(... ?? '0', 10)` into `selectedSlot`, looks up `#powerup-inventory-picker` and sets its `style.display` to `'block'` when present, then re-iterates every `.powerup-slot` to rewrite `borderColor`: the clicked slot gets `var(--mod-text-heading)`, other `.filled` slots get `var(--mod-text-heading)44`, and remaining slots get `#2a2a3e`. No early return when the picker is missing; the border repaint still runs.

It then queries `.powerup-inv-item` globally and binds an `async` click listener per item. The handler early-returns if `selectedSlot === null`; otherwise it reads `dataset.id ?? ''`, grabs `#powerup-equip-error`, sets the clicked item's `opacity` to `'0.5'` and `pointerEvents` to `'none'`, and awaits `equip(debateId, powerUpId, selectedSlot)` from `powerups.rpc.ts`. On `result.success` it hides `errorEl` when present and invokes `onEquipped(result)` if the optional callback was passed. On failure it restores `opacity` to `'1'` and `pointerEvents` to `'auto'`, and when `errorEl` exists writes `result.error ?? 'Equip failed'` into `textContent` and sets `display` to `'block'`. No try/catch — a thrown/rejected `equip` would leave the item in the disabled visual state; uncertain whether `equip` can reject (not inspected here). `selectedSlot` is not reset after a successful equip.

## Agent 02

### renderLoadout
Synchronous function accepting `inventory`, `equipped`, `questionsAnswered`, and an unused `_debateId`; produces an HTML string. Reads `questionsAnswered` (coerced via `|| 0`) through `getPowerUpSlots` into local `maxSlots` and through `getTier` into local `tier`. Branches on `maxSlots === 0`: on the locked path it re-derives progress by calling `getNextTier`, computes `remaining` from `next.questionsNeeded` when `next` is truthy otherwise `0`, and returns an early locked-state HTML template that embeds `remaining` directly without `escapeHTML` (numeric source, not user input, but unlabeled — uncertain whether `questionsNeeded` is always numeric). No loops, no try/catch on this path.

On the unlocked path it builds a local `equippedMap: Record<number, EquippedItem>`, guarded by `equipped?.length` then populated via `forEach` keying on `e.slot_number`. A `for` loop from `i = 1` to `maxSlots` pushes into a local `slots: string[]`: if `equippedMap[i]` exists it reads `eq.power_up_id` to index `CATALOG`, emitting a filled-slot template with `escapeHTML` on `eq.icon ?? cat?.icon ?? '?'` and `eq.name ?? cat?.name ?? ''`; otherwise it pushes an empty-slot template with a literal `+`. Then filters `inventory || []` to items with `quantity > 0`, mapping each to an inventory-item row indexed against `CATALOG` by `item.power_up_id`, with `escapeHTML` on icon and name and raw interpolation of `item.power_up_id` into `data-id` and `item.quantity` into the `x{quantity}` span. The final return concatenates `slots.join('')` and, conditionally on `invItems.length > 0`, either `invItems.join('')` or a fallback anchor offering `Buy some →`. The picker starts hidden (`display:none`) and an error slot (`#powerup-equip-error`) is emitted hidden as well. No awaits, no promises.

### wireLoadout
Synchronous function accepting `debateId` and optional `onEquipped` callback; returns `void`. Declares one closure-scoped mutable `selectedSlot: number | null` initialized to `null`. First pass queries the document globally for `.powerup-slot.empty` (the landmine LM-PU-003 flagged in the file header) and attaches a synchronous `click` listener per slot: the listener parses `dataset.slot` via `Number.parseInt` with fallback `'0'` and radix 10 into `selectedSlot`, reveals `#powerup-inventory-picker` by setting `display:block` if the element exists (no-op otherwise), then iterates all `.powerup-slot` nodes to rewrite `borderColor` — the clicked one gets `var(--mod-text-heading)`, other `.filled` ones get `var(--mod-text-heading)44`, and everything else gets the hardcoded `#2a2a3e`. No errors thrown, no await.

Second pass queries `.powerup-inv-item` globally and attaches an `async` click listener per item. Early return when `selectedSlot === null`. Reads `dataset.id` into `powerUpId` (fallback `''`, uncertain whether downstream `equip` rejects empty strings), grabs `#powerup-equip-error` into `errorEl` (may be null), then optimistically dims the clicked item (`opacity = '0.5'`) and disables it (`pointerEvents = 'none'`). Awaits `equip(debateId, powerUpId, selectedSlot)` — no try/catch, so a thrown rejection propagates out of the handler as an unhandled promise rejection and leaves the item dimmed and disabled. On `result.success` it hides `errorEl` if present and fires `onEquipped(result)` if the callback was supplied (fire-and-forget of a potentially async consumer, though the signature types it as sync). On failure it restores the item's `opacity` to `'1'` and `pointerEvents` to `'auto'`, then if `errorEl` exists writes `result.error ?? 'Equip failed'` into `textContent` and sets `display:block`. No listener cleanup / `destroy()` is exposed — uncertain whether re-invoking `wireLoadout` after a re-render stacks duplicate listeners on newly-rendered nodes (probably safe since old DOM nodes are discarded on re-render, but not verified here).
