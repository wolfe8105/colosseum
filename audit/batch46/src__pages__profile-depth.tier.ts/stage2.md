## Agent 01

### renderTierBannerUI

`renderTierBannerUI` is a synchronous, void-returning function that accepts a single numeric parameter `qa` (questions answered). It reads four module-level constants cast from `window` at module load time — `getTier`, `getNextTier`, `renderTierBadge`, and `renderTierProgress` — which are typed function references or `undefined`. It writes to the DOM only; it holds no module-level state.

The function begins with two consecutive early returns. First, it checks whether `getTier` is truthy; if the window global was absent at module load time, `getTier` is `undefined` and the function returns immediately without touching the DOM. Second, it queries the DOM for an element with id `tier-banner`; if that element does not exist, it returns immediately. If both guards pass, it calls `getTier(qa)` to obtain a `tier` object with fields `maxStake`, `slots`, and `name`, then conditionally calls `getNextTier(qa)` — guarded by a truthiness check on `getNextTier` itself — to obtain a `next` object with `questionsNeeded` and `name`, or `null`.

With those results, the function constructs a `perkText` string. If `tier.maxStake` is greater than zero, `perkText` contains an HTML span showing either the literal string `'Unlimited'` (when `maxStake === Infinity`) or the numeric value followed by `' tokens'`; if `tier.slots` is also greater than zero, a second span for power-up slots is appended. If `tier.maxStake` is not greater than zero (the zero case — no staking unlocked), `perkText` becomes a plain-text fallback asking the user to answer more questions, substituting `next.questionsNeeded` when `next` is non-null or the hardcoded string `'10'` otherwise.

The function then assigns a single HTML string to `banner.innerHTML`. The structure is: a `tier-header` div containing a `tier-rank-label` span and the output of `renderTierBadge(qa)` (called when `renderTierBadge` is truthy, omitted otherwise); an optional `tier-unlock-hint` div present only when `next` is non-null, whose content passes `next.questionsNeeded` and `next.name` through `escapeHTML` and `String()` conversion before injection; the output of `renderTierProgress(qa)` when `renderTierProgress` is truthy; and a `tier-perks` div containing `perkText`. Note that `perkText` is injected without `escapeHTML` — it contains hardcoded string literals and numeric values derived from `tier.maxStake` and `tier.slots` via JavaScript operators, not raw user content. Finally, `banner.style.display` is set to `'block'`. The function returns `void`.

### updateMilestoneBar

`updateMilestoneBar` is a synchronous, void-returning function that takes no parameters. It reads two external values: the module-level import `serverQuestionsAnswered` from `profile-depth.state.ts` (a numeric value reflecting the server-sourced count of questions the user has answered) and the module-level import `DEPTH_MILESTONES` from `profile-depth.data.ts` (an array of milestone objects, each with at minimum `threshold`, `name`, `desc`, and `icon` fields). It writes to the DOM only and holds no module-level state.

The function queries the DOM for an element with id `milestone-bar`. If that element does not exist, it returns immediately — this is the only early-return path. Otherwise, it sets a local constant `totalQ` to the hardcoded value `100`, reads `serverQuestionsAnswered` into `answered`, and computes `pct` as `Math.min(100, Math.round((answered / totalQ) * 100))`, clamping the percentage to a maximum of 100.

It then assigns a template-literal HTML string to `bar.innerHTML`. The structure contains: a `milestone-label` div with static text; a `milestone-track` div holding a `milestone-fill` div whose inline `width` style is set to `pct%`; and a mapped array of `milestone-pip` divs generated by iterating over `DEPTH_MILESTONES`. For each milestone `m`, the function computes `earned` as a boolean (`answered >= m.threshold`). Each pip div receives the CSS class `'earned'` when `earned` is true, has its `left` style set to `m.threshold%`, and has a `title` attribute constructed by passing `m.name` and `m.desc` individually through `escapeHTML`. Inside each pip, a `pip-icon` span shows `'✅'` when earned or `m.icon` when not — `m.icon` is injected without `escapeHTML`, and is a potential XSS surface if `DEPTH_MILESTONES` ever contains user-controlled icon content, though that array is an imported constant from a data file. A `pip-label` span shows `escapeHTML(m.name)`. The `.milestone-pct` div displays `answered`, `totalQ`, and `pct` as plain numeric values. The function returns `void`.
