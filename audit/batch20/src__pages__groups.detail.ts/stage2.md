# Stage 2 Outputs — src/pages/groups.detail.ts

## Agent 01

### openGroup
This async function receives a `groupId` string parameter and returns a Promise<void>. It first calls `setCurrentGroupId(groupId)` from `groups.state.ts` to write the module-level current group identifier. It then reads DOM elements by id and writes: `view-lobby` display to `'none'`, `view-detail` display to `'flex'`, `detail-name` textContent to `'Loading…'`, `detail-emoji` to `'⚔️'`, `detail-desc` to empty, `detail-members` and `detail-elo` to `'—'`, `detail-hot-takes`/`detail-challenges`/`detail-members-list` innerHTML to loading-state divs, and `gvg-challenge-btn` display to `'none'`. It then calls `switchDetailTab('hot-takes')` from `groups.nav.ts`.

It enters a try block awaiting `safeRpc('get_group_details', { p_group_id: groupId })`. If the response contains `error`, it throws; otherwise it reads `data`, parsing it via `JSON.parse` if it is a string, producing the local `g`. It writes DOM: `detail-top-name` textContent to `g.name.toUpperCase()`, `detail-emoji` to `g.avatar_emoji || '⚔️'`, `detail-name` to `g.name`, `detail-desc` to `g.description || ''`, `detail-members` to `g.member_count`, `detail-elo` to `g.elo_rating`. It then reads `detail-banner`; if present, calls `renderGroupBanner(bannerEl, g, g.my_role === 'leader' || g.my_role === 'co_leader')`. It reads `detail-fate`; if present, reads `g.shared_fate_pct ?? 0` into `pct` and writes its textContent to `'+'+pct+'%'` when `pct > 0` else `'—'`, and writes `fateEl.style.color` to `'var(--mod-accent)'` when `pct > 0` else empty string. It calls `setIsMember(g.is_member)` and `setCallerRole(g.my_role ?? null)`, then writes the exported module-level `currentGroupData` to `g`. It calls `updateJoinBtn(g)`, which updates the join button DOM based on the group data. It writes `gvg-challenge-btn` display to `'block'` if `g.is_member` else `'none'`. It reads `detail-gear-btn`; if present, writes its display to `'flex'` when `g.my_role === 'leader'` else `'none'`. It reads `detail-auditions-tab`; if present, writes its display to `'inline-block'` when `g.join_mode === 'audition'` else `'none'`. The catch clause (no binding) writes `detail-name` textContent to `'Error loading group'`. After the try/catch, it calls `loadGroupHotTakes(groupId)`, `loadGroupChallenges(groupId)`, and `loadGroupMembers(groupId)` in that order as fire-and-forget (their return values are not awaited). The only awaited call is `safeRpc`.

### updateJoinBtn
This synchronous function receives a `GroupDetail` argument `g` and returns void. It reads the DOM element with id `join-btn` cast to `HTMLButtonElement`. It reads the module-level `currentUser` from `groups.state.ts`: if falsy, it writes `btn.textContent` to `'SIGN IN TO JOIN'`, `btn.className` to `'join-btn join'`, `btn.style.display` to `'block'`, `btn.disabled` to `false`, then returns early.

If `g.is_member` is true, it writes `btn.textContent` to `'YOU OWN THIS GROUP'` when `g.my_role === 'leader'` otherwise `'LEAVE GROUP'`, writes `btn.className` to `'join-btn leave'`, writes `btn.disabled` to the boolean `g.my_role === 'leader'`, writes `btn.style.display` to `'block'`, then returns. Otherwise it reads `g.join_mode ?? 'open'` into `mode`. If `mode === 'invite_only'`, it writes `btn.style.display` to `'none'` and returns. Otherwise it writes `btn.style.display` to `'block'`, `btn.disabled` to `false`, `btn.className` to `'join-btn join'`, and `btn.textContent` to `'REQUEST AUDITION'` when `mode === 'audition'` else `'JOIN GROUP'`. It calls no other functions and returns no value.

### toggleMembership
This async function takes no arguments and returns Promise<void>. It reads module-level `currentUser` from `groups.state.ts`: if falsy, it writes `window.location.href` to `'moderator-plinko.html?returnTo=' + encodeURIComponent(window.location.pathname + '?group=' + currentGroupId)` (reading module-level `currentGroupId`) and returns early. Otherwise, it reads `join-btn` from the DOM cast to `HTMLButtonElement` and writes `btn.disabled = true`.

Inside a try block, it branches on the module-level `isMember`. If true, it awaits `safeRpc('leave_group', { p_group_id: currentGroupId })`; if the response has `error`, it throws. Otherwise it calls `setIsMember(false)` and `setCallerRole(null)`, writes `btn.textContent` to `'JOIN GROUP'`, `btn.className` to `'join-btn join'`, writes `gvg-challenge-btn` display to `'none'`, reads the `detail-members` element into `mc`, reads its textContent, parses it via `Number.parseInt`, subtracts 1, and writes the stringified result back to `mc.textContent`. If `isMember` is false, it awaits `safeRpc('join_group', { p_group_id: currentGroupId })`; on error it throws. Otherwise it calls `setIsMember(true)` and `setCallerRole('member')`, writes `btn.textContent` to `'LEAVE GROUP'`, `btn.className` to `'join-btn leave'`, writes `gvg-challenge-btn` display to `'block'`, reads `detail-members` into `mc`, increments its parsed value by 1, and writes the stringified result back to `mc.textContent`. After the branch, it calls `loadGroupMembers(currentGroupId!)` fire-and-forget (non-null assertion on `currentGroupId`). The catch block binds `e` and calls the browser `alert()` with `(e as Error).message || 'Something went wrong'`. The finally block writes `btn.disabled = false`. The only awaited call per branch is the `safeRpc` invocation.

### openGroup
This async function accepts `groupId: string` and returns `Promise<void>`. It first writes to module/external state by calling `setCurrentGroupId(groupId)` (from `groups.state.ts`). It then reads and mutates DOM: hides `#view-lobby` (`display: none`), shows `#view-detail` (`display: flex`), and overwrites `textContent`/`innerHTML` on `#detail-name`, `#detail-emoji`, `#detail-desc`, `#detail-members`, `#detail-elo`, `#detail-hot-takes`, `#detail-challenges`, `#detail-members-list` with placeholder loading values. It hides `#gvg-challenge-btn` and calls `switchDetailTab('hot-takes')` to activate the hot-takes tab.

Inside a `try` block, it awaits `safeRpc('get_group_details', { p_group_id: groupId })`. If the destructured `error` is truthy, it throws, jumping to `catch`. Otherwise it normalizes `data` (parsing JSON if it is a string) into `g`. It then writes `g.name.toUpperCase()`, `g.avatar_emoji` (falling back to a crossed-swords emoji), `g.name`, `g.description` (fallback empty string), `g.member_count`, and `g.elo_rating` into the corresponding detail DOM nodes. If `#detail-banner` exists, it calls `renderGroupBanner(bannerEl, g, leaderFlag)` where `leaderFlag` is true only when `g.my_role` is `'leader'` or `'co_leader'`. If `#detail-fate` exists, it reads `g.shared_fate_pct` (defaulting to 0 via `??`) into `pct`; when `pct > 0` it writes `+{pct}%` and sets `color` to `var(--mod-accent)`, otherwise writes an em-dash and clears the color. It then calls `setIsMember(g.is_member)`, `setCallerRole(g.my_role ?? null)`, assigns `g` to the exported module-level `currentGroupData`, calls `updateJoinBtn(g)` to recompute the join button, and toggles `#gvg-challenge-btn` display to `'block'` or `'none'` based on `g.is_member`. If `#detail-gear-btn` exists, it sets its display to `'flex'` when `my_role === 'leader'`, else `'none'`. If `#detail-auditions-tab` exists, it sets display to `'inline-block'` when `g.join_mode === 'audition'`, else `'none'`. After the try/catch, it fires three fetches without awaiting: `loadGroupHotTakes(groupId)`, `loadGroupChallenges(groupId)`, and `loadGroupMembers(groupId)`.

The `finally` block writes `btn.disabled = false` regardless of outcome.
