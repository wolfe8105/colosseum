# Stage 2 Outputs — src/share.ts

## Agent 01

### getBaseUrl
Synchronous. Reads `APP.baseUrl` imported from `./config.ts`. If that value is truthy, returns it; otherwise reads and returns `window.location.origin`. Writes no state, calls no other functions on the anchor list, and performs no I/O. Returns a string.

### getStableInviteUrl
Async. Reads the module-level variable `_cachedInviteUrl`; if it is non-null, returns that cached string immediately without any further work. Otherwise enters a `try` block that awaits a dynamic `import('./auth.ts')` to obtain `safeRpc`, then awaits `safeRpc('get_my_invite_link', {})`. It narrows `result.data` to an object that may contain `url` and `ref_code`; if `data?.url` is truthy, it writes that string to `_cachedInviteUrl` and returns it. If the dynamic import or the RPC throws, the `catch` block swallows the error (comment "fall through") and control proceeds past the try. If the try block completes without returning (either because the RPC threw, `data` was null, or `data.url` was falsy), the function falls through to a `return` that composes a fallback URL by calling `getBaseUrl()` and appending `/moderator-plinko.html`. Returns a `Promise<string>`. The dynamic import and the RPC call are both awaited.

### share
Async. Reads its `title`, `text`, and `url` destructured parameters and the `navigator.share`, `navigator.clipboard`, and `document.execCommand` browser APIs. First branch: if `navigator.share` exists, it enters a `try` and awaits `navigator.share({ title, text, url })`, returning on success. If that call rejects, the `catch` inspects `e`; when `e` is an `Error` instance with `name === 'AbortError'` (user cancelled the share sheet) it returns without further action. Any other error falls through out of the catch.

After the first branch (either because `navigator.share` was absent or because a non-abort error was caught), a second `try` awaits `navigator.clipboard.writeText(text)` and on success calls `showToast('Copied to clipboard!')` (imported from `./config.ts`), then returns. If `writeText` rejects, the `catch` creates a `<textarea>` via `document.createElement`, assigns `text` to its `value`, sets `style.cssText` to position it off-screen at `left:-9999px`, appends it to `document.body`, calls `.select()`, invokes `document.execCommand('copy')`, removes the element with `.remove()`, and calls `showToast('Copied to clipboard!')`. Returns `Promise<void>`; writes nothing to module-level state.

### shareResult
Synchronous (returns `void`). Reads the destructured `debateId`, `topic`, `winner`, `winnerElo`, `loser`, `loserElo`, and `spectators` parameters. Calls `getBaseUrl()` and interpolates its result with `encodeURIComponent(debateId ?? 'demo')` into a `/debate/...` URL. Builds a `text` string via template literal using `??` defaults: `'Winner'`, `1200` (both Elos), `'Loser'`, `'Debate'`, and `0` spectators. Calls `share({ title: 'Debate Result — The Moderator', text, url })` with its return value discarded via `void`, so the promise is fire-and-forget; any rejection inside `share` is already caught internally. Writes no module-level state; returns `undefined`.

### shareProfile
Synchronous (returns `void`). Reads the destructured `userId`, `username`, `displayName`, `elo`, `wins`, `losses`, and `streak` parameters. Calls `getBaseUrl()` and appends `/u/` plus `encodeURIComponent(username ?? userId ?? 'debater')` — if both `username` and `userId` are nullish the URL contains the literal `'debater'`. Computes `name` as `displayName ?? username ?? 'Debater'` and builds a template-literal `text` substituting `??` defaults of `1200`, `0`, `0`, `0` for elo/wins/losses/streak. Invokes `share({ title: \`${name} — The Moderator\`, text, url })` as a fire-and-forget call prefixed with `void`. No writes to module state; returns `undefined`.

### inviteFriend
Synchronous (returns `void`) but schedules async work. Calls `getStableInviteUrl()`, then chains `.then(url => ...)` on the returned promise; the whole expression is prefixed with `void` to discard the promise. Inside the `.then` callback it builds a `text` template literal containing the fixed marketing copy plus the resolved `url`, then calls `share({ title: 'Join The Moderator', text, url })` also prefixed with `void`. There is no `.catch`: any rejection from `getStableInviteUrl` is unhandled at this site, though `getStableInviteUrl` itself catches internally and returns a fallback, so in practice it does not reject. Reads nothing beyond what the two callees read; writes nothing to module state.

### shareTake
Synchronous (returns `void`). Reads its `takeId` and `takeText` string parameters. Calls `getBaseUrl()` and appends `/take/` plus `encodeURIComponent(takeId)`. Applies `decodeURIComponent(takeText)` to produce `decoded` — if `takeText` contains malformed `%`-escape sequences `decodeURIComponent` throws a `URIError`, and the function does not catch it, so the exception propagates to the caller. Builds a `text` template with the fixed prefix, the quoted `decoded` value, and a trailing line containing `url`. Invokes `share({ title: 'Hot Take — The Moderator', text, url })` fire-and-forget via `void`. Writes no module-level state; returns `undefined`.

### handleDeepLink
Synchronous (returns `void`). Constructs `new URLSearchParams(window.location.search)` and extracts three entries: `ref`, `debate`, and `challenge` (from key `'from'`).

First block: if `ref` is truthy and matches the regex `/^[a-z0-9]{5}$/` (exactly five lowercase alphanumeric characters), it writes `ref` to `localStorage` under key `'colosseum_referrer'` via `localStorage.setItem`. It then calls `getCurrentUser()` imported from `./auth.ts`; if that returns a truthy user, it starts a dynamic `import('./auth.ts')` and chains `.then(({ safeRpc }) => ...)`, prefixed with `void`. Inside that callback it reads `localStorage.getItem('mod_visitor_id')` and coalesces `null` to `undefined` as `deviceId`, then invokes `safeRpc('attribute_signup', { p_ref_code: ref, ...(deviceId ? { p_device_id: deviceId } : {}) })` — spreading the `p_device_id` field only when `deviceId` is truthy — with its promise discarded via `void`. No `.catch` is attached on either the import or the RPC; rejections are unhandled at this site. If `ref` fails the regex or is falsy, nothing in this block runs.

Second block: if `debate` is truthy, it schedules a `setTimeout` with a 500 ms delay whose callback calls `navigateTo('arena')` imported from `./navigation.ts`. The `setTimeout` return value is discarded. Third block: if `challenge` is truthy, it computes `safeName` by stringifying `challenge`, stripping any character outside `[a-zA-Z0-9_]`, and slicing to the first 30 characters. It reads `topic` from the params (defaulting to `'Open challenge'`), applies `decodeURIComponent` to it, and slices the result to 100 characters as `safeTopic` — if `decodeURIComponent` throws, the exception propagates. It then schedules a `setTimeout` with a 1000 ms delay whose callback calls `showToast` with an interpolated string containing `safeName` and `safeTopic`. The three `if` blocks are independent — multiple can execute in one call. The function writes to `localStorage` and schedules timers; it does not return a value.

First branch: if `ref` is truthy and matches `/^[a-z0-9]{5}$/`, writes `ref` to `localStorage` under key `'colosseum_referrer'`, then reads the current user via `getCurrentUser()`; if that user is truthy, performs a dynamic `import('./auth.ts')` whose promise is discarded via `void`, and inside `.then(({ safeRpc }) => …)` reads `localStorage.getItem('mod_visitor_id')` (coerced via `?? undefined`) and calls `safeRpc('attribute_signup', { p_ref_code: ref, ...(deviceId ? { p_device_id: deviceId } : {}) })` with that inner promise also discarded via `void`. No `.catch` is attached on either the import or the RPC; rejections are unhandled at this site. If `ref` fails the regex or is falsy, nothing in this block runs.

Second branch: if `debate` is truthy, schedules a `setTimeout` with 500ms delay whose callback calls `navigateTo('arena')`. The timer id is not captured or stored.

Third branch: if `challenge` is truthy, computes `safeName` as `String(challenge).replace(/[^a-zA-Z0-9_]/g, '').slice(0, 30)`, reads `topic` from the params (defaulting to `'Open challenge'`), and computes `safeTopic` as `String(decodeURIComponent(topic)).slice(0, 100)`. If `topic` is malformed, `decodeURIComponent` throws `URIError`, uncaught. Schedules `setTimeout` with a 1000ms delay that calls `showToast(\`⚔️ Challenge from ${safeName}: "${safeTopic}"\`)`. The three branches are independent `if` statements, not `else if`, so multiple can fire on one call. Returns `void`.

## Agent 05

### getBaseUrl
Reads `APP.baseUrl` (imported from `./config.ts`) and, if falsy, reads `window.location.origin`. Writes nothing to module-level state or external state. Returns the first truthy value of the two. Synchronous; no other function calls beyond property access. No branches other than the `||` short-circuit; no error paths.

### getStableInviteUrl
Async function. Reads the module-level variable `_cachedInviteUrl`; if it is truthy, returns it immediately without any further work. Otherwise enters a `try` block: dynamically imports `./auth.ts` (awaited), destructures `safeRpc`, awaits `safeRpc('get_my_invite_link', {})`, then casts `result.data` to a shape with optional `url` and `ref_code`. If `data?.url` is present, writes that value to `_cachedInviteUrl` and returns it. If any step in the `try` throws, the `catch` swallows the error silently and control falls through. On fall-through (cache miss, no `url` in data, or thrown error), calls `getBaseUrl()` and returns the string `` `${base}/moderator-plinko.html` ``. The `safeRpc` import and call are awaited; there are no fire-and-forget calls.

### share
Async helper function. Reads the `{ title, text, url }` parameter object. Reads `navigator.share` to branch: if it exists, enters a `try` that awaits `navigator.share({ title, text, url })` and returns on success; the `catch` inspects the thrown value, and if it is an `Error` instance with `name === 'AbortError'` it returns silently; any other error falls through to the clipboard path. The clipboard path enters a second `try` that awaits `navigator.clipboard.writeText(text)` and then calls `showToast('Copied to clipboard!')`. If clipboard writing throws, the `catch` creates a detached `<textarea>` DOM element, assigns `text` to its `value`, sets inline `cssText` to position it off-screen, appends it to `document.body`, calls `ta.select()`, invokes the deprecated `document.execCommand('copy')`, removes the element via `ta.remove()`, and calls `showToast('Copied to clipboard!')`. Writes to the DOM only on the textarea fallback path. Returns `Promise<void>`. No module-level state is written.

### shareResult
Synchronous exported function. Reads the destructured fields `debateId`, `topic`, `winner`, `winnerElo`, `loser`, `loserElo`, `spectators` from its params. Calls `getBaseUrl()` once and concatenates it with `/debate/` and `encodeURIComponent(debateId ?? 'demo')` to build `url`. Builds a multi-line `text` string with emoji prefixes using the `??` operator to supply defaults (`'Winner'`, `1200`, `'Loser'`, `1200`, `'Debate'`, `0`). Calls `share({ title: 'Debate Result — The Moderator', text, url })` and discards the returned promise via `void`. No branches beyond the nullish coalescing defaults; no try/catch; no module state writes; returns `void`.

### shareProfile
Synchronous exported function. Reads the destructured fields `userId`, `username`, `displayName`, `elo`, `wins`, `losses`, `streak`. Calls `getBaseUrl()` and concatenates it with `/u/` and `encodeURIComponent(username ?? userId ?? 'debater')` to form `url`. Derives `name` as `displayName ?? username ?? 'Debater'`. Builds a `text` string containing the name, ELO (default 1200), wins (0), losses (0), streak (0), and the URL. Calls `share({ title: \`${name} — The Moderator\`, text, url })` and discards its promise via `void`. No branches beyond `??` defaults; no error handling; no module-level writes; returns `void`.

### inviteFriend
Synchronous exported function (no `async` keyword, returns `void`). Calls `getStableInviteUrl()` and attaches a `.then(url => ...)` callback; the outer promise is discarded via `void`. Inside the `.then` callback, builds `text` as `` `Think you can hold your own? Join me on The Moderator.\n\n${url}` `` and calls `share({ title: 'Join The Moderator', text, url })`, discarding that promise with `void` as well. No `.catch` is attached to either promise, so any rejection from `getStableInviteUrl` (it has its own internal try/catch so this is unlikely) or from `share` will surface as an unhandled rejection. No parameters, no module-level writes, no DOM reads.

### shareTake
Synchronous exported function. Reads parameters `takeId` (string) and `takeText` (string). Calls `getBaseUrl()` and builds `url` as `` `${base}/take/${encodeURIComponent(takeId)}` ``. Computes `decoded` by calling `decodeURIComponent(takeText)` — if `takeText` contains malformed percent-encoding this call throws `URIError` and propagates out of the function, since there is no try/catch. Builds `text` with a fire emoji prefix, the decoded string in quotes, and the URL. Calls `share({ title: 'Hot Take — The Moderator', text, url })` and discards the promise via `void`. Returns `void`; writes nothing to module-level state.

### handleDeepLink
Synchronous exported function. Reads `window.location.search` and constructs a `URLSearchParams` from it. Reads three query params: `ref`, `debate`, and `from` (bound to `challenge`). First branch: if `ref` is truthy and matches the regex `/^[a-z0-9]{5}$/`, writes it to `localStorage` under key `colosseum_referrer`, then calls `getCurrentUser()` (imported from `./auth.ts`); if that returns a truthy user, dynamically imports `./auth.ts` and in the `.then` reads `localStorage.getItem('mod_visitor_id')` (coerced `null` to `undefined` via `??`), then calls `safeRpc('attribute_signup', { p_ref_code: ref, ...(deviceId ? { p_device_id: deviceId } : {}) })`. Both the dynamic import promise and the `safeRpc` promise are fire-and-forget (discarded via `void`); no `.catch` is attached. Second branch: if `debate` is truthy, schedules `navigateTo('arena')` via `setTimeout` at 500 ms. Third branch: if `challenge` is truthy, computes `safeName` by coercing to `String`, stripping anything outside `[a-zA-Z0-9_]`, and slicing to 30 chars; reads param `topic` defaulting to `'Open challenge'`, coerces and `decodeURIComponent`s it then slices to 100 chars as `safeTopic`; schedules `showToast(\`⚔️ Challenge from ${safeName}: "${safeTopic}"\`)` via `setTimeout` at 1000 ms. The `decodeURIComponent` call inside the challenge branch is not wrapped in try/catch and will throw `URIError` on malformed input. The three branches are independent (not `else if`), so all three can fire in one invocation. Returns `void`; writes to `localStorage` only on the `ref` branch; invoked automatically at module load via `ready.then(() => handleDeepLink())` on line 204.
