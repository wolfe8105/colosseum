## Agent 01

### rarityClass

`rarityClass` takes a single parameter `tier` of type `RarityTier`. It reads no module-level state and accesses no external state. Inside the function body, a `Record<RarityTier, string>` literal called `map` is constructed inline, mapping each of the five rarity tier values (`common`, `uncommon`, `rare`, `legendary`, `mythic`) to the string of the same name. The function returns `map[tier]` if the key is present, or the string `'common'` if the lookup returns `undefined` (via the `??` nullish coalescing operator). It is synchronous, has no branches beyond the fallback, no loops, and no error paths.

### openBottomSheet

`openBottomSheet` is a synchronous function that takes three parameters: `effect` (a `ModifierEffect` object), `state` (a `ShopState` object), and `onBuySuccess` (a callback with no arguments). It reads `state.productType` to decide whether this is a `'modifier'` or `'powerup'` purchase, and uses that to select the cost value from either `effect.mod_cost` or `effect.pu_cost`. It then reads `state.tokenBalance` and compares it against `cost` to compute the boolean `canAfford`.

The function calls `tierLabel` and `categoryLabel` (imported from `modifiers-render.ts`) and `rarityClass` to build badge markup, and calls `escapeHTML` (from `config.ts`) around `effect.name`, `effect.description`, and `effect.id` before embedding those values in HTML. It creates a `<div>` element via `document.createElement`, sets its `className` to `'bottom-sheet-overlay'`, writes a full sheet HTML structure into its `innerHTML`, and appends it to `document.body`. The rendered sheet contains rarity, timing, and category badges, a description paragraph, a socket note paragraph (the text differs based on `productType`), a cost row styled differently when `canAfford` is false, a conditionally rendered "not enough tokens" note when `canAfford` is false, a confirm button that is `disabled` when `canAfford` is false, and a cancel button.

A `close` closure is defined as `(): void => { overlay.remove(); }`, which removes the overlay element from the DOM when called. Three event listeners are registered: a click listener on `overlay` itself that calls `close()` only if `e.target === overlay` (i.e., a click on the backdrop outside the sheet); a click listener on the `#sheet-cancel` button that calls `close()`; and, only if `confirmBtn` is non-null and `canAfford` is true, an async click listener on the `#sheet-confirm` button.

The async confirm listener reads `confirmBtn.dataset.product` as a `ProductType`, immediately sets `confirmBtn.disabled = true` and `confirmBtn.textContent = 'Purchasing…'`, then branches: if `productType === 'modifier'` it awaits `handleBuyModifier(effect.id, effect.name)`; otherwise it awaits `handleBuyPowerup(effect.id, effect.name)`. Both calls return a boolean assigned to `ok`. After the awaited call resolves, `close()` is called unconditionally, and then `onBuySuccess()` is called only if `ok` is `true`. The entire await block sits inside a `try/finally`; the `finally` block always re-enables the confirm button (`confirmBtn.disabled = false`) and resets its text to `Buy ${productLabel} · ${cost} tokens`, regardless of whether the awaited call threw. There is no `catch` block, so any exception thrown by `handleBuyModifier` or `handleBuyPowerup` propagates out of the async listener as an unhandled promise rejection after `finally` runs.

`openBottomSheet` returns the `close` function, giving the caller a way to remove the overlay imperatively.
