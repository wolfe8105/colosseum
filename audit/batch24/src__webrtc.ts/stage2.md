# Stage 2 Outputs — webrtc.ts

## Agent 01

### joinDebate
This async function receives `debateId`, `role`, and optional `totalRounds`. It reads `totalRounds ?? MAX_ROUNDS` into a local `rounds`, then writes module state: `state.turnSequence` is reassigned by calling `buildTurnSequence(rounds)`; `state.debateState.debateId`, `.role`, `.status` ('connecting'), `.totalRounds`, and `.turn` (shallow-copied from `DEFAULT_TURN_STATE`) are all overwritten. It then calls `isPlaceholder()`; if that returns true, it calls `fire('placeholderMode', ...)` with a message string and returns early without touching mic, signaling, or firing `joining`. Otherwise it awaits `requestMic()` to obtain the local audio track, calls `enforceMute()` synchronously to keep the side muted until `startLive()`, then awaits `setupSignaling(debateId)` to open the realtime channel, and finally calls `fire('joining', { debateId, role })`. Any rejection from `requestMic` or `setupSignaling` propagates out of the function as an unhandled promise rejection; there is no try/catch. The awaited calls occur in this order: `requestMic` → `setupSignaling`. `enforceMute` runs between them synchronously. No return value (resolves to `void`).

### startLive
This async function takes no parameters and reads only `state.debateState.role`. It branches on whether `role === 'a'`: if true, it calls `sendSignal('turn-start', { stepIndex: 0 })` to broadcast the start signal over the signaling channel, then calls `beginStep(0)` to start the first turn step locally. If `role !== 'a'`, the function does nothing and resolves to `undefined`. Despite being marked `async`, it contains no `await`; the two inner calls run synchronously in the order `sendSignal` → `beginStep`. It has no return value, no try/catch, and writes no module state directly (state mutation happens inside `beginStep`).

### leaveDebate
This synchronous function takes no parameters and performs full teardown. It reads and writes extensively on `state`. First it calls `stopWorkerTimer()` then `terminateWorkerTimer()` to halt the countdown worker. It sets `state.iceRestartAttempts = 0`. If `state.disconnectTimer` is truthy, it calls `clearTimeout` on it and nulls the slot; likewise for `state.setupTimer`. If `state.peerConnection` is non-null, it calls `.close()` on the RTCPeerConnection and nulls the slot. If `state.localStream` exists, it iterates every track via `getTracks().forEach` calling `t.stop()` on each, then nulls the slot. `state.remoteStream` is unconditionally set to null. If `state.signalingChannel` is non-null, it calls `.unsubscribe()` on the Supabase realtime channel and nulls the slot. If `state.activeWaveform` is non-null, it calls `.stop()`, then fires `.audioCtx.close()` whose returned promise has a `.catch` attached that logs via `console.warn('[WebRTC] audioCtx close failed:', e)` — this close is fire-and-forget (not awaited); `state.activeWaveform` is then nulled. After resource teardown, `state.debateState` is reassigned to a fresh object literal with `debateId: null`, `role: null`, `status: 'idle'`, `round: 0`, `totalRounds: MAX_ROUNDS`, `roundTimer: null`, `breakTimer: null`, `timeLeft: 0`, `isMuted: false`, and a shallow-copied `DEFAULT_TURN_STATE` for `turn`. `state.turnSequence` is rebuilt via `buildTurnSequence(MAX_ROUNDS)`. Finally it calls `fire('left', {})`. No try/catch surrounds the body, so exceptions thrown by `close()`, `stop()`, or `unsubscribe()` would propagate and abort the remaining teardown steps; only the `audioCtx.close()` promise rejection is caught. No return value.

### getState
This synchronous function takes no parameters and returns a shallow-cloned snapshot of `state.debateState` typed as `Readonly<DebateState>`. It spreads `state.debateState` into a new object, and separately spreads `state.debateState.turn` into a new nested `turn` object, overwriting the top-level spread's `turn` reference. The returned object is a two-level copy: top-level fields and the `turn` sub-object are fresh, but any other object-valued fields (e.g. `roundTimer`, `breakTimer` timer handles) remain shared references. It writes nothing, calls no other functions, has no branches or error paths.

### getLocalStream
This synchronous function takes no parameters, reads `state.localStream`, and returns it as-is (either the live `MediaStream` or `null`). It writes nothing, calls nothing, has no branches.

### getRemoteStream
This synchronous function takes no parameters, reads `state.remoteStream`, and returns it as-is (either the live `MediaStream` or `null`). It writes nothing, calls nothing, has no branches.

### isConnected
This synchronous function takes no parameters and returns a `boolean`. It reads `state.peerConnection` using optional chaining — `state.peerConnection?.connectionState` — and compares the result to the string `'connected'`. If `state.peerConnection` is null, the optional chain yields `undefined`, and the `===` comparison returns `false`. Otherwise it reads the native `RTCPeerConnection.connectionState` property and returns true only when it is exactly `'connected'` (any other state such as `'connecting'`, `'disconnected'`, `'failed'`, `'closed'`, or `'new'` returns false). It writes nothing, calls no other functions.

### getTurnSequence
This synchronous function takes no parameters, reads `state.turnSequence`, and returns it directly typed as `readonly TurnStep[]`. It returns the live module-level reference (no copy), so mutation by callers would affect module state, though the `readonly` type discourages it. It writes nothing, calls nothing, has no branches.

There is no try/catch at the top level; only the `audioCtx.close()` rejection is captured. The function returns `void`.
