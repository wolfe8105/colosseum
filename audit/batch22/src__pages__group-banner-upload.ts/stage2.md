# Stage 2 Outputs — group-banner-upload.ts

## Agent 01

### openBannerUploadSheet
This synchronous, non-async function reads four parameters: `groupId`, `currentTier`, `wins`, and `losses`. It first calls `document.getElementById('gb-backdrop')?.remove()` to delete any existing element with that ID from the DOM. It then computes `total` as `wins + losses`, computes `winRate` as `Math.round((wins / total) * 100)` when `total > 0` (otherwise `0`), and derives two booleans `t2Unlocked` and `t3Unlocked` from `currentTier >= 2` and `currentTier >= 3`. It calls `document.createElement('div')` twice to build a `backdrop` element (className `gb-backdrop`, id `gb-backdrop`) and a `sheet` element (className `gb-sheet`). The `sheet.innerHTML` is assigned a template literal that interpolates `wins`, `losses`, `winRate`, `currentTier`, and the two unlocked booleans directly into the markup without escaping; the markup conditionally includes upload buttons (`#gb-t2-btn`, `#gb-t3-btn`) and hidden file inputs (`#gb-t2-input` accepting `image/*`, `#gb-t3-input` accepting `video/*,image/gif`) only when the corresponding tier is unlocked. The function then calls `backdrop.appendChild(sheet)` and `document.body.appendChild(backdrop)`.

After mounting, it queries the sheet for `#gb-t2-btn` and attaches a click listener that queries `#gb-t2-input` and calls `.click()` on it; it queries `#gb-t2-input` and attaches an async `change` listener that reads `e.target.files?.[0]` and, if a file exists, awaits `_uploadBanner(groupId, file, 'static', backdrop)`. The same pattern is repeated for `#gb-t3-btn`/`#gb-t3-input` with the `'animated'` type. Finally it attaches a click listener on `backdrop` that calls `_closeSheet(backdrop)` only when `e.target === backdrop` (i.e., clicks on the sheet itself do not close). The function returns `void`. It writes no module-level state — all state is in the DOM.

### _closeSheet
This synchronous function takes one parameter, `backdrop`, which is an `HTMLElement`. It writes `backdrop.style.transition = 'opacity 0.2s'` and then `backdrop.style.opacity = '0'`, mutating the inline style of the passed element. It then calls `setTimeout` with a 220ms delay; the timer callback invokes `backdrop.remove()`, removing the element from the DOM. The function returns immediately (it does not await the timeout) and returns `void`. It reads no module-level state, calls no other functions on the anchor list, and has no branches or error paths.

### _uploadBanner
This async function takes `groupId`, `file`, `type` (`'static'` or `'animated'`), and `backdrop`. It first reads `file.size`; if greater than `10 * 1024 * 1024` (10MB), it calls `showToast('File too large — max 10MB', 'error')` and returns early without resetting any button state (no button has been disabled yet at this point). It then queries `backdrop` for either `#gb-t2-btn` (when `type === 'static'`) or `#gb-t3-btn` (otherwise) and assigns the result to `btn`; if `btn` is non-null, it sets `btn.disabled = true` and `btn.textContent = 'UPLOADING…'`. This button mutation happens outside the subsequent `try` block.

Inside the `try`, it calls `getSupabaseClient()` synchronously and throws `new Error('Not connected')` if the result is falsy. It computes `ext` as `file.name.split('.').pop()` falling back to `'jpg'` for static or `'mp4'` for animated, and `path` as `${groupId}/${type}.${ext}`. It then awaits `client.storage.from('group-banners').upload(path, file, { upsert: true, contentType: file.type })`; if `upErr` is truthy it throws `new Error(upErr.message)`. It next awaits `client.storage.from('group-banners').createSignedUrl(path, 60 * 60 * 24 * 365)` (a one-year-expiry signed URL); if `signErr` is truthy or `signedData?.signedUrl` is missing it throws `new Error('Could not get URL')`, otherwise it assigns the signed URL to `url`. It then awaits `safeRpc('save_group_banner', { p_group_id: groupId, p_static_url: type === 'static' ? url : null, p_animated_url: type === 'animated' ? url : null })`; if `error` is truthy it throws `new Error(error.message ?? 'Save failed')`, and it casts `data` to `{ error?: string } | null` as `result` and throws `new Error(result.error)` if `result?.error` is truthy. On the success path it calls `showToast('Banner updated ✓', 'success')`, calls `_closeSheet(backdrop)` (the anchor-list function that fades and removes the backdrop), and calls `window.dispatchEvent(new CustomEvent('group-banner-updated', { detail: { groupId } }))`.

The `catch (err)` branch calls `showToast((err as Error).message || 'Upload failed', 'error')` and, if `btn` was non-null, restores `btn.disabled = false` and `btn.textContent = 'UPLOAD'`. There is no `finally` block, so the button reset only occurs on the `catch` path; on the success path the button is not reset because `_closeSheet` removes the entire backdrop. Note that the early-return for oversized files happens before `btn` is queried, so the button reset in the catch block does not apply to the size-check path. The function returns `Promise<void>`.

Awaited calls in order: `client.storage.from(...).upload(...)`, `client.storage.from(...).createSignedUrl(...)`, `safeRpc('save_group_banner', ...)`.
