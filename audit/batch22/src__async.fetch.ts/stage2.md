# Stage 2 Outputs — async.fetch.ts

## Agent 01

### fetchTakes
This async function accepts an optional `section` string parameter. It first reads the module-imported `FEATURES.hotTakes` flag and returns immediately if false. It calls `getSupabaseClient()` to obtain a client `sb`, then calls `getIsPlaceholderMode()`; if `sb` is null or placeholder mode is active, it returns early. Inside a try block, it builds a PostgREST query chain on `sb.from('hot_takes')` selecting an explicit column list including a nested `profiles(username, elo_rating, verified_gladiator)` join, ordered by `created_at` descending and limited to 30 rows. If `section` was provided and is not the literal string `'all'`, it appends an `.eq('section', section)` filter to the query. It then awaits the query as a promise yielding `{ data, error }`; if `error` is truthy it throws, which is caught by the outer catch that logs `'fetchTakes error:'` with the exception via `console.error` and returns without rethrowing.

When `data` is non-null and non-empty, it overwrites `state.hotTakes` by mapping each row into an object with fields `id`, `user_id`, `username` (from joined profile or empty string), `user` (uppercased username or `'ANON'`), `elo` (profile rating or 1200), `verified_gladiator` (profile flag or false), `tokens` set to literal 0, `text` (from `content`), `section`, `reactions` and `challenges` (from `reaction_count`/`challenge_count` or 0), `time` produced by calling `_timeAgo` on `created_at`, and `userReacted` initialized to false. It then reads `getCurrentUser()?.id`; if a user id exists, it enters a nested try block that awaits a second query against `hot_take_reactions` selecting `hot_take_id` filtered by `eq('user_id', userId)` and `in('hot_take_id', ...)` with the ids from `state.hotTakes`. If that result's `data` (`reacts`) is truthy, it constructs a `Set` of reacted ids and mutates each entry of `state.hotTakes` in place by setting `userReacted` to whether the set contains the entry's id. The inner catch swallows any error silently with the comment `non-critical`. Both awaited supabase calls are awaited; there are no fire-and-forget calls. Return value is `void`.

### fetchPredictions
This async function takes no parameters. It reads `FEATURES.predictions` and returns early if false, then calls `getSupabaseClient()` and `getIsPlaceholderMode()`, returning early if `sb` is null or placeholder mode is active. Inside a try block it awaits `safeRpc<Record<string, unknown>[]>('get_hot_predictions', { p_limit: 10 })`. If the returned `error` is truthy it throws into the catch, which logs `'fetchPredictions error:'` with the exception via `console.error` and returns.

When `data` is non-null and the array length is greater than zero, it overwrites `state.predictions` with a mapping of each row into an object containing `debate_id`, `topic`, `p1` (preferring `p1_username`, falling back to `p1_display_name`, then literal `'Side A'`), `p2` (same fallback chain ending in `'Side B'`), `p1_elo` and `p2_elo` (defaulting to 1200), `total` from `prediction_count` or 0, and `pct_a`/`pct_b`. The percentage fields branch on whether `prediction_count` is greater than zero: if so they compute `Math.round((picks_a / prediction_count) * 100)` and the analogous expression for `picks_b`; otherwise both default to 50. `user_pick` is set to literal null and `status` is copied from the row. The single `safeRpc` call is awaited; no fire-and-forget calls. Return value is `void`.

### fetchStandaloneQuestions
This async function accepts an optional `category` string parameter. It calls `getSupabaseClient()` and `getIsPlaceholderMode()`, returning early if `sb` is null or placeholder mode is active. Note this function does not consult any `FEATURES` flag, unlike the previous two. Inside a try block it awaits `safeRpc<StandaloneQuestion[]>('get_prediction_questions', { p_limit: 20, p_category: category || null })`, passing null when `category` is falsy. If `error` is truthy it throws into the catch, which logs `'fetchStandaloneQuestions error:'` with the exception via `console.error` and returns. If `data` is truthy, is an array per `Array.isArray`, and has length greater than zero, it assigns the cast value to `state.standaloneQuestions`, overwriting prior contents. The single `safeRpc` call is awaited; no fire-and-forget calls. Return value is `void`.
