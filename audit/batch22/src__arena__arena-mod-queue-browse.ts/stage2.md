# Stage 2 Outputs — arena-mod-queue-browse.ts

## Agent 01

### showModQueue
Synchronous void function with no parameters. Calls `set_view('modQueue')` to mutate module-level `view` state, then `history.pushState` with state `{ arenaView: 'modQueue' }` and empty URL. Reads module-level `screenEl`; if truthy, clears its `innerHTML`. Calls `getCurrentProfile()` from `auth.ts` and stores the result in local `profile`. Constructs a new `div` with class `arena-lobby arena-fade-in`, then assigns a template-literal `innerHTML` containing a hero header, a back button, a conditional "CREATE DEBATE" button (rendered only when `profile?.is_moderator` is truthy), and an empty `mod-queue-list` div. Appends the container to `screenEl` via optional chaining (no-op if `screenEl` is null). Attaches a click listener on `#mod-queue-back` that calls `stopModQueuePoll()` (anchor 5), dynamically imports `./arena-lobby.ts` via awaited `import()`, then calls `renderLobby()`. Attaches a click listener on `#mod-queue-create-debate` that calls `stopModQueuePoll()` then `showModDebatePicker()`. Both event handlers use optional chaining on `getElementById`, so missing elements silently skip listener attachment. Finally fires `loadModQueue()` (anchor 2) as fire-and-forget via `void`, then calls `startModQueuePoll()` (anchor 4) synchronously.

### loadModQueue
Async function returning `Promise<void>`, no parameters. Reads DOM element `#mod-queue-list` into `listEl`; early-returns if null. Awaits `safeRpc<ModQueueItem[]>('browse_mod_queue')` and destructures `data` and `error`. On error: extracts `msg` from `error.message` or coerces error to string; if `msg` includes the literal string `'Not an available moderator'`, writes a "not Available" message into `listEl.innerHTML` and returns; otherwise calls `showToast(msg)` and returns. On success: coerces `data` to `ModQueueItem[]` (defaulting to `[]`) into `rows`. If `rows.length === 0`, writes an empty-state message into `listEl.innerHTML` and returns. Otherwise maps each row to an HTML string: computes `waitMs` as `Date.now()` minus parsed `row.created_at`, derives `waitMin`/`waitSec`, formats `waitStr` as either `${m}m ${s}s` or `${s}s`, defaults `nameA`/`nameB` from `row.debater_a_name`/`row.debater_b_name` to `'Unknown'`/`'TBD'`, then interpolates `row.category`, `row.mode`, `row.topic`, `nameA`, `nameB`, `waitStr`, and `row.debate_id` into a card template. The mapped strings are joined and assigned to `listEl.innerHTML` (note: per the file's own LANDMINE comment LM-MODQUEUE-001, none of the user-supplied fields pass through `escapeHTML`). Then queries all `.mod-queue-claim-btn` buttons and attaches a click listener to each that reads `btn.dataset.debateId` (non-null asserted) and fire-and-forgets `claimModRequest(debateId, btn)` (anchor 3) via `void`.

### claimModRequest
Async function returning `Promise<void>`, parameters `debateId: string` and `btn: HTMLButtonElement`. Sets `btn.disabled = true` and `btn.textContent = 'REQUESTING…'`. Awaits `safeRpc('request_to_moderate', { p_debate_id: debateId })` and destructures `error`. On error: re-enables the button (`btn.disabled = false`), restores `btn.textContent` to `'REQUEST TO MOD'`, calls `showToast('Another mod got there first — queue refreshed')`, then fire-and-forgets `loadModQueue()` (anchor 2) via `void`, and returns. On success: calls `stopModQueuePoll()` (anchor 5) to halt polling, then reads DOM element `#mod-queue-list` into `listEl`; if non-null, overwrites its `innerHTML` with a "Request sent" / "Waiting for the debaters to accept" success message. No return value.

### startModQueuePoll
Synchronous void function with no parameters. Reads module-level `modQueuePollTimer`; if truthy, calls `clearInterval(modQueuePollTimer)` (note: does not null it before the next line). Calls `setInterval` with a 5000 ms delay and stores the resulting timer handle via `set_modQueuePollTimer(...)`. The interval callback reads module-level `view`; if it is not equal to `'modQueue'`, the callback calls `clearInterval(modQueuePollTimer!)` (non-null asserted), then `set_modQueuePollTimer(null)`, then returns without polling. Otherwise the callback fire-and-forgets `loadModQueue()` (anchor 2) via `void`. The interval continues firing every 5 seconds until the view-check self-terminates it or until `stopModQueuePoll` (anchor 5) clears it externally.

### stopModQueuePoll
Synchronous void function with no parameters. Reads module-level `modQueuePollTimer`; if truthy, calls `clearInterval(modQueuePollTimer)` then `set_modQueuePollTimer(null)`. If `modQueuePollTimer` is null/falsy, the function is a no-op. No return value, no error paths, no other calls.
