# Stage 2 Outputs — arena-feed-wiring-spectator.ts

## Agent 01

### wireSpectatorTipButtons

This async function reads the debate parameter and DOM elements with IDs `feed-tip-status` and class `feed-tip-btn`. It makes an async RPC call to `get_user_watch_tier` via `safeRpc` and parses the returned data to extract a tier string, defaulting to `'Unranked'` if the call errors or returns unexpected formats. If tier is `'Unranked'`, it sets the status element's text content to a prompt message and returns early without enabling buttons. Otherwise, it sets `disabled = false` on all tip buttons, updates the status element to show the tier and "Tap to tip" text, and attaches click event listeners to each button. Each listener invokes `handleTip` without awaiting (via `void handleTip()`), passing the button, debate, and status element. The RPC call is awaited; errors during the RPC are caught, logged to console, and silently ignored. The function returns a resolved promise after all DOM updates and event listener registration complete.

### handleTip

This async function reads the button's `data-side` and `data-amount` attributes and parses amount as a number, returning early if either is falsy. It disables all tip buttons to prevent double-tapping, then makes an async RPC call to `cast_sentiment_tip` with the debate ID, side, and amount, awaiting the result. If the RPC returns an error or no data, it logs a warning, shows an error toast, re-enables all buttons, and returns. If data is present, it casts to a result object and checks the `result.error` field: if present, it shows a context-specific error toast (checking for `'insufficient_tokens'`, `'unranked_blocked'`, `'debate_not_live'`, or a generic error), re-enables buttons, and returns. On success, it reads the current `pendingSentimentA` and `pendingSentimentB` module-level state, adds the tipped amount to the appropriate variable via `set_pendingSentimentA` or `set_pendingSentimentB`, calls `applySentimentUpdate()` to update the UI, updates the status element text to show confirmation with a checkmark, shows a success toast, and schedules a 800ms timeout callback that re-enables all buttons and resets the status text. The entire function body is wrapped in try/catch; if an exception is thrown, it logs a warning, shows an error toast, and re-enables buttons.

## Agent 02

### wireSpectatorTipButtons

At runtime, this async function receives a `CurrentDebate` object and performs DOM wiring for sentiment tip buttons. It reads three pieces of state on entry: the DOM element with id `feed-tip-status` (may be null), and all HTML button elements matching the class `.feed-tip-btn`. It then calls `safeRpc('get_user_watch_tier', {})` and awaits the result, which returns a tuple `{ data, error }`. The function attempts to extract a `tier` string from the response data using two fallback patterns: first checking if data is an array with a first element containing a `tier` field, then checking if data itself is an object with a `tier` field; if both patterns fail or the RPC errors, the tier defaults to 'Unranked' and the error is caught and logged to console. If the tier is 'Unranked', the function updates the status element's text content to 'Watch a full debate to unlock tipping.' and returns early without modifying buttons. For non-Unranked tiers, the function re-enables all tip buttons by setting their `disabled` property to false, updates the status element text to display the tier and 'Tap to tip', and then attaches a click event listener to each button that invokes `handleTip(btn, debate, statusEl)` wrapped in `void` to discard its promise (fire-and-forget).

### handleTip

At runtime, this async function receives an `HTMLButtonElement`, a `CurrentDebate`, and an optional `HTMLElement` for status display. It reads the `data-side` and `data-amount` attributes from the button as 'a' or 'b' and a number respectively; if either is missing or falsy, the function returns early. It then queries and disables all `.feed-tip-btn` buttons by setting their `disabled` property to true to prevent double-tap. The function calls `safeRpc('cast_sentiment_tip', {...})` with the debate id, selected side, and amount, awaiting the result. If the RPC errors or returns no data, it logs a warning, shows an error toast via `showToast('Tip failed', 'error')`, re-enables all buttons, and returns. If the RPC succeeds but the result object contains an `error` field, the function shows a context-specific error toast ('insufficient_tokens', 'unranked_blocked', 'debate_not_live', or generic 'Tip failed'), re-enables all buttons, and returns. On success (no error field in result), the function reads the current `pendingSentimentA` and `pendingSentimentB` state, increments the appropriate one by the tip amount, calls `set_pendingSentimentA` or `set_pendingSentimentB` to write the new value, calls `applySentimentUpdate()` to refresh the UI, updates the status element to show the tip amount and target side with a checkmark, shows a success toast, and then schedules a `setTimeout` callback for 800ms that re-enables all buttons and resets the status text back to 'Tap to tip'. Errors during the entire try block are caught, logged, and result in showing an error toast and re-enabling buttons.

It then disables all tip buttons to prevent double-taps and calls `safeRpc('cast_sentiment_tip', { p_debate_id, p_side, p_amount })`.
