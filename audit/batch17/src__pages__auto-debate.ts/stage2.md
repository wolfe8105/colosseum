# Stage 2 Outputs — src/pages/auto-debate.ts

## Agent 01

### showError
This synchronous function takes a `msg: string` parameter and returns `void`. It reads the module-level `loadingEl` (the `#loading` DOM element captured at module load) and `app` (the `#app` DOM element). If `loadingEl` is truthy, it writes `'none'` to its `style.display`. If `app` is truthy, it writes a string to its `innerHTML` composed of an error-state div containing the literal `❌ ` followed by the result of calling `escapeHTML(msg)` (imported from `../config.ts`), and a secondary div containing an anchor back to `/` labeled "Go to The Moderator". The only function call it makes is `escapeHTML(msg)`. There are no branches beyond the two null checks on the DOM references, no loops, no try/catch, and no early return other than implicit falls-through when the elements are absent.

### shareDebateImpl
This synchronous function takes a `method: string` parameter and returns `void`. It reads `(window as unknown as Record<string, unknown>)._debate`, casts the result to `AutoDebateData | undefined` into local `d`, and returns early if `d` is falsy. It reads `window.location.href` into `url`, and reads `d.share_hook`, defaulting to the literal template `` `AI judged this debate: "${d.topic}" — the result is wild.` `` into `text`. Control then branches on `method`. When `method === 'copy'`, it calls `navigator.clipboard.writeText(url)` and chains a `.then()` that queries `.share-bar .share-btn:first-child`, and if that element exists writes `'✅ Copied!'` to its `textContent`, then schedules a `setTimeout` after 2000ms that writes `'📋 Copy Link'` back to the same `textContent`; a `.catch` on the clipboard promise writes a warning via `console.warn` with prefix `'[AutoDebate] clipboard copy failed:'`. When `method === 'twitter'`, it calls `window.open` with a Twitter intent URL built via `encodeURIComponent(text)` and `encodeURIComponent(url)`, targeting `_blank`.

When `method === 'native'`, control splits three ways based on `d.yes_votes`. If `d.yes_votes !== undefined`, it calls `shareCard` (imported from `../cards.ts`) with an object containing `topic`, `sideA` (defaulting to `'Side A'`), `sideB` (defaulting to `'Side B'`), `yesVotes` (defaulting to `0`), `noVotes` (defaulting to `0`), and `size: 'og'`. Otherwise, if `navigator.share` exists, it calls `navigator.share({ title: d.topic, text, url })` and attaches a `.catch` that swallows the rejection with a no-op comment `/* cancelled */`. If neither branch applies, it falls back to `navigator.clipboard.writeText(url)` with a `.catch` that logs `'[AutoDebate] clipboard fallback failed:'` via `console.warn`. The function is not `async`; all promise chains are fire-and-forget (no `await`).

### loadDebate
This `async` function takes no parameters and returns `Promise<void>`. It reads the module-level `debateId` (parsed from `URLSearchParams` of `window.location.search` at module load) and returns early if it is falsy. Inside a `try` block, it awaits `sb.from('auto_debates').select('*').eq('id', debateId).single()`, destructuring `data` and `error` from the result. If `error` is truthy or `data` is falsy, it calls `showError('Debate not found or has been removed.')` and returns. Otherwise, it enters a nested `try` that awaits `sb.rpc('log_event', ...)` with `p_event_type: 'debate_view'` and a `p_metadata` object containing `debate_id: debateId` and `debate_topic` read from `(data as AutoDebateData).topic` defaulting to `null`; the nested `catch` is an empty `/* non-blocking */` swallow.

After the logging attempt, it calls `renderDebate` (imported from `./auto-debate.render.ts`) passing `data` cast to `AutoDebateData`, the module-level `loadingEl`, the module-level `app`, and a callback `(id, cat) => void loadMoreDebates(sb as Parameters<typeof loadMoreDebates>[0], id, cat)` that forwards to `loadMoreDebates` (imported from `./auto-debate.render.ts`); the callback itself is synchronous from `renderDebate`'s perspective and `void`s the returned promise. The outer `catch` block (with no bound error variable) calls `showError('Failed to load debate. Please try again.')`. The awaited calls are the `single()` query and the `log_event` RPC; `renderDebate` is called without `await` (its return value is discarded).
