# Stage 2 Outputs — profile-depth.section.ts

## Agent 01

### openSection

`openSection` receives a `sectionId` string and an `onSectionClick` callback. It immediately calls `setActiveSection(sectionId)`, writing the active section into module-level state imported from `profile-depth.state.ts`. It then searches the `SECTIONS` array (imported from `profile-depth.data.ts`) for the entry whose `id` matches `sectionId`; if no match is found it returns early. It queries the DOM for the element with id `question-panel`; if that element is absent it returns early. On the happy path it adds the CSS class `open` to the panel element, then overwrites the element's `innerHTML` with a template string that includes the section's `icon`, `name`, and `reward.text` — all passed through `escapeHTML` — followed by the result of mapping each entry in `section.questions` through `renderQuestion` (imported from `profile-depth.render.ts`) and joining those strings, and finally a save button element. After setting innerHTML it calls `wireQuestions()`, which attaches input listeners to all interactive question elements now present in the DOM. It then queries for the element with id `save-section-btn` and, if found, attaches a click listener that calls `saveSection(section, onSectionClick)`. It calls `panel.scrollIntoView` with `behavior: 'smooth'` and `block: 'start'` to scroll the panel into view. Finally it calls `renderGrid(onSectionClick)`, which repaints the section grid with the current completion state. The function is synchronous and returns `void`.

### wireQuestions

`wireQuestions` reads from the DOM exclusively; it takes no parameters and reads no module-level state directly. It runs four independent `querySelectorAll` passes over the document.

For elements matching `.q-input` (text inputs), it attaches an `input` event listener to each. When that listener fires, it reads `el.dataset.qid`; if the value is present, it calls `setAnswer(qid, el.value)`, writing the raw string value into the `answers` state map.

For elements matching `.chip-group`, it reads `group.dataset.multi` and `group.dataset.max` to determine whether the group allows multiple selections and what the selection ceiling is (defaulting to 99 if the attribute is absent or unparseable). For each `.chip` child element it attaches a click listener. When a chip is clicked, the listener reads `group.dataset.qid` and `chip.dataset.val`; if either is absent it returns without action. In multi-select mode, it reads the current value from `answers[qid]` — treating it as a string array if it already is one, or starting from an empty array otherwise. If the clicked chip's value is already in the array, that value is filtered out and `chip.classList.remove('selected')` is called; if the array has not yet reached `max`, the value is pushed in and `chip.classList.add('selected')` is called. The updated array is written back via `setAnswer`. In single-select mode, it calls `classList.remove('selected')` on every sibling chip, then adds `'selected'` to the clicked chip and calls `setAnswer(qid, val)` with the string value.

For elements matching `.q-slider` (range inputs), it attaches an `input` listener. The listener reads `el.dataset.qid`; if absent it returns without action. It calls `setAnswer(qid, parseInt(el.value))`, storing the parsed integer. It then queries for the element with id `'slider-val-' + qid` and, if found, sets its `textContent` to the raw string `el.value` to update a visible label.

For elements matching `.q-select` (select dropdowns), it attaches a `change` listener that reads `el.dataset.qid` and, if present, calls `setAnswer(qid, el.value)` with the selected string value.

The function is synchronous and returns `void`. It writes no module-level state directly; all state writes are delegated to `setAnswer`.

### saveSection

`saveSection` is `async` and receives a `Section` object and the `onSectionClick` callback. It begins by iterating `section.questions` synchronously, calling `hasAnswer(answers[q.id])` for each question and setting the local variable `allAnswered` to `false` if any question lacks an answer. It then writes the entire `answers` map to `localStorage` under the key `'colosseum_profile_depth'` as a JSON string, unconditionally and before any network call.

If `allAnswered` is true it calls `addCompletedSection(section.id)`, writing the section id into the `completedSections` Set in state, then writes the updated set to `localStorage` under `'colosseum_depth_complete'`. It calls `checkProfileMilestones(completedSections.size)` (imported from `tokens.ts`), passing the current count of completed sections. It then calls `showReward(section.reward)`, triggering the reward toast UI.

It reads `getIsPlaceholderMode()` and `getCurrentUser()`. If a user is present and placeholder mode is not active, it enters a `try` block. Inside, it builds a `sectionAnswers` record containing only the questions in this section that have a defined value in `answers`, then `await`s `safeRpc('save_profile_depth', { p_section_id, p_answers })`. If the RPC returns an error object, it logs it with `console.error` and continues. It then iterates `section.questions` a second time, checking each question id against `previouslyAnsweredIds` (a `Set` in module state) to count newly answered questions and mutating `previouslyAnsweredIds` by adding each newly answered id. If `newCount` is greater than zero, it `await`s `safeRpc('increment_questions_answered', { p_count: newCount })`. If that call returns an error, it logs it. If the response has `data.ok` truthy, it calls `setServerQuestionsAnswered` with the returned `questions_answered` value (falling back to the current `serverQuestionsAnswered` if absent), then calls `renderTierBannerUI(serverQuestionsAnswered)` and `updateMilestoneBar()` to repaint the tier UI. If `allAnswered` is true, it `await`s `safeRpc('claim_section_reward', { p_section_id: section.id })`; if the response has `data.success` truthy, it logs the `power_up_name`. Any exception thrown anywhere inside the `try` block is caught and logged via `console.error`. No error is re-thrown and execution continues past the `catch` block regardless.

After the conditional network block, `updateMilestoneBar()` is called unconditionally. `renderGrid(onSectionClick)` is then called to repaint the grid. If `allAnswered` is true, a `setTimeout` with a 2000 ms delay is set — the interval handle is not stored anywhere. When it fires it queries the DOM for `'question-panel'`, removes the `'open'` class if the element exists, calls `setActiveSection(null)`, and calls `renderGrid(onSectionClick)` again.

### showReward

`showReward` receives a `SectionReward` object. It defines a local lookup object mapping four power-up id strings (`'reveal'`, `'multiplier_2x'`, `'silence'`, `'shield'`) to emoji strings. It queries the DOM for the element with id `'reward-icon'`; if found it sets `textContent` to the emoji corresponding to `reward.powerUpId`, falling back to `'⚡'` if the id is not in the map. It queries for `'reward-text'`; if found it sets `textContent` to the literal string `'POWER-UP EARNED'`. It queries for `'reward-desc'`; if found it sets `textContent` to `reward.text`. It queries for `'reward-toast'`; if found it adds the class `'show'` to make the toast visible, then schedules a `setTimeout` for 2500 ms that removes the `'show'` class — the timer handle is not stored. The function is synchronous, writes nothing outside the DOM, and returns `void`. All four DOM lookups are independent null-guarded branches; the function produces a partial result if any element is absent.

### wireQuestions

`wireQuestions` is a synchronous function that takes no parameters and returns nothing. It reads the DOM by running four separate `querySelectorAll` sweeps, each targeting a different CSS class.

The first sweep selects all `.q-input` elements. For each one, it attaches an `input` event listener that reads `el.dataset.qid` and, if present, calls `setAnswer(qid, el.value)` to write the text value into module state.

The second sweep selects all `.chip-group` elements. For each group it reads `group.dataset.multi` and `group.dataset.max` to determine whether the chip group accepts multiple selections and what the maximum selection count is (defaulting to 99 if the attribute is absent or non-numeric). It then attaches a click listener to each `.chip` child. Inside each listener it reads `group.dataset.qid` and `chip.dataset.val`; if either is missing the listener returns early. In multi-select mode it reads the current answer from the module-level `answers` map (casting it to a string array if it is already an array, otherwise starting with an empty array). If the clicked value is already in the array, it is removed and the chip's `selected` class is removed. If the value is not present and the array has not reached `max`, the value is pushed and the `selected` class is added. Either way `setAnswer` is called with the updated array. In single-select mode the listener removes `selected` from all sibling chips, adds it to the clicked chip, and calls `setAnswer(qid, val)` with the string value.

The third sweep selects all `.q-slider` elements. For each one it attaches an `input` event listener that reads `el.dataset.qid`, returns early if absent, calls `setAnswer(qid, parseInt(el.value))` to write a numeric value into state, then queries the DOM for a label element by id `slider-val-<qid>` and, if found, sets its `textContent` to the raw string value of the slider.

The fourth sweep selects all `.q-select` elements. For each one it attaches a `change` event listener that reads `el.dataset.qid` and, if present, calls `setAnswer(qid, el.value)`.

### saveSection

`saveSection` is an async function that takes a `Section` object and the `onSectionClick` callback. It computes `allAnswered` by iterating over `section.questions` and calling `hasAnswer(answers[q.id])` for each question, reading from the module-level `answers` map; `allAnswered` becomes false if any question has no valid answer.

It unconditionally writes the full `answers` map to `localStorage` under the key `colosseum_profile_depth` as a JSON string.

If `allAnswered` is true, it calls `addCompletedSection(section.id)` to update the `completedSections` Set in module state, then writes that Set (spread into an array) to `localStorage` under `colosseum_depth_complete`. It then calls `checkProfileMilestones(completedSections.size)` and calls `showReward(section.reward)`, which triggers the reward toast UI.

Next it reads `getIsPlaceholderMode()` and `getCurrentUser()`. If a user is present and placeholder mode is off, it enters a try block with three sequential awaited network calls. First it collects only the current section's answered questions into a local `sectionAnswers` object and awaits `safeRpc('save_profile_depth', ...)` with the section id and answers map; if the result carries an error, it is logged to the console. Second it counts questions in this section that now have valid answers but whose ids are not yet in the `previouslyAnsweredIds` Set, incrementing the Set as it goes. If the new count is greater than zero it awaits `safeRpc('increment_questions_answered', { p_count: newCount })`; on error it logs to console, and on success (when `data.ok` is truthy) it calls `setServerQuestionsAnswered` with the returned count (falling back to the current `serverQuestionsAnswered` value if the field is absent), then calls `renderTierBannerUI(serverQuestionsAnswered)` and `updateMilestoneBar()`. Third, if `allAnswered` is true, it awaits `safeRpc('claim_section_reward', { p_section_id: section.id })`; if the response data carries `success: true`, the claimed power-up name is logged to the console. Any thrown exception from the entire try block is caught and logged to console with the label `save_profile_depth exception`.

After the conditional auth block (whether it ran or not), it always calls `updateMilestoneBar()` and `renderGrid(onSectionClick)`. If `allAnswered` is true, it schedules an anonymous `setTimeout` at 2000 ms (noted in a comment as having no stored handle and therefore being uncancellable) that removes the `open` class from `question-panel`, calls `setActiveSection(null)`, and calls `renderGrid(onSectionClick)` again.

### showReward

`showReward` is a synchronous function that takes a `SectionReward` object. It defines a local lookup object `puIcons` mapping four power-up id strings (`reveal`, `multiplier_2x`, `silence`, `shield`) to emoji strings. It queries the DOM for an element with id `reward-icon`; if found, it sets that element's `textContent` to the emoji corresponding to `reward.powerUpId`, falling back to `'⚡'` if the id is not in the map. It queries for `reward-text`; if found, it sets `textContent` to the static string `'POWER-UP EARNED'`. It queries for `reward-desc`; if found, it sets `textContent` to `reward.text`. It then queries for `reward-toast`; if found, it adds the CSS class `show` to that element and schedules a `setTimeout` at 2500 ms to remove the `show` class again, producing a timed-visibility toast. No handle is stored for this timeout. The function does not return a value.
