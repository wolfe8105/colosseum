# Stage 2 Outputs — src/modifiers-render.ts

## Agent 01

### tierLabel
Reads the single `tier` parameter (a `RarityTier` string) and returns a new string produced by concatenating `tier.charAt(0).toUpperCase()` with `tier.slice(1)`. It calls `String.prototype.charAt`, `String.prototype.toUpperCase`, and `String.prototype.slice` in that order. There are no branches, no module-level state reads or writes, no external state access, and no error paths. It is synchronous.

### timingLabel
Reads the single `timing` parameter (a `ModifierTiming`) and returns the string literal `'Post-Match'` when `timing === 'end_of_debate'`, otherwise returns the string literal `'In-Debate'`. The comparison is the only branch; no other functions are called, no module-level or external state is touched, and the function is synchronous with no error handling.

### categoryLabel
Reads the single `cat` parameter (a `ModifierCategory`). On each call it constructs a fresh local object `map` literal containing twelve fixed key/value pairs (`token` → `'Token'`, `point` → `'Point'`, `reference` → `'Reference'`, `elo_xp` → `'Elo / XP'`, `crowd` → `'Crowd'`, `survival` → `'Survival'`, `self_mult` → `'Multiplier'`, `self_flat` → `'Flat Bonus'`, `opponent_debuff` → `'Debuff'`, `cite_triggered` → `'Cite'`, `conditional` → `'Conditional'`, `special` → `'Special'`). It then returns `map[cat] ?? cat`: if the lookup yields `undefined` or `null`, the original `cat` argument is returned instead. No other functions are called, no module-level or external state is read or written, and the function is synchronous with no error handling.

### rarityClass
Reads the single `tier` parameter (a `RarityTier`) and returns it unchanged. The function is an identity function over the tier string — no transformation, no branches, no calls, no state access, no error paths, synchronous.

### renderEffectCard
Reads the `effect` parameter (a `ModifierEffect`) and the `opts` parameter (defaulting to `{}` when omitted), accessing `effect.timing`, `effect.id`, `effect.name`, `effect.description`, `effect.tier_gate`, `effect.category`, `effect.mod_cost`, `effect.pu_cost`, and from `opts` the fields `showModButton`, `showPuButton`, `modButtonLabel`, and `puButtonLabel`. No module-level or external state is read. It first computes `timingBadge` by comparing `effect.timing === 'in_debate'`: when true it assigns the live-variant `<span>` string, otherwise it assigns the post-variant `<span>` string. It then computes `modBtn`: if `opts.showModButton` is truthy it builds a `<button>` template literal in which `escapeHTML(effect.id)` supplies the `data-effect-id` attribute and the button text is `escapeHTML(opts.modButtonLabel ?? \`Buy Modifier · ${Number(effect.mod_cost)} tokens\`)` — `Number()` is invoked on `effect.mod_cost` only when the fallback label is used, and `escapeHTML` is invoked on whichever label string is selected; if `opts.showModButton` is falsy, `modBtn` becomes the empty string. The same pattern runs for `puBtn` using `opts.showPuButton`, `opts.puButtonLabel`, `effect.pu_cost`, and `effect.id`.

It then returns a single template-literal HTML string for a `.mod-effect-card` wrapper. During construction of that template literal the following calls occur in this order: `rarityClass(effect.tier_gate)` for the class suffix on the wrapper `div`, `escapeHTML(effect.id)` for the wrapper's `data-effect-id`, `escapeHTML(effect.name)` for the name span, `rarityClass(effect.tier_gate)` a second time for the rarity-badge class suffix, `tierLabel(effect.tier_gate)` for the badge text, `escapeHTML(effect.description)` for the description paragraph, and `categoryLabel(effect.category)` for the category tag. The previously-computed `timingBadge`, `modBtn`, and `puBtn` strings are interpolated into the template as-is. The final string has `.trim()` called on it before being returned. The function is synchronous and has no branches beyond the three ternary-style assignments above, no loops, and no try/catch.

### renderModifierRow
Reads the `mod` parameter (an `OwnedModifier`) and the `opts` parameter (defaulting to `{}`), accessing `mod.modifier_id`, `mod.name`, `mod.tier_gate`, `mod.timing`, `mod.description`, and `opts.showSocketButton`. No module-level or external state is read. It first computes `socketBtn`: when `opts.showSocketButton` is truthy it builds a `<button class="mod-socket-btn">` template literal whose `data-modifier-id` attribute is produced by `escapeHTML(mod.modifier_id)` and whose label is the literal text `Socket`; when falsy, `socketBtn` is the empty string.

It then returns a `.mod-inventory-row` template-literal HTML string. During construction, in order: `escapeHTML(mod.modifier_id)` sets the row's `data-modifier-id`, `escapeHTML(mod.name)` fills the name span, `rarityClass(mod.tier_gate)` supplies the rarity-badge class suffix, `tierLabel(mod.tier_gate)` supplies the badge text, an inline comparison `mod.timing === 'in_debate'` picks either `'live'` or `'post'` as the timing-badge class suffix, `timingLabel(mod.timing)` supplies the timing-badge text, and `escapeHTML(mod.description)` fills the description paragraph. The previously-computed `socketBtn` is interpolated in, and the resulting string has `.trim()` called on it before being returned. The function is synchronous, has only the one ternary branch plus the inline timing-class ternary, no loops, and no try/catch.

### renderPowerupRow
Reads the `pu` parameter (a `PowerUpStock`) and the `opts` parameter (defaulting to `{}`), accessing `pu.effect_id`, `pu.name`, `pu.quantity`, `pu.timing`, `pu.description`, and from `opts` the fields `showEquipButton` and `debateId`. No module-level or external state is read. It first computes `equipBtn` using the compound condition `opts.showEquipButton && opts.debateId`: when both are truthy it builds a `<button class="mod-equip-btn">` template literal whose `data-effect-id` is `escapeHTML(pu.effect_id)` and whose `data-debate-id` is `escapeHTML(opts.debateId)`, with the literal label `Equip`; when either is falsy, `equipBtn` is the empty string. Note that if `showEquipButton` is true but `debateId` is missing, no button is rendered.

It then returns a `.mod-powerup-row` template-literal HTML string. During construction, in order: `escapeHTML(pu.effect_id)` sets the row's `data-effect-id`, `escapeHTML(pu.name)` fills the name span, `Number(pu.quantity)` is interpolated after a literal `×` for the quantity span, an inline `pu.timing === 'in_debate'` ternary selects either `'live'` or `'post'` as the timing-badge class suffix, `timingLabel(pu.timing)` supplies the timing-badge text, and `escapeHTML(pu.description)` fills the description paragraph. The previously-computed `equipBtn` is interpolated in, and the resulting string has `.trim()` called on it before being returned. The function is synchronous, has only the one compound-condition branch plus the inline timing-class ternary, no loops, and no try/catch.

No module-level state is written, no other functions are called, and there are no loops or try/catch.

### renderEffectCard
Synchronous function taking an `effect` parameter typed as `ModifierEffect` and an `opts` object that defaults to `{}` and optionally carries `showModButton`, `showPuButton`, `modButtonLabel`, and `puButtonLabel`. It reads only those parameters; no module-level state or DOM is touched. It first computes `timingBadge` via a ternary on `effect.timing`: if the value equals `'in_debate'` it is set to a `<span class="mod-timing-badge mod-timing-badge--live">In-Debate</span>` literal; otherwise it is set to a `<span class="mod-timing-badge mod-timing-badge--post">Post-Match</span>` literal. It then computes `modBtn` via a ternary on `opts.showModButton`: if truthy, a template-literal `<button>` string is built containing `data-effect-id="${escapeHTML(effect.id)}"` and a body that is `escapeHTML(opts.modButtonLabel ?? \`Buy Modifier · ${Number(effect.mod_cost)} tokens\`)` — so when `modButtonLabel` is undefined, a fallback label is composed from `Number(effect.mod_cost)` (numeric cast) before being passed through `escapeHTML`; if `showModButton` is falsy, `modBtn` is the empty string. `puBtn` is computed the same way against `opts.showPuButton`, using `opts.puButtonLabel`, `Number(effect.pu_cost)`, and an `escapeHTML(effect.id)` on the data attribute.

It then returns a trimmed template-literal string comprising a `<div class="mod-effect-card mod-effect-card--${rarityClass(effect.tier_gate)}">` wrapper whose `data-effect-id` is `escapeHTML(effect.id)`. The header contains a `<span>` with `escapeHTML(effect.name)`, a rarity badge whose class suffix is `rarityClass(effect.tier_gate)` and whose text is `tierLabel(effect.tier_gate)`, and the previously computed `timingBadge` inlined. A description paragraph uses `escapeHTML(effect.description)`. A meta block holds a category tag built from `categoryLabel(effect.category)` (not passed through `escapeHTML` — the category label values are hard-coded in `categoryLabel`). An actions block inlines `modBtn` and `puBtn`. The entire string has `.trim()` applied before it is returned. The function calls `escapeHTML` multiple times, `Number` twice (conditionally), `rarityClass` twice, `tierLabel` once, and `categoryLabel` once; nothing is awaited and nothing is written back to module state.

### renderModifierRow

It computes `socketBtn` via a ternary: when `opts.showSocketButton` is truthy, the variable holds a template-literal string of a `<button class="mod-socket-btn" data-modifier-id="${escapeHTML(mod.modifier_id)}">` whose body text is the literal `Socket`; otherwise `socketBtn` is the empty string. It then returns a trimmed template-literal string containing a `<div class="mod-inventory-row" data-modifier-id="${escapeHTML(mod.modifier_id)}">` wrapper. Inside the info block, a name span uses `escapeHTML(mod.name)`, a rarity badge uses `rarityClass(mod.tier_gate)` as a class suffix and `tierLabel(mod.tier_gate)` as the text, and a timing badge has a class suffix built from an inline ternary on `mod.timing === 'in_debate'` (`'live'` when true, `'post'` otherwise) with its body text produced by `timingLabel(mod.timing)`. A description paragraph uses `escapeHTML(mod.description)`. `socketBtn` is inlined after the description. The final string is returned after `.trim()`. The function calls `escapeHTML` three times, `rarityClass` once, `tierLabel` once, and `timingLabel` once. No module-level state is read or written, no DOM access occurs, nothing is awaited, and there are no try/catch blocks.

### renderPowerupRow

It computes `equipBtn` via a ternary whose condition is `opts.showEquipButton && opts.debateId`: if both are truthy, the variable is assigned a template-literal string for a `<button class="mod-equip-btn">` carrying `data-effect-id="${escapeHTML(pu.effect_id)}"` and `data-debate-id="${escapeHTML(opts.debateId)}"` with body text `Equip`; if either is falsy, `equipBtn` is the empty string, meaning no button renders unless a `debateId` was supplied. It then returns a trimmed template-literal string containing a `<div class="mod-powerup-row" data-effect-id="${escapeHTML(pu.effect_id)}">` wrapper. Inside the info block, a name span uses `escapeHTML(pu.name)`, a quantity span renders `×${Number(pu.quantity)}` (numeric cast, not escaped), and a timing badge has a class suffix built from an inline ternary on `pu.timing === 'in_debate'` (`'live'` when true, `'post'` otherwise) with body text produced by `timingLabel(pu.timing)`. A description paragraph uses `escapeHTML(pu.description)`. `equipBtn` is inlined after the description. The final string is returned after `.trim()`. The function calls `escapeHTML` four times (one of which, on `opts.debateId`, only runs inside the truthy branch of the `equipBtn` ternary), `Number` once, and `timingLabel` once. No module-level state is touched, nothing is awaited, and there are no try/catch blocks.
