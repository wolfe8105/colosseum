# Stage 2 Outputs — spectate.render-messages.ts

## Agent 01

### renderMessages

When `renderMessages` is called with a `messages` array and a `SpectateDebate` object `d`, it reads the `messages` parameter and the `state.lastMessageTime` module-level variable. It iterates through each message in the array. For each message, it checks if the `round` property differs from the previously seen round; if so, it appends a round divider HTML element. It then determines the name to display: if `m.is_ai` is true, it uses the robot emoji with "AI"; otherwise it uses the debater name from `d` based on the message's `side` property (defaulting to side 'a'), or falls back to generic side labels. It calls `escHtml` on the side, name, and content to escape HTML entities. It constructs a message HTML element containing the escaped name, escaped content, and round number, wrapping these in divs with appropriate classes. After building the HTML for each message, it checks if `m.created_at` exists and is greater than `state.lastMessageTime`; if so, it updates `state.lastMessageTime` to the new timestamp. After processing all messages, it returns the accumulated HTML string.

### formatPointBadge

When `formatPointBadge` is called with a `ReplayPointAward` object `pa`, it reads the metadata from `pa.metadata`, and the `base_score` and `final_contribution` properties. It extracts numeric values: `base` from `meta.base_score` with fallback to `pa.base_score` defaulting to 0; `mult` from `meta.in_debate_multiplier` defaulting to 1.0; `flat` from `meta.in_debate_flat` defaulting to 0; and `final` from `meta.final_contribution` with fallback to `base`. It then computes two boolean flags: `hasMult` checks if `mult` differs from 1.0, and `hasFlat` checks if `flat` is nonzero. The function returns different formatted strings based on these flags: if neither modifier exists, it returns the base score with " pts"; if only a multiplier exists, it returns the base, multiplier, and final score in the format "+base × mult = final pts"; if only a flat modifier exists, it returns the final score with " pts"; if both modifiers exist, it returns all components in the format "+base × mult + flat = final pts". No module state is written and no other functions are called.

## Agent 02

### renderMessages

When called, this function takes a `messages` array of `DebateMessage` objects and a `SpectateDebate` object `d`, then builds and returns an HTML string. It initializes an empty `html` string and a `lastRound` variable set to 0, then iterates through each message in the `messages` array. For each message, it checks if the message has a `round` property and if that round differs from `lastRound`; if so, it appends an HTML div with the round number and updates `lastRound` to the new round value. It then extracts the message's `side` (defaulting to 'a'), determines the `name` by checking if the message is from an AI (using the robot emoji "🤖 AI") or otherwise using the corresponding debater name from `d` (debater_a_name or debater_b_name) or a fallback side label. It builds an HTML structure containing the name, message content (both HTML-escaped via `escHtml`), and round number, appending all of this to `html`. After the loop, it checks if the message has a `created_at` timestamp and compares it against the module-level state variable `state.lastMessageTime`; if `created_at` exists and is either greater than the stored time or the stored time does not exist, it updates `state.lastMessageTime`. The function returns the accumulated `html` string.

### formatPointBadge

When called with a `ReplayPointAward` object `pa`, this function reads its `metadata` property and extracts several numeric fields: `base_score` (defaulting to the top-level `base_score` or 0), `in_debate_multiplier` (defaulting to 1.0), `in_debate_flat` (defaulting to 0), and `final_contribution` (defaulting to `base_score`). It then evaluates two boolean flags: `hasMult` checks if the multiplier is not equal to 1.0, and `hasFlat` checks if the flat value is not equal to 0. The function returns one of four formatted strings based on these flags: if neither multiplier nor flat is present, it returns just the base score with "+ pts" suffix; if only multiplier is present, it returns the calculation "+base × mult = final pts"; if only flat is present, it returns "+final pts"; if both are present, it returns "+base × mult + flat = final pts". All numeric values are converted to strings via JavaScript's automatic type coercion in string concatenation.

## Agent 03

### renderMessages

When called with an array of `DebateMessage` objects and a `SpectateDebate` object, this function builds and returns an HTML string. It iterates through the messages array. For each message, it checks if the message's round number differs from the previously seen round; if it does, it appends a round divider div to the HTML string and updates the `lastRound` local variable. For each message, it constructs the sender's name by checking if the message is from an AI (rendered as "🤖 AI") or extracting the appropriate debater name from the `SpectateDebate` object, falling back to "Side A" or "Side B" if no name exists. It appends a message div containing the sender's name, the message content, and the round number, all escaped via the `escHtml` utility function. As it processes messages, if a message has a `created_at` timestamp and that timestamp is later than the current value of `state.lastMessageTime` (a module-level variable from the imported `state` object), it updates `state.lastMessageTime` to that timestamp. The function returns the accumulated HTML string after processing all messages.

### formatPointBadge

When called with a `ReplayPointAward` object, this function reads numeric values from the object's metadata and properties—specifically `base_score`, `in_debate_multiplier`, `in_debate_flat`, and `final_contribution`—with fallback logic that checks the metadata property first, then the direct property, then a default. It computes two boolean flags: `hasMult` checks whether the multiplier differs from 1.0, and `hasFlat` checks whether the flat adjustment is nonzero. Based on these flags, the function returns a formatted string badge with four possible patterns: if neither flag is true, it returns "+[base] pts"; if only the multiplier flag is true, it returns "+[base] × [mult] = [final] pts"; if only the flat flag is true, it returns "+[final] pts"; if both flags are true, it returns "+[base] × [mult] + [flat] = [final] pts". The function performs no mutations and makes no external calls.

## Agent 04

### renderMessages

When called with a messages array and a SpectateDebate object, this function reads the messages array and the module-level `state` object. It iterates through each message in the array. For each message, it checks if the message has a `round` property that differs from the `lastRound` variable (initially 0); if so, it appends a round divider HTML element to the accumulated html string and updates `lastRound`. It then constructs the speaker name by checking if the message is from an AI (using the robot emoji), and if not, falling back to debater names from the debate object or default side labels based on the `side` property (defaulting to 'a' if absent). It builds HTML div elements for the message container, name, content (all HTML-escaped), and round number, concatenating them to the html string. After building the HTML for each message, it reads the `state.lastMessageTime` property; if the message has a `created_at` timestamp and either `state.lastMessageTime` does not exist or the message timestamp is greater than it, the function updates `state.lastMessageTime` to the message timestamp. After processing all messages, the function returns the accumulated html string containing all rendered messages with round dividers.

### formatPointBadge

When called with a ReplayPointAward object, this function reads properties from the object's `metadata` field and directly from the object itself, extracting `base_score`, `in_debate_multiplier`, `in_debate_flat`, and `final_contribution` via nullish coalescing operators (defaulting to 0 for base/flat and 1.0 for multiplier, with final contribution defaulting to base_score). It coerces all these values to numbers. It then evaluates two boolean flags: `hasMult` (true if multiplier is not 1.0) and `hasFlat` (true if flat is not 0). The function then branches on these flags to determine which format string to return. If neither modifier is present, it returns the base score with ' pts' suffix; if only a multiplier is present, it returns the base, multiplier, and final contribution in the format '+base × mult = final pts'; if only a flat modifier is present, it returns the final score with ' pts' suffix; if both modifiers are present, it returns '+base × mult + flat = final pts'. The function always returns a string.

## Agent 05

### renderMessages

When called with a messages array and a SpectateDebate object, this function initializes an empty string for `html` and a `lastRound` variable set to 0, then iterates over each message in the messages array. For each message, it checks whether the message's round differs from `lastRound`; if so, it appends an HTML round divider with the round number to `html` and updates `lastRound` to the new round. Next, it determines the sender's name based on the message: if the message is marked as AI, the name is a robot emoji followed by "AI"; otherwise, it uses the side property (defaulting to 'a') to select either `debater_a_name` or `debater_b_name` from the debate object, falling back to "Side A" or "Side B" if those names are undefined. The function then appends a message div containing the sender's name (HTML-escaped via `escHtml`), the message content (HTML-escaped), and the round number. After constructing the full message HTML, it compares the message's `created_at` timestamp to the `state.lastMessageTime` module-level state variable; if the timestamp is present and either `lastMessageTime` is undefined or the new timestamp is greater, it updates `state.lastMessageTime` to the new value. Finally, the function returns the accumulated `html` string.

### formatPointBadge

When called with a ReplayPointAward object, this function extracts numeric values from the object's metadata and direct properties, reading `base_score`, `in_debate_multiplier`, `in_debate_flat`, and `final_contribution` fields with fallback values (0 for base and flat, 1.0 for multiplier, and base_score for final). It then creates boolean flags `hasMult` and `hasFlat` by testing whether the multiplier differs from 1.0 and whether the flat value is nonzero. The function then follows a four-way branch: if neither modifier is present, it returns a string containing the base score followed by " pts"; if only a multiplier is present, it returns a string showing the base score, the multiplication symbol, the multiplier, an equals sign, the final value, and " pts"; if only a flat bonus is present, it returns just the final value and " pts"; if both modifiers are present, it returns the base score, the multiplier, the flat bonus, an equals sign, the final value, and " pts". All numeric values are converted to strings for concatenation.
