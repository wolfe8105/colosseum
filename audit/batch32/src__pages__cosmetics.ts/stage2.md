# Stage 2 Outputs — cosmetics.ts

## Agent 01

### rerender
The function calls `_renderTab` (the imported `renderTab` function from cosmetics.render.ts) with five arguments: the `activeTab` module-level state variable, the `catalog` module-level state variable, the `isLoggedIn` module-level state variable, `activeTab` a second time, and a reference to `rerender` itself. It does not read or write any parameters of its own, does not return a value, and has no branching or loops.

### loadShop
This async function calls `showLoading(true)`, then calls the `safeRpc` function with the argument `'get_cosmetic_catalog'` and awaits the result, which is destructured into `data` and `error`. If either `error` is truthy or `data` is falsy, the function calls `showToast` with an error message and error level, calls `showLoading(false)`, and returns early. Otherwise, it assigns the returned `data` to the `catalog` module-level state variable, calls `showLoading(false)`, calls `renderShell()`, and calls `rerender()`. The function is asynchronous and awaits only the `safeRpc` call; the `showLoading` and `renderShell` calls are not awaited. It does not return a value.

### renderShell
The function reads the `isLoggedIn` and `activeTab` module-level state variables and the `tokenBalance` module-level state variable. It queries the DOM for an element with id `'cosmetics-app'` and returns early if the element is not found. Otherwise, it sets the `innerHTML` of that element to a multiline string containing HTML for a shop header, tab navigation, tab content area, and two modal overlays. The header conditionally includes a token display if `isLoggedIn` is true, otherwise a sign-in link. The tab navigation is generated by mapping over `TABS` and creating buttons with classes and attributes determined by comparing each tab's key to `activeTab`. After setting the innerHTML, the function queries for all elements matching `'.tab-btn'` and attaches a click listener to each; when clicked, each button updates the `activeTab` module-level state variable with its `data-tab` attribute (cast as a Category), toggles the `'active'` class on all tab buttons, updates their `'aria-selected'` attribute, and calls `rerender()`. The function then queries for elements with ids `'confirm-modal'`, `'modal-cancel'`, `'info-modal'`, and `'info-modal-close'` and attaches click listeners that call `closeConfirmModal()` or `closeInfoModal()` imported from cosmetics.modal.ts. The function does not return a value and has no async operations.

### showLoading
The function takes a boolean parameter `on`. It queries the DOM for elements with ids `'cosmetics-loader'` and `'cosmetics-app'` (using optional chaining, so it does not error if either is absent), and toggles the `'hidden'` class on the `'cosmetics-loader'` element based on the negation of `on`, and toggles the `'hidden'` class on the `'cosmetics-app'` element based on `on`. The function does not return a value and has no async operations or branching.

## Agent 02

### rerender
Calls `_renderTab` (the imported `renderTab` from `cosmetics.render.ts`) with five arguments: the current `activeTab`, the `catalog` array, the `isLoggedIn` boolean, `activeTab` again, plus `rerender` as a callback. The function returns `void`. This function has no branches, loops, or error paths.

### loadShop
An async function that first calls `showLoading(true)` to display a loading indicator, then awaits a call to `safeRpc<CosmeticItem[]>('get_cosmetic_catalog')`. If the returned `error` is truthy or `data` is falsy, it calls `showToast` with an error message, calls `showLoading(false)`, and returns early. On success, it assigns the returned `data` to the module-level `catalog` variable, calls `showLoading(false)`, then sequentially calls `renderShell()` and `rerender()`. The function is async and awaits only the `safeRpc` call.

### renderShell
Retrieves the DOM element with id `'cosmetics-app'` and returns early if it does not exist. Otherwise, it replaces the element's `innerHTML` with a template string containing the header, tab navigation, tab content area, and two modal overlays. The template includes conditional rendering: if `isLoggedIn` is true, it renders a token display showing `tokenBalance.toLocaleString()`; otherwise a sign-in link. Renders tab buttons by iterating over `TABS`, marking the button for `activeTab` as active. After inserting the HTML, it attaches click event listeners to all `.tab-btn` elements; each listener updates `activeTab`, toggles the `'active'` class, updates `aria-selected`, and calls `rerender()`. It also attaches click listeners to the confirm modal overlay, cancel button, info modal overlay, and close button that call `closeConfirmModal()` or `closeInfoModal()`. The function returns `void`.

### showLoading
Takes a boolean parameter `on`. It retrieves the element with id `'cosmetics-loader'` and calls `classList.toggle('hidden', !on)` on it, and retrieves the element with id `'cosmetics-app'` and calls `classList.toggle('hidden', on)` on it. When `on` is `true`, the loader becomes visible and the app becomes hidden; when `on` is `false`, the loader becomes hidden and the app becomes visible. Returns `void`.

## Agent 03

### rerender
Calls `_renderTab` (the imported `renderTab` function) with five arguments: the current `activeTab` category, the `catalog` module-level array of cosmetic items, the `isLoggedIn` boolean flag, the `activeTab` category again (unclear why passed twice), and a reference to the `rerender` function itself. It returns nothing and has no side effects beyond the effects of `_renderTab`.

### loadShop
Calls `showLoading(true)` to display the loading indicator, then makes an awaited call to `safeRpc<CosmeticItem[]>('get_cosmetic_catalog')`. If `error` is truthy or `data` is falsy, it calls `showToast` with `'Failed to load Armory. Please refresh.'` and `'error'`, then calls `showLoading(false)`, then returns early. If the RPC call succeeds, it assigns the returned `data` to the module-level `catalog` variable, calls `showLoading(false)`, calls `renderShell()`, and calls `rerender()`. The function is async and awaits only the `safeRpc` call.

### renderShell
Reads the module-level variables `isLoggedIn`, `tokenBalance`, and `activeTab`. Queries the DOM for an element with id `'cosmetics-app'` and returns early if not found. If found, sets the element's `innerHTML` to a large template string containing the shop header (conditionally renders token display or sign-in link based on `isLoggedIn`), a tab navigation bar (mapping over `TABS` and marking `activeTab`), an empty tab-content div, and two hidden modal overlays. Then queries all `.tab-btn` buttons and for each adds a click listener that sets `activeTab` to the button's `data-tab` attribute (cast as a `Category`), updates all tab buttons' `active` class and `aria-selected`, and calls `rerender()`. Finally, queries for the confirm modal, modal cancel button, info modal, and info modal close button by id and adds click listeners delegating to `closeConfirmModal` and `closeInfoModal`. All listeners are optional — they execute only if the elements exist.

### showLoading
Queries the DOM for elements with ids `'cosmetics-loader'` and `'cosmetics-app'` and toggles their `hidden` CSS class based on the `on` parameter. If `on` is true, the loader's `hidden` class is removed (showing it) and the app's `hidden` class is added (hiding it); if `on` is false, the opposite occurs. Both queries are optional — they execute only if the elements exist. Returns nothing.
