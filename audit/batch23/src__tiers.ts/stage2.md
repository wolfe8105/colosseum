# Stage 2 Outputs — src/tiers.ts

## Agent 01

### getTier
Reads the `questionsAnswered` parameter and the module-level constant `TIER_THRESHOLDS`. It coerces the parameter to a local `qa`: if `typeof questionsAnswered === 'number'` is true, `qa` is `questionsAnswered`, otherwise `qa` is `0`. It then iterates `TIER_THRESHOLDS` in array order (which is descending by `min`: 100, 75, 50, 25, 10, 0) and on the first entry `t` where `qa >= t.min`, it returns a new object that spreads `t` and adds `stakeCap: t.maxStake` and `questionsAnswered: qa`. If the loop completes without matching (the comment labels this as unreachable because the last threshold has `min: 0`), it returns a hardcoded fallback object `{ tier: 0, name: 'Unranked', icon: '🔒', maxStake: 0, stakeCap: 0, slots: 0, min: 0, questionsAnswered: qa }`. The function is synchronous, writes nothing to module-level state or external state, and calls no other functions.

### canStake
Reads the `questionsAnswered` parameter. It calls `getTier(questionsAnswered)` (see above entry) and returns the boolean result of comparing the returned object's `maxStake` against `0` with `> 0`. Given the thresholds in `TIER_THRESHOLDS`, this yields `false` when the resolved tier is `Unranked` (maxStake 0) and `true` for every other tier including `Spectator+` (maxStake 5) upward. There is no branching, no error path, no async work, and nothing is written.

### getPowerUpSlots
Reads the `questionsAnswered` parameter. It calls `getTier(questionsAnswered)` and returns the `slots` property of the returned `TierInfo` object. Per the `TIER_THRESHOLDS` entries, this returns `0` for tiers 0 and 1, `1` for tier 2, `2` for tier 3, `3` for tier 4, and `4` for tier 5. The function is synchronous, performs no branching beyond whatever `getTier` does internally, and writes nothing.

### getNextTier
Reads the `questionsAnswered` parameter and the module-level `TIER_THRESHOLDS`. It normalizes to `qa` using the same `typeof === 'number'` check as `getTier`, producing `0` if the argument is not a number. It then calls `getTier(qa)` to produce `current`. If `current.tier >= 5`, it returns `null` as an early exit. Otherwise it runs `TIER_THRESHOLDS.find(t => t.tier === (current.tier + 1 as TierLevel))` to locate the entry whose `tier` is one above the current tier; if `find` returns `undefined`, the function returns `null`. On a hit, it returns an object literal with `tier: next.tier`, `name: next.name`, `icon: next.icon`, `questionsNeeded: next.min - qa`, `totalRequired: next.min`, and `minQuestions: next.min`. Note that `questionsNeeded` is computed as a raw subtraction with no floor at zero — if `qa` exceeds `next.min` it will be negative, though reaching that state also means `current.tier` would equal or exceed `next.tier` and the early return would fire first. The function is synchronous and writes nothing externally.

### renderTierBadge
Reads the `questionsAnswered` parameter and the module-level `TIER_COLORS` record, and imports `escapeHTML` from `./config.ts`. It calls `getTier(questionsAnswered)` to obtain `t`, then looks up `TIER_COLORS[t.tier]` and uses `?? '#6b7280'` as the fallback when the lookup yields `undefined` or `null`. It then returns a single-line HTML string built by concatenation: `<span class="tier-badge" style="color:` + the resolved color + `; font-weight:600;">` + `escapeHTML(t.name)` + `</span>`. The tier name is passed through `escapeHTML`, but the color value interpolated into the `style` attribute is not escaped — it comes from the constant `TIER_COLORS` table or the hardcoded fallback, both of which contain only hex strings. The function is synchronous, returns a string, and does not touch the DOM itself.

### renderTierProgress
Reads the `questionsAnswered` parameter, the module-level `TIER_THRESHOLDS`, and the imported `escapeHTML`. It calls `getTier(questionsAnswered)` to produce `current`, then `getNextTier(questionsAnswered)` to produce `next`. If `next` is `null`, it returns the static string `'<div class="tier-progress-complete">Legend — Max Tier</div>'` as an early exit. Otherwise it runs `TIER_THRESHOLDS.find(t => t.tier === current.tier)` to locate `prevEntry`, and sets `prevMin = prevEntry?.min ?? 0` so a missing entry collapses to `0`. It computes `filled = questionsAnswered - prevMin` (using the raw parameter, not a normalized `qa`, so a non-number parameter would produce `NaN` here even though `getTier` and `getNextTier` internally normalized it), `band = next.totalRequired - prevMin`, and `pct = band > 0 ? Math.min(100, Math.round((filled / band) * 100)) : 0`. The `band > 0` guard avoids division by zero and also replaces any negative-band case with `0`. It then returns a concatenated HTML string containing a `div.tier-progress` wrapper with two children: a `div.tier-progress-label` whose text is `escapeHTML(String(questionsAnswered)) + '/' + escapeHTML(String(next.totalRequired)) + ' — ' + escapeHTML(String(next.questionsNeeded)) + ' more to unlock ' + escapeHTML(next.name)`, and a `div.tier-progress-bar` containing a `div.tier-progress-fill` whose inline style sets `width:` to the numeric `pct` followed by `%`. The `pct` value is interpolated directly without escaping; all four text fragments from user-facing values are passed through `escapeHTML`. The function is synchronous and does not write to the DOM or any module state.

## Agent 02

### getTier
Reads its single parameter `questionsAnswered` and the module-level `TIER_THRESHOLDS` constant. It first coerces the parameter: if `typeof questionsAnswered === 'number'` the value is kept in local `qa`, otherwise `qa` is set to `0`. It then iterates `TIER_THRESHOLDS` in declaration order (descending `min`: 100, 75, 50, 25, 10, 0) and for the first entry where `qa >= t.min` returns a new object built by spreading that threshold entry and adding two extra fields: `stakeCap` set to the entry's `maxStake`, and `questionsAnswered` set to `qa`. Control flow exits the loop on the first match via `return`. If no entry matches (the inline comment states this should never happen because tier 0 has `min: 0`), it falls through to a hardcoded fallback return: `{ tier: 0, name: 'Unranked', icon: '🔒', maxStake: 0, stakeCap: 0, slots: 0, min: 0, questionsAnswered: qa }`. It writes nothing to module-level or external state, calls no other functions, and is synchronous.

### canStake
Reads its `questionsAnswered` parameter and passes it directly to `getTier` on the anchor list; at this call site that produces a `TierInfo` whose `maxStake` field is read. The function returns the boolean expression `getTier(questionsAnswered).maxStake > 0`. It writes nothing, performs no branching of its own beyond the comparison, is synchronous, and calls no APIs other than `getTier`.

### getPowerUpSlots
Reads its `questionsAnswered` parameter and forwards it to `getTier` on the anchor list; at this call site that yields a `TierInfo` whose `slots` field is returned as a number. No module-level or external state is read or written, there are no branches, and the function is synchronous.

### getNextTier
Reads `questionsAnswered` and the module-level `TIER_THRESHOLDS`. It first coerces the parameter the same way `getTier` does: `qa` is the input if it is a `number`, otherwise `0`. It then calls `getTier(qa)` (an anchor-list function) to obtain `current`. First early return: if `current.tier >= 5` it returns `null`. Otherwise it runs `TIER_THRESHOLDS.find(t => t.tier === (current.tier + 1 as TierLevel))` to look up the entry whose `tier` is exactly one above the current tier, assigning the result to `next`. Second early return: if `next` is falsy it returns `null`. On the success path it returns a new object literal with `tier`, `name`, and `icon` copied from `next`, `questionsNeeded` computed as `next.min - qa`, and both `totalRequired` and `minQuestions` set to `next.min`. It is synchronous, writes nothing to module or external state, and its only function calls are `getTier` and `Array.prototype.find`.

### renderTierBadge
Reads `questionsAnswered` and the module-level `TIER_COLORS` map. It calls `getTier(questionsAnswered)` (anchor-list function) and stores the result as `t`. It then reads `TIER_COLORS[t.tier]`; if that lookup is nullish the nullish-coalescing operator substitutes the literal `'#6b7280'`, and the result is stored in `color`. It calls the imported `escapeHTML` (from `./config.ts`) on `t.name`. It returns a single concatenated string: `'<span class="tier-badge" style="color:' + color + '; font-weight:600;">' + escapeHTML(t.name) + '</span>'`. There are no branches beyond the `??` fallback, it is synchronous, and it writes nothing to module-level or external state (the caller is expected to insert the returned HTML).

### renderTierProgress
Reads `questionsAnswered` and the module-level `TIER_THRESHOLDS`. It calls `getTier(questionsAnswered)` (anchor-list function) and stores the result as `current`, then calls `getNextTier(questionsAnswered)` (anchor-list function) and stores the result as `next`. First branch: if `next` is falsy it returns the literal HTML string `'<div class="tier-progress-complete">Legend — Max Tier</div>'` immediately. Otherwise it runs `TIER_THRESHOLDS.find(t => t.tier === current.tier)` to find the threshold row for the current tier, assigns it to `prevEntry`, and computes `prevMin` as `prevEntry?.min ?? 0`. It then computes `filled = questionsAnswered - prevMin` and `band = next.totalRequired - prevMin`.

Percentage calculation branches on `band`: if `band > 0`, `pct` is `Math.min(100, Math.round((filled / band) * 100))`; otherwise `pct` is `0`. The function then calls the imported `escapeHTML` four times — once on `String(questionsAnswered)`, once on `String(next.totalRequired)`, once on `String(next.questionsNeeded)`, and once on `next.name` — and concatenates the results with literal markup to produce an HTML string containing a `.tier-progress` wrapper with a label (`"<filled>/<total> — <needed> more to unlock <name>"`) and a `.tier-progress-bar` whose inner `.tier-progress-fill` has `style="width:<pct>%"` (the numeric `pct` is interpolated directly, not passed through `escapeHTML`). The returned string is the function's only output. It is synchronous and writes nothing to module-level or external state.

## Agent 03

### getTier
Reads a single parameter `questionsAnswered`. It begins by checking whether the parameter's runtime type is `number`; if so it binds `qa` to that value, otherwise it binds `qa` to `0`. It then iterates the module-level constant `TIER_THRESHOLDS` in declared order (which is descending by `min`: 100, 75, 50, 25, 10, 0), and for each entry `t` it compares `qa >= t.min`. The first entry that satisfies the comparison causes the loop to return a new object that spreads all fields of `t` (`min`, `tier`, `name`, `icon`, `maxStake`, `slots`), then adds `stakeCap` set to the same value as `t.maxStake` and `questionsAnswered` set to `qa`. If the loop exits without a match, the function returns a hardcoded fallback object representing tier 0 (`name: 'Unranked'`, `icon: '🔒'`, `maxStake: 0`, `stakeCap: 0`, `slots: 0`, `min: 0`, `questionsAnswered: qa`). The function is synchronous, calls no other functions, reads no DOM or storage, and writes no module-level state.

### canStake
Reads the `questionsAnswered` parameter and passes it directly to `getTier`, then accesses `.maxStake` on the returned `TierInfo` object and returns the boolean result of `> 0`. With the module-level `TIER_THRESHOLDS` data, this produces `false` for `qa < 10` (tier 0 has `maxStake: 0`) and `true` for `qa >= 10` (tier 1 and above have positive `maxStake` values, with tier 5 being `Infinity`). The function is synchronous, performs no I/O, writes no state, and its only call is to `getTier`.

### getPowerUpSlots
Reads the `questionsAnswered` parameter, passes it to `getTier`, and returns the `.slots` field of the resulting `TierInfo`. Given the `TIER_THRESHOLDS` values, the returned number is 0 for tiers 0 and 1, 1 for tier 2, 2 for tier 3, 3 for tier 4, and 4 for tier 5. Synchronous, no side effects, no state writes, single call to `getTier`.

### getNextTier
Reads the `questionsAnswered` parameter, normalising non-number inputs to `0` into a local `qa`. It calls `getTier(qa)` and stores the result as `current`. If `current.tier >= 5` it returns `null` immediately. Otherwise it calls `Array.prototype.find` on `TIER_THRESHOLDS` looking for an entry whose `tier` equals `current.tier + 1` (cast to `TierLevel`); if `find` returns `undefined` the function returns `null`. When a matching `next` entry is found, it returns a new `NextTierInfo` object containing `tier`, `name`, and `icon` copied from `next`; `questionsNeeded` computed as `next.min - qa` (which can be zero or negative if `qa` has already reached or exceeded `next.min` without the tier promotion being reflected, though in normal use `qa < next.min` because otherwise `getTier` would have returned that higher tier); `totalRequired` set to `next.min`; and `minQuestions` also set to `next.min` as an alias. Synchronous, no side effects, calls `getTier` and `Array.prototype.find`.

### renderTierBadge
Reads `questionsAnswered`, calls `getTier(questionsAnswered)` and binds the result to `t`. It then looks up `TIER_COLORS[t.tier]`, applying the nullish-coalescing fallback `'#6b7280'` if the lookup is `undefined` or `null` (given the module-level `TIER_COLORS` map includes all six tier levels 0-5, the fallback is only reachable if `t.tier` somehow falls outside that set). It calls `escapeHTML(t.name)` from `./config.ts` to sanitize the tier name, then returns a string built by concatenation: `<span class="tier-badge" style="color:` + color + `; font-weight:600;">` + escaped name + `</span>`. Synchronous, returns an HTML string, does not insert into the DOM itself, writes no state.

### renderTierProgress
Reads `questionsAnswered`. It calls `getTier(questionsAnswered)` storing the result in `current`, then calls `getNextTier(questionsAnswered)` storing the result in `next`. If `next` is `null` (meaning the user is at tier 5), it returns the string `<div class="tier-progress-complete">Legend — Max Tier</div>` and exits. Otherwise it calls `Array.prototype.find` on `TIER_THRESHOLDS` looking for the entry whose `tier` matches `current.tier`, then reads `.min` from that entry via optional chaining with a `?? 0` fallback, binding the result as `prevMin`. It computes `filled = questionsAnswered - prevMin`, `band = next.totalRequired - prevMin`, and `pct` as: if `band > 0`, then `Math.min(100, Math.round((filled / band) * 100))`; otherwise `0`. It then constructs and returns an HTML string via concatenation. The string opens a `<div class="tier-progress">` wrapper containing a `<div class="tier-progress-label">` with three `escapeHTML(String(...))` calls — on `questionsAnswered`, on `next.totalRequired`, and on `next.questionsNeeded` — separated by literal `/` and `—` text with the message ` more to unlock `, followed by `escapeHTML(next.name)` and the closing label tag. The wrapper also contains a `<div class="tier-progress-bar">` with a nested `<div class="tier-progress-fill" style="width:` + `pct` + `%">`. Note that `pct` is concatenated as a raw number without `escapeHTML` — it is the only numeric value produced by `Math.round`/`Math.min` on internal arithmetic. Synchronous, returns an HTML string, does not touch the DOM, writes no state, calls `getTier`, `getNextTier`, `Array.prototype.find`, and `escapeHTML`.
