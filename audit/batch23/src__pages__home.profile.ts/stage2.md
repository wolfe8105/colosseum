# Stage 2 Outputs — src/pages/home.profile.ts

## Agent 01

### _renderAvatar
This synchronous function receives an `HTMLElement` and a `Profile` object. It reads `profile.avatar_url`, coalescing a missing value to the empty string and storing it in `url`. It branches on whether `url` begins with the prefix `emoji:`. In the emoji branch, it slices off the six-character prefix to obtain `emoji`, clears `el.textContent` by assigning the empty string, sets `el.style.fontSize` to `'32px'`, and assigns `el.innerHTML` to the result of calling `escapeHTML(emoji)` from `../config.ts` concatenated with the literal span markup `<span class="avatar-hint">✏️</span>`. In the else branch, it computes `initial` by indexing position `[0]` of the first truthy value among `profile.display_name`, `profile.username`, and the literal `'?'`, then calls `.toUpperCase()` on that character; it clears `el.textContent`, resets `el.style.fontSize` to the empty string, and assigns `el.innerHTML` to `escapeHTML(initial)` concatenated with the same span markup. It returns no value. The only external call is `escapeHTML`; there is no `try`/`catch` and no `await`.

### _renderNavAvatar
This synchronous function receives an `HTMLElement` and a `Profile` object, reads `profile.avatar_url` (defaulting to the empty string), and branches on whether that string starts with `emoji:`. In the emoji branch it writes `url.slice(6)` into `el.textContent` and sets `el.style.fontSize` to `'18px'`. In the else branch it writes the uppercased first character of `profile.display_name`, `profile.username`, or `'?'` (whichever is the first truthy value) into `el.textContent` and resets `el.style.fontSize` to the empty string. Unlike `_renderAvatar`, this function writes directly via `textContent` rather than `innerHTML`, calls no helpers, and emits no escape span. It returns no value and has no error handling.

### updateUIFromProfile
This exported synchronous function receives a `User | null` and a `Profile | null`. It returns early with no value if `profile` is falsy; the `user` parameter is not referenced anywhere in the body. Otherwise, it calls `document.getElementById('user-avatar-btn')` and passes the non-null-asserted result together with `profile` to `_renderNavAvatar`, writing the nav avatar's `textContent` and `fontSize`. It then calls `document.getElementById('profile-avatar')` and passes the result to `_renderAvatar`, writing that element's `textContent`, `fontSize`, and `innerHTML` with an escaped initial or emoji plus an edit-hint span. Both lookups use the `!` non-null assertion, so a missing DOM node will throw a `TypeError` at the callee's property access.

The function then writes `textContent` on a sequence of elements looked up by id: `profile-display-name` receives the uppercased display name / username / `'GLADIATOR'`; `profile-tier` and `dropdown-tier` receive a label resolved from the local `tierLabels` record keyed by `profile.subscription_tier || 'free'`, falling back to `'FREE TIER'` when the key is absent; `dropdown-name` receives the raw display name / username / `'Gladiator'`; `stat-elo` receives `profile.elo_rating || 1200` cast via `String`; `stat-wins`, `stat-losses`, `stat-streak`, and `stat-debates` receive the matching numeric profile fields cast via `String`, each with a `|| 0` fallback; `stat-tokens` and `token-count` receive `profile.token_balance || 0` formatted via `.toLocaleString()`. It then looks up `shop-token-balance` without `!` and, only if the element exists, writes the formatted token balance to its `textContent`. It computes `depth = profile.profile_depth_pct || 0`, writes `depth + '%'` to the `style.width` of `profile-depth-fill` (cast to `HTMLElement`), and writes an `innerHTML` string to `profile-depth-text` containing the template literal `` `Profile ${depth}% complete — <a href="moderator-profile-depth.html" style="color:var(--mod-text-heading);">unlock rewards</a>` ``; `depth` is interpolated without passing through `escapeHTML` or `Number()`. It then performs null-checked writes on a desktop-panel group (`dp-name`, `dp-tier`, `dp-avatar`, `dp-elo`, `dp-wins`, `dp-losses`, `dp-streak`, `dp-tokens`, `dp-depth-fill`, `dp-depth-pct`), each guarded by an `if` so a missing id is silently skipped; `dp-depth-fill` is cast to `HTMLElement | null` and assigns `style.width`. Finally it looks up `profile-bio-display`; if present, it reads `profile.bio || ''`, and if `bio.trim()` is truthy, writes `bio` to `textContent` and calls `classList.remove('placeholder')`, otherwise writes the literal `'Tap to add bio'` to `textContent` and calls `classList.add('placeholder')`. There is no `try`/`catch`; the function returns no value.

### loadFollowCounts
This exported `async` function takes no parameters. It calls `getCurrentUser()` imported from `../auth.ts` synchronously and assigns the result to `user`, then returns early with no value if `user?.id` is falsy. It enters a `try` block and `await`s `getFollowCounts(user.id)`, also imported from `../auth.ts`, assigning the resolved object to `counts`. It then writes `String(counts.followers || 0)` into the `textContent` of the element with id `profile-followers` and `String(counts.following || 0)` into the `textContent` of the element with id `profile-following`; both lookups use the `!` non-null assertion, so a missing element throws a `TypeError` that propagates into the `catch`. The `catch` block binds the thrown value to `e` and calls `console.warn('[Home] follow counts render failed:', e)`; no rethrow occurs and the promise resolves to `undefined`. The only awaited call is `getFollowCounts`; `getCurrentUser` is synchronous.

## Agent 02

### _renderAvatar
This synchronous function reads two parameters: `el` (an `HTMLElement`) and `profile` (a `Profile`). It reads `profile.avatar_url` into a local `url`, defaulting to the empty string when null/undefined. Control flow branches on whether `url` begins with the literal prefix `emoji:`. In the emoji branch, it slices the prefix off to produce `emoji`, writes an empty string to `el.textContent` (clearing any existing text node content), sets `el.style.fontSize` to `'32px'`, and then writes into `el.innerHTML` the result of concatenating `escapeHTML(emoji)` (called from the `config.ts` import) with the static HTML fragment `<span class="avatar-hint">✏️</span>`. In the else branch, it computes `initial` by reading `profile.display_name`, falling back to `profile.username`, falling back to the string `'?'`, taking index `[0]`, and uppercasing it; it then clears `el.textContent`, resets `el.style.fontSize` to the empty string, and writes `escapeHTML(initial)` plus the same pencil-hint span into `el.innerHTML`. It returns nothing. It is not async, has no try/catch, and makes no network or storage calls.

### _renderNavAvatar
This synchronous function reads parameters `el` (`HTMLElement`) and `profile` (`Profile`), and reads `profile.avatar_url` into `url` with an empty-string fallback. Control flow branches on whether `url` starts with `emoji:`. In the emoji branch it writes `url.slice(6)` to `el.textContent` and sets `el.style.fontSize` to `'18px'`. In the else branch it computes the first character of `profile.display_name` (falling back to `profile.username`, then to `'?'`), uppercases it, writes it to `el.textContent`, and resets `el.style.fontSize` to the empty string. Unlike `_renderAvatar`, it does not call `escapeHTML`, writes only to `textContent` (never `innerHTML`), and does not append a pencil-hint span. It returns nothing, is not async, has no try/catch, and makes no network or storage calls.

### updateUIFromProfile

Finally it looks up `profile-bio-display`; when present, it reads `profile.bio || ''` into `bio` and branches on `bio.trim()`: the truthy branch writes `bio` to `bioEl.textContent` and calls `bioEl.classList.remove('placeholder')`; the falsy branch writes the literal `'Tap to add bio'` to `bioEl.textContent` and calls `bioEl.classList.add('placeholder')`. It then writes `String(counts.followers || 0)` into `#profile-followers.textContent` and `String(counts.following || 0)` into `#profile-following.textContent`, using non-null assertions on both `getElementById` calls. If the awaited call or either DOM write throws, the catch block receives the error as `e` and calls `console.warn` with the literal prefix `'[Home] follow counts render failed:'` and the error; it does not rethrow. The function's returned promise resolves to `undefined` on both the happy path and the catch path, and to `undefined` on the early-return path.
