## Agent 01

### placeStake

`placeStake` is async and accepts `debateId` (string), `side` (string), and `amount` (number or string). It reads no module-level state at the point of entry. Its first branch checks whether `debateId` or `side` are falsy; if either is, it returns immediately with `{ success: false, error: 'Missing required fields' }` ŌĆö no network call is made. Next it coerces `amount` to an integer via `Number.parseInt(String(amount), 10)` and checks whether the result is `NaN` or non-positive; if so it returns `{ success: false, error: 'Amount must be a positive number' }`. It then calls `getBalance()` (imported from `tokens.ts`), which returns the cached token balance or `null`. If the balance is non-null and the parsed amount exceeds it, `placeStake` returns `{ success: false, error: 'Insufficient balance (N tokens)' }`, formatting the balance with `toLocaleString()`. Passing all three guards, it `await`s `safeRpc<StakeResult>('place_stake', { p_debate_id, p_side, p_amount })`. If `result.error` is set it returns `{ success: false, error: result.error.message ?? 'Stake failed' }`. Otherwise it returns `result.data`, falling back to `{ success: false, error: 'No response' }` if `data` is nullish. The function writes nothing to module-level state and makes no DOM calls.

### getPool

`getPool` is async and accepts a single `debateId` string. It reads no module-level state. It constructs an inline `emptyPool` constant of type `PoolData` with `exists: false`, zeroed totals, `pool_status: 'none'`, and `user_stake: null`. It immediately `await`s `safeRpc<PoolData>('get_stake_pool', { p_debate_id: debateId })`. There is no pre-call validation ŌĆö the call is made regardless of the value of `debateId`. If `result.error` is set, the function returns `emptyPool` without logging. If the call succeeds but `result.data` is nullish, it also returns `emptyPool`. Otherwise it returns `result.data`. The function writes nothing to module-level state, makes no DOM calls, and has no try/catch of its own ŌĆö error containment is entirely through the falsy check on `result.error`.

### settleStakes

`settleStakes` is async and accepts a single `debateId` string. It reads no module-level state and performs no pre-call validation. It `await`s `safeRpc<SettleResult>('settle_stakes', { p_debate_id: debateId })`. The RPC passes only the debate ID; per the inline comment, the server reads the winner from `arena_debates.winner` and applies a hardcoded multiplier ŌĆö no winner or multiplier argument is sent by the client. If `result.error` is set, it calls `console.error('[Staking] settle error:', result.error)` and returns `{ success: false, error: result.error.message }` ŌĆö note that `result.error.message` may be `undefined` if the error object lacks a `message` property, which would yield `{ success: false, error: undefined }`. If the call succeeds but `result.data` is nullish, it returns `{ success: false, error: 'No response' }`. Otherwise it returns `result.data`. The function writes nothing to module-level state and makes no DOM calls.

### getOdds

`getOdds` is synchronous. It accepts `totalA` and `totalB`, both numbers, and returns an `Odds` object. It reads no module-level state, makes no network calls, and touches no external APIs. It computes `total = totalA + totalB`. If `total === 0`, it returns immediately with `{ a: 50, b: 50, multiplierA: '2.00', multiplierB: '2.00' }`, skipping all further computation. Otherwise it computes `pctA` as `Math.round((totalA / total) * 100)` and derives `pctB` as `100 - pctA`, so the two percentages always sum to exactly 100 regardless of rounding. The multiplier for side A is computed as `(total / totalA).toFixed(2)` if `totalA > 0`, or the string `'Ōł×'` if `totalA` is zero. The same logic applies symmetrically to side B with `totalB`. It returns `{ a: pctA, b: pctB, multiplierA: multA, multiplierB: multB }`. There are no loops, no try/catch blocks, and no mutations of any external state.
