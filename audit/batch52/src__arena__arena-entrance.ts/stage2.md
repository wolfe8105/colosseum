# Stage 2 Outputs — arena-entrance.ts

## Agent 01

### _getTier

The function reads three parameters: `wins`, `losses`, and `debatesCompleted`. It returns a numeric literal of type 1, 2, or 3 representing the tier. The function has no side effects on module-level state or external state. Control flow branches based on the debate completion count and win rate. If `debatesCompleted` is less than 5, it immediately returns 1. Otherwise, it calculates the total number of debates by adding wins and losses; if this total is zero, it returns 1. For non-zero totals, it computes the win rate as `wins / total`, then returns 3 if the win rate exceeds 50%, returns 2 if the win rate exceeds 25%, and returns 1 if neither condition is met. The function is synchronous.

### playEntranceSequence

The function is asynchronous and returns a Promise that resolves to void. It reads the `debate` parameter (a CurrentDebate object) and calls `getCurrentProfile()` to read the current user's profile, extracting wins, losses, debates_completed, display_name, username, and elo_rating from the profile (with fallback defaults). It also reads debate properties including opponentName, opponentElo, mode, ranked, and topic. The function creates a DOM element with className and id set to 'ent-stage', appends it to document.body, and eventually removes it from the DOM. It calls `injectEntranceCSS()` immediately with no parameters. It calls `_getTier()` with the extracted wins, losses, and completed counts to determine which tier renderer to invoke. Based on the tier value (1, 2, or 3), it calls exactly one of three render functions: `renderTier1()`, `renderTier2()`, or `renderTier3()`, each receiving the stage element and various profile and debate properties. The function calls `playSound('roundStart')` in a try-catch block; if tier equals 3, it schedules a second `playSound('roundStart')` call via setTimeout after 600 milliseconds, wrapped in its own try-catch. Sound failures are silently ignored. After 2450 milliseconds, the function schedules a second setTimeout that sets the stage element's transition style to 'opacity 0.15s', opacity to '0', then schedules a third setTimeout that removes the stage element from the DOM and resolves the promise after 160 additional milliseconds. The entire function is wrapped in a Promise constructor with a resolve callback, making all timeout operations asynchronous and non-blocking.
