# Stage 2 Outputs — arena-sounds-core.ts

## Agent 01

### getCtx

When called, checks if the module-level `_ctx` variable is null. If null, attempts to instantiate an AudioContext by calling the constructor on `window.AudioContext` or falling back to `window.webkitAudioContext`. If construction throws any exception, logs a warning to the console and returns null immediately without assigning to `_ctx`. If construction succeeds and `_ctx` is now set, checks whether `_ctx.state` equals the string `'suspended'`; if so, calls `_ctx.resume()`. After the check completes, returns the `_ctx` variable (which is either an AudioContext instance or null from a prior failed instantiation attempt). This function is not async, but the internal `_ctx.resume()` method is async; no awaiting occurs here.

### sfxEnabled

When called, reads the value at the `'moderator_settings'` key from localStorage. If the value is null or missing, uses the string `'{}'` as a fallback. Parses the result using `JSON.parse()`. If parsing throws any exception, returns the boolean true immediately. If parsing succeeds, accesses the `audio_sfx` property of the parsed object and returns the result of evaluating `s.audio_sfx !== false` (which is true if the property is absent, undefined, true, or any value other than the literal false). This function is not async.

### osc

When called, receives an AudioContext, an OscillatorType, a numeric frequency, numeric start and end times, and a numeric gain value. Creates an Oscillator by calling `ctx.createOscillator()` and a Gain node by calling `ctx.createGain()`. Sets the oscillator's type property to the received type parameter, sets its frequency.value to the received freq parameter, and schedules the gain node to ramp down exponentially: starting at the provided gain value at time `ctx.currentTime + start`, ramping to 0.001 at time `ctx.currentTime + end`. Connects the oscillator to the gain node, then connects the gain node to `ctx.destination`. Calls `o.start()` with the time `ctx.currentTime + start` and `o.stop()` with the time `ctx.currentTime + end + 0.05`. Returns undefined. This function is not async.

### noise

When called, receives an AudioContext, numeric start and duration times, and a numeric gain value. Calculates a buffer size by multiplying `ctx.sampleRate` by the duration parameter, then flooring the result. Creates a one-channel audio buffer with the calculated size at the context's sample rate via `ctx.createBuffer()`. Retrieves the channel data array from the buffer. Loops through every index from 0 to bufSize (exclusive), writing a random value between approximately -0.3 and 0.3 to each array position (generated by `Math.random() * 2 - 1` multiplied by 0.3). Creates a BufferSource via `ctx.createBufferSource()` and assigns the buffer to its buffer property. Creates a Gain node via `ctx.createGain()`. Schedules the gain to start at the provided gain value at time `ctx.currentTime + start` and ramp exponentially to 0.001 at time `ctx.currentTime + start + duration`. Connects the source to the gain node, then connects the gain to `ctx.destination`. Calls `src.start()` with time `ctx.currentTime + start` and `src.stop()` with time `ctx.currentTime + start + duration + 0.05`. Returns undefined. This function is not async.

## Agent 02

### getCtx

Reads the module-level variable `_ctx` and `window.AudioContext` (or `window.webkitAudioContext` as a fallback). If `_ctx` is null, attempts to instantiate a new AudioContext using a try-catch block; if instantiation fails, logs a warning via `console.warn()` and returns null. If `_ctx` exists and its state is 'suspended', calls the resume method on it. Returns `_ctx`, which may be an AudioContext instance or null. This function is synchronous and produces a side effect of calling resume() on a suspended context.

### sfxEnabled

Reads from `localStorage` using the key 'moderator_settings', defaulting to the string '{}' if not found. Parses the retrieved string as JSON inside a try-catch block. Returns false only if the parsed object's `audio_sfx` property is explicitly false; all other outcomes (including property absence, parse errors, or localStorage errors) return true. This function is synchronous and reads external client-side persistent storage.

### osc

Takes six parameters: an AudioContext, an OscillatorType, a frequency number, a start time offset, an end time offset, and a gain value. Creates an oscillator node by calling `ctx.createOscillator()` and a gain node by calling `ctx.createGain()`. Sets the oscillator's type and frequency properties. Schedules the gain node to begin at a given value using `setValueAtTime()`, then schedules an exponential ramp down to 0.001 using `exponentialRampToValueAtTime()`. Connects the oscillator to the gain node, then the gain node to the context's destination. Starts the oscillator at the specified time and stops it 0.05 seconds after the end time. The function returns void and is synchronous; it schedules Web Audio API playback but does not await completion.

### noise

Takes four parameters: an AudioContext, a start time offset, a duration, and a gain value. Calculates a buffer size based on the context's sample rate and the duration parameter. Creates a new audio buffer with one channel using `ctx.createBuffer()`. Populates the buffer's channel data with random values (each multiplied by 0.3) using a for loop that iterates from 0 to bufSize. Creates a buffer source node and assigns the buffer to it. Creates a gain node, sets its gain to start at the provided value using `setValueAtTime()`, then schedules an exponential ramp to 0.001 at the start time plus duration using `exponentialRampToValueAtTime()`. Connects the source to the gain node and the gain node to the destination. Starts the source at the specified time and stops it 0.05 seconds after the start plus duration. The function returns void and is synchronous.

## Agent 03

### getCtx

When getCtx is called, it reads the module-level state variable `_ctx` (initially null). If `_ctx` is null, it attempts to construct an AudioContext via `new (window.AudioContext || (window as any).webkitAudioContext)()`, storing the result in `_ctx`. If the constructor throws an exception, the catch block logs a warning to console and returns null without storing anything in `_ctx`. After initialization (or if already initialized), the function checks if the AudioContext state is "suspended" and calls `resume()` if true. Finally, it returns either the initialized AudioContext or null. This function is synchronous and side-effects include potentially modifying `_ctx` and calling `resume()` on the AudioContext.

### sfxEnabled

When sfxEnabled is called, it attempts to read the `moderator_settings` item from localStorage. If the item does not exist, it defaults to an empty JSON object string. It then parses the retrieved value as JSON; if parsing throws an exception, the catch block immediately returns true. If parsing succeeds, it accesses the `audio_sfx` property of the parsed object and returns the negation of whether that property is strictly false. This function is synchronous and has no side effects beyond reading from localStorage.

### osc

When osc is called with an AudioContext, OscillatorType, frequency, start time offset, end time offset, and gain value, it creates a new oscillator via `ctx.createOscillator()` and a new gain node via `ctx.createGain()`. The oscillator type is set to the provided type parameter, and its frequency is set to the provided freq value. The gain node's gain is set to the provided gain value at time `ctx.currentTime + start`, then scheduled to exponentially ramp down to 0.001 at time `ctx.currentTime + end`. The oscillator connects to the gain node, which connects to the AudioContext destination. The oscillator starts at `ctx.currentTime + start` and stops at `ctx.currentTime + end + 0.05`, giving a 50ms tail beyond the scheduled end time. This function is synchronous and has no return value.

### noise

When noise is called with an AudioContext, start time offset, duration, and gain value, it calculates a buffer size by multiplying the sample rate by duration, then creates an audio buffer with that size in a single channel at the AudioContext's sample rate. It fills the buffer channel data with random samples in the range [-0.3, 0.3] via a for-loop from index 0 to bufSize. It creates a buffer source, assigns the buffer to it, and creates a gain node. The gain node's gain is set to the provided gain value at time `ctx.currentTime + start`, then scheduled to exponentially ramp down to 0.001 at time `ctx.currentTime + start + duration`. The buffer source connects to the gain node, which connects to the AudioContext destination. The buffer source starts at `ctx.currentTime + start` and stops at `ctx.currentTime + start + duration + 0.05`, providing a 50ms tail. This function is synchronous and has no return value.

## Agent 04

### getCtx

When called, `getCtx` checks whether the module-level variable `_ctx` (initially `null`) holds an AudioContext. If not, it attempts to instantiate one via `new (window.AudioContext || (window as any).webkitAudioContext)()`, catching any error and logging a warning `[Sounds] Web Audio API not available` before returning `null` on failure. If `_ctx` is successfully created or already exists, it reads `_ctx.state` and, if equal to `'suspended'`, calls `_ctx.resume()` to unsuspend the context. It returns the AudioContext or `null`. The function is synchronous.

### sfxEnabled

When called, `sfxEnabled` reads the `localStorage` item keyed `'moderator_settings'`, defaulting to `'{}'` if missing. It parses that string as JSON into object `s`, catching any parse error and returning `true` on failure. It reads the property `s.audio_sfx` and returns `true` if the value is not strictly `false`. The function is synchronous.

### osc

When called, `osc` receives an AudioContext, oscillator type string, frequency number, start time, end time (both relative offsets from `ctx.currentTime`), and gain value. It creates an Oscillator node `o` and a Gain node `g` via `ctx.createOscillator()` and `ctx.createGain()`. It writes the oscillator type to `o.type`, sets `o.frequency.value` to the `freq` parameter, and schedules two gain ramps: `setValueAtTime(gain, ctx.currentTime + start)` at the start moment and `exponentialRampToValueAtTime(0.001, ctx.currentTime + end)` to decay to near-silence by the end time. It connects the oscillator to the gain node and the gain node to `ctx.destination`. Finally, it calls `o.start(ctx.currentTime + start)` and `o.stop(ctx.currentTime + end + 0.05)`, adding 50ms to the stop time as padding. The function is synchronous and returns `void`.

### noise

When called, `noise` receives an AudioContext, start time offset, duration, and gain value. It calculates `bufSize` as the floor of `ctx.sampleRate * duration`. It creates an AudioBuffer with one channel, that many samples, and the context's sample rate via `ctx.createBuffer(1, bufSize, ctx.sampleRate)`. It reads `buf.getChannelData(0)` to obtain a typed array and populates each sample with a random value: `(Math.random() * 2 - 1) * 0.3`. It creates a BufferSource node `src` and a Gain node `g`, assigns `buf` to `src.buffer`, schedules gain ramps identical to `osc` (value at start, exponential decay to 0.001 by duration end), and connects source through gain to `ctx.destination`. Finally, it calls `src.start(ctx.currentTime + start)` and `src.stop(ctx.currentTime + start + duration + 0.05)`. The function is synchronous and returns `void`.

## Agent 05

### getCtx

Reads module-level state `_ctx` (initially null). If `_ctx` is null, attempts to instantiate an `AudioContext` via `new (window.AudioContext || (window as any).webkitAudioContext)()`. On constructor exception, logs a warning to console and returns null; on success, stores the context in `_ctx`. If `_ctx` is already set, skips instantiation. After instantiation or on subsequent calls, checks if `_ctx.state` is `'suspended'` and calls `_ctx.resume()` if true. Returns the `AudioContext` or null. Not async.

### sfxEnabled

Reads from `localStorage` using key `'moderator_settings'`, defaulting to `'{}'` if absent. Parses the retrieved string as JSON. Reads the `audio_sfx` property from the parsed object. Returns false if `audio_sfx` is explicitly false; otherwise returns true. On JSON parse exception, catches and returns true. Not async.

### osc

Takes an `AudioContext`, `OscillatorType`, and three numeric parameters: `freq`, `start`, and `end` (interpreted as time offsets in seconds), plus `gain`. Creates an oscillator via `ctx.createOscillator()` and a gain node via `ctx.createGain()`. Sets oscillator type and frequency to `freq`. Schedules gain to jump to the `gain` value at time `ctx.currentTime + start`, then exponential ramp to 0.001 at time `ctx.currentTime + end`. Connects oscillator to gain node, then gain node to `ctx.destination`. Starts the oscillator at `ctx.currentTime + start` and stops at `ctx.currentTime + end + 0.05`. Returns undefined (void). Not async.

### noise

Takes an `AudioContext` and three numeric parameters: `start` (time offset), `duration`, and `gain`. Computes a buffer size as `Math.floor(ctx.sampleRate * duration)`. Creates a mono audio buffer via `ctx.createBuffer(1, bufSize, ctx.sampleRate)` and retrieves its channel data. Fills the channel with random values between -0.3 and 0.3 via `(Math.random() * 2 - 1) * 0.3` loop. Creates a buffer source via `ctx.createBufferSource()` and assigns the buffer to it. Creates a gain node and schedules gain to jump to the `gain` value at `ctx.currentTime + start`, then exponential ramp to 0.001 at `ctx.currentTime + start + duration`. Connects buffer source to gain node, then gain to `ctx.destination`. Starts the source at `ctx.currentTime + start` and stops at `ctx.currentTime + start + duration + 0.05`. Returns undefined (void). Not async.
