## Agent 01

### renderLoadoutPicker

`renderLoadoutPicker` is an `async` function that accepts three parameters: `container` (an `HTMLElement` into which it writes all output), `debateId` (a string used when persisting selections), and `initialRefs` (an optional array of reference ID strings, defaulting to an empty array). It has no module-level state of its own; all persistent state across the inner render cycle lives in the `selected` `Set` declared in its local scope.

On entry, it immediately sets `container.innerHTML` to a loading message. It then calls `safeRpc<ArsenalReference[]>('get_my_arsenal', {})` and awaits the result. If the call returns an error, returns no data, or throws, `arsenal` is set to an empty array. If it succeeds, `arsenal` is assigned the returned data cast to `ArsenalReference[]`. After that network call, if `arsenal` is empty, the function replaces `container.innerHTML` with a "no references forged" message and returns early. When `arsenal` is non-empty, it filters out any entries whose `challenge_status` is `'frozen'`, then sorts the remaining entries descending by `current_power`, with ties broken by descending `created_at` timestamp via `Date.parse`. It then constructs `selected`, a `Set<string>`, by taking `initialRefs`, filtering down to IDs that exist in the fetched arsenal, and capping the result at five entries.

The function defines an inner function `render()` (synchronous, no return value) and calls it once. `render()` builds an HTML string for the full picker UI: a header row showing the current selection count as `${selected.size}/5`, followed by a grid of cards, one per entry in `arsenal`. For each card it reads `SOURCE_TYPES` (imported constant map) to look up a label for `ref.source_type`, reads `RARITY_COLORS` (imported constant map) to look up a border color for `ref.rarity`, calls `powerDisplay(ref)` (imported from `reference-arsenal.utils`) to produce the power string, and calls `escapeHTML()` on all user-supplied string fields (`ref.id`, `ref.claim_text`, `ref.source_title`, `ref.source_author`, `srcInfo?.label || ref.source_type`) before interpolating them into the markup. A card receives the CSS class `selected` if its ID is in `selected`, and `disabled` if it is not selected and `selected.size` is already 5. Selected cards additionally receive a checkmark element. After writing the full HTML string to `container.innerHTML`, `render()` queries `container` for all `.ref-loadout-card` elements that do not also have `.disabled`, and attaches a `click` event listener to each. Each listener reads `dataset.refId` from the clicked element; if absent, it returns immediately. If the ID is already in `selected`, it removes it; otherwise, if `selected.size` is less than 5, it adds it. Either branch then calls `render()` recursively, which re-renders the entire grid into `container` and re-attaches all listeners. After calling `render()`, the listener calls `saveDebateLoadout(debateId, Array.from(selected))` as a fire-and-forget promise; any rejection is caught by `.catch()` and logged to the console as a warning. `renderLoadoutPicker` itself returns `Promise<void>` and resolves after the initial `render()` call completes.
