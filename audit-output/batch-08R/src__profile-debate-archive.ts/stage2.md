# Stage 2 Outputs — src/profile-debate-archive.ts

## Agent 01

### _injectCSS

`_injectCSS` takes no parameters. On entry it reads the module-level boolean `_cssInjected`. If that flag is already `true`, the function returns immediately without any side effect. If `_cssInjected` is `false`, it sets `_cssInjected` to `true`, creates a new `<style>` element via `document.createElement('style')`, assigns a static CSS string to that element's `textContent`, and appends the element to `document.head`. The function has no return value. After the first call succeeds, all subsequent calls are no-ops.

### _archiveUrl

`_archiveUrl` accepts a single `ArchiveEntry` object. It reads the `debate_mode` field from that entry. If `debate_mode` equals the string `'ai'`, it returns the string `/moderator-auto-debate.html?id=` with the entry's `debate_id` URI-encoded via `encodeURIComponent`. For any other value of `debate_mode` (including `null` or any non-`'ai'` string), it returns `/moderator-spectate.html?id=` with the same encoded `debate_id`. The function touches no module-level state and makes no DOM mutations or network calls.

### _filtered

`_filtered` takes no parameters and returns a new array of `ArchiveEntry` objects. It reads the module-level arrays and strings `_entries`, `_filterCat`, `_filterResult`, and `_filterSearch`. It calls `Array.prototype.filter` on `_entries`, evaluating each entry `e` against up to four conditions in sequence. First, if `_filterCat` is not `'all'`, any entry whose `category` (defaulting to `'general'` when null) does not equal `_filterCat` is excluded. Second, if `_filterResult` is `'win'`, entries where `is_win` is falsy are excluded. Third, if `_filterResult` is `'loss'`, entries where `is_win` is truthy are excluded. Fourth, if `_filterSearch` is a non-empty string, it lowercases that string into `q`, then lowercases the entry's `topic`, its `opponent_name` or `opponent_username` (whichever is non-null first, falling back to empty string), and its `custom_name`. If none of those three lowercase strings includes `q` as a substring, the entry is excluded. An entry that passes all applicable conditions is included in the returned array. The function writes no state and makes no network calls.

### _renderTable

`_renderTable` accepts one parameter, an `HTMLElement` named `container`. It reads the module-level variables `_entries`, `_isOwner`, `_filterResult`, `_filterCat`, and `_filterSearch`. It also calls `escapeHTML` (aliased locally as `esc`) and calls `_filtered()` to obtain the currently visible rows. It computes a deduplicated list of category strings (`cats`) from all entries in `_entries` (not just the filtered rows), defaulting null categories to `'general'`. It builds an HTML string in sections. The header section always contains a title div and, if `_isOwner` is `true`, an add button with id `dba-add-btn`. The filters section always contains a text search input pre-filled with the current `_filterSearch` value (escaped), three result chips (ALL / W / L) with the active class applied to the one matching `_filterResult`, and one chip per unique category with the active class applied to the one matching `_filterCat`, plus an "ALL CATS" chip if `cats` is non-empty. If the filtered rows array is empty, an empty-state div is appended. When `_entries` is also empty and `_isOwner` is true, the message prompts the owner to add. When `_entries` is empty and `_isOwner` is false, it says "No debates in this archive." When `_entries` is non-empty but no rows passed the filter, it says "No debates match your filters." If the filtered rows array is non-empty, a scrollable table is built. For each entry in the filtered rows, the function computes: a locale-formatted date from `debate_created_at`; a result string based on whether `winner` is null or `is_win` is true; escaped opponent, topic/custom_name, description, and category labels; score strings formatted to one decimal place or `'—'` when null; a lock badge span if owner and `hide_from_public` is true; and `data-entry` and `data-url` attributes (with `_archiveUrl(e)` providing the URL, then escaped). When `_isOwner` is true, each row also contains three action buttons (`edit`, `toggle-hide`, `remove`). After the HTML string is fully assembled it is assigned to `container.innerHTML`. Finally, `_wireTable(container)` is called to attach event listeners. `_renderTable` has no return value and makes no network calls.

### _wireTable

`_wireTable` accepts one parameter, an `HTMLElement` named `container`. It makes no network calls and writes no module-level state directly, but it attaches event listeners that read and write module-level state and call other functions. It queries `#dba-search` within the container; if found, adds an `input` event listener that writes the input element's value to `_filterSearch` and calls `_renderTable(container)`. It queries all `.dba-chip[data-result]` elements and attaches a `click` listener to each that writes the chip's `data-result` dataset value to `_filterResult` and calls `_renderTable(container)`. It queries all `.dba-chip[data-cat]` elements and attaches a `click` listener to each that writes the chip's `data-cat` value to `_filterCat` and calls `_renderTable(container)`. It queries all `.dba-row` elements and attaches a `click` listener to each; that listener first checks whether the click's target is inside a `.dba-actions` element and returns immediately if so; otherwise it reads `row.dataset.url` and, if that URL exists, calls `showAdInterstitial` with a callback that performs `window.open(url, '_blank')`. It queries `#dba-add-btn` and, if found, attaches a `click` listener that calls `_showAddPicker(container)` and discards the returned promise with `void`. It queries all `[data-action]` elements and attaches a `click` listener to each; each listener calls `e.stopPropagation()` to prevent the click from bubbling to the row listener, reads `action` and `entryId` from the button's dataset, looks up the matching entry in `_entries` by `entry_id`; if no entry is found it returns; if `action` is `'edit'` it calls `_showEditSheet(entry, container)`; if `action` is `'toggle-hide'` it calls `_toggleHide(entry, container)` discarding the promise; if `action` is `'remove'` it calls `_removeEntry(entry, container)` discarding the promise. `_wireTable` has no return value.

### _showAddPicker

`_showAddPicker` accepts one parameter, an `HTMLElement` named `container`, and is an async function returning `Promise<void>`. It immediately makes a network call via `safeRpc<RecentDebate[]>('get_my_recent_debates_for_archive', { p_limit: 30 })`. If that call returns an error, it calls `showToast('Could not load debates', 'error')` and returns, making no DOM changes. If successful, the `rows` data is cast to `RecentDebate[]` (defaulting to an empty array if null). The function creates a div element with class `dba-picker-overlay` and sets its `innerHTML` to a bottom-sheet overlay. The sheet contains a drag handle, a title, and a list of `.dba-picker-row` elements; if the list is empty, a message is shown. For each row, the display shows the escaped topic, escaped opponent name, locale-formatted date, and a result indicator computed by checking `d.winner === undefined` (the `RecentDebate` type does not have a `winner` field, so this check will always be true, producing no result indicator). The function appends this overlay to `document.body`. A `click` listener on the overlay itself calls `overlay.remove()` when the click target is the overlay element directly. After appending, it queries all `.dba-picker-row` elements and attaches an async `click` listener to each. That listener removes the overlay from the DOM, reads `debateId` from the row's dataset, and calls `safeRpc('add_debate_to_archive', { p_debate_id: debateId })`. If that call returns an error, it calls `showToast('Could not add debate', 'error')` and returns. On success it calls `showToast('Added to archive', 'success')` and then awaits `_loadAndRender(container)`. The function does not modify any module-level state directly.

### _showEditSheet

`_showEditSheet` accepts two parameters: an `ArchiveEntry` named `entry` and an `HTMLElement` named `container`. It is synchronous and returns `void`. It creates a div element with class `dba-picker-overlay` and sets its `innerHTML` to a bottom-sheet form containing a text input for a custom name (pre-filled with `entry.custom_name` escaped, or an empty string if null) with max length 80, a text input for a custom description (pre-filled with `entry.custom_desc` escaped, or empty string if null) with max length 200, a checkbox for hide-from-public (checked if `entry.hide_from_public` is true), and two buttons: CANCEL and SAVE. The overlay is appended to `document.body`. A `click` listener on the overlay backdrop calls `overlay.remove()`. A `click` listener is attached to the CANCEL button and simply calls `overlay.remove()`. A `click` listener is attached to the SAVE button. When triggered, it reads the trimmed value of the name input (treating an empty string as `null`), the trimmed value of the description input (treating an empty string as `null`), and the checked state of the hide checkbox. It then disables the save button and sets its text to `'…'`. It awaits `safeRpc('update_archive_entry', { p_entry_id: entry.entry_id, p_custom_name: name, p_custom_desc: desc, p_hide_from_public: hide })`. If that call returns an error, it calls `showToast('Could not save', 'error')`, re-enables the save button, restores its text to `'SAVE'`, and returns. On success it removes the overlay, calls `showToast('Saved', 'success')`, and awaits `_loadAndRender(container)`. The function does not directly modify module-level state.

### _toggleHide

`_toggleHide` accepts an `ArchiveEntry` and an `HTMLElement` container. It is async and returns `Promise<void>`. It computes `newHide` as the boolean negation of `entry.hide_from_public`. It awaits `safeRpc('update_archive_entry', { p_entry_id: entry.entry_id, p_hide_from_public: newHide })`. If that call returns an error, it calls `showToast('Could not update', 'error')` and returns. On success it calls `showToast` with the message `'Hidden from public'` and level `'info'` when `newHide` is true, or `'Now visible'` and `'info'` when `newHide` is false. It then awaits `_loadAndRender(container)`. The function does not directly modify module-level state; all state changes flow through `_loadAndRender`.

### _removeEntry

`_removeEntry` accepts an `ArchiveEntry` and an `HTMLElement` container. It is async and returns `Promise<void>`. It first calls the synchronous browser built-in `confirm('Remove this debate from your archive?')`. If the user dismisses the dialog (confirm returns false), the function returns immediately with no network call and no DOM change. If the user confirms, it awaits `safeRpc('remove_from_archive', { p_entry_id: entry.entry_id })`. If that call returns an error, it calls `showToast('Could not remove', 'error')` and returns. On success it calls `showToast('Removed from archive', 'info')` and then awaits `_loadAndRender(container)`. The function does not directly modify module-level state.

### _loadAndRender

`_loadAndRender` accepts one parameter, an `HTMLElement` named `container`. It is async and returns `Promise<void>`. It awaits `safeRpc<ArchiveEntry[]>('get_my_debate_archive', {})`. If that call returns an error, it assigns `'<div class="dba-empty">Could not load archive.</div>'` to `container.innerHTML` and returns, leaving `_entries` unchanged. If the call succeeds, it writes the result (coerced from null to an empty array, then cast to `ArchiveEntry[]`) to the module-level `_entries`. It then calls `_renderTable(container)`. `_loadAndRender` has no return value.

### loadDebateArchive

`loadDebateArchive` is an exported async function that accepts an `HTMLElement` named `container` and an optional boolean `isOwner` defaulting to `false`. It returns `Promise<void>`. It calls `_injectCSS()`, which is a no-op after the first call. It writes `isOwner` to the module-level `_isOwner`. It resets `_entries` to an empty array, `_filterCat` to `'all'`, `_filterResult` to `'all'`, and `_filterSearch` to an empty string. It then sets `container.innerHTML` to a loading placeholder string. If `isOwner` is `true`, it awaits `_loadAndRender(container)`, which makes a network call to `get_my_debate_archive`, writes `_entries`, and calls `_renderTable`. If `isOwner` is `false`, it skips the network call entirely and sets `container.innerHTML` to `'<div class="dba-empty">Archive unavailable.</div>'`, leaving `_entries` as the empty array that was just set. The function does not call `getCurrentUser` despite it being imported; it relies on the caller to determine ownership.

### loadPublicDebateArchive

`loadPublicDebateArchive` is an exported async function that accepts an `HTMLElement` named `container` and a `string` named `userId`. It returns `Promise<void>`. It calls `_injectCSS()`. It writes `false` to `_isOwner`. It resets `_entries` to an empty array, `_filterCat` to `'all'`, `_filterResult` to `'all'`, and `_filterSearch` to an empty string. It sets `container.innerHTML` to the loading placeholder. It then awaits `safeRpc<ArchiveEntry[]>('get_public_debate_archive', { p_user_id: userId })`. If that call returns an error, it sets `container.innerHTML` to `'<div class="dba-empty">Archive unavailable.</div>'` and returns, leaving `_entries` as the empty array. If the call succeeds, it writes the result (defaulting null to empty array, cast to `ArchiveEntry[]`) to `_entries` and calls `_renderTable(container)`. Because `_isOwner` is `false`, the rendered table will omit the add button, per-row action buttons, and the hidden-entry badge. Unlike `loadDebateArchive`, this function does not route through `_loadAndRender` — it performs its own inline RPC and calls `_renderTable` directly.

### _injectCSS

`_injectCSS` takes no parameters and returns `void`. On entry it reads the module-level boolean `_cssInjected`. If `_cssInjected` is already `true`, the function returns immediately without doing anything. If it is `false`, the function sets `_cssInjected` to `true`, creates a new `<style>` DOM element via `document.createElement('style')`, assigns a CSS string literal to `s.textContent`, and appends the element to `document.head`. The function is therefore guaranteed to mutate the DOM at most once per page lifetime regardless of how many times it is called.

### _archiveUrl

`_archiveUrl` accepts a single `ArchiveEntry` parameter and returns a `string`. It reads `entry.debate_mode` into a local constant `mode`. If `mode` is the string `'ai'`, it returns the path `/moderator-auto-debate.html?id=` with `entry.debate_id` percent-encoded via `encodeURIComponent`. ### _filtered

`_filtered` takes no parameters and returns a new `ArchiveEntry[]`. It reads four module-level variables: `_entries`, `_filterCat`, `_filterResult`, and `_filterSearch`. It calls `_entries.filter()` and applies up to three successive rejection tests per entry. ### _renderTable

It then creates a new `div` element with class `dba-picker-overlay`, sets its `innerHTML` to a sheet containing one row per recent debate (with user-supplied text passed through `escapeHTML`); the result display uses `d.winner === undefined` rather than `=== null` to govern result display, noting that the `RecentDebate` interface has no `winner` field, so this will always produce the no-result path. A click listener is added directly to the overlay so that clicking the overlay background (where `e.target === overlay`) removes the overlay from the DOM. The overlay is appended to `document.body`. Each `.dba-picker-row` element receives a `click` listener that removes the overlay from the DOM, reads `row.dataset.debate` as `debateId`, and makes a second network call via `safeRpc('add_debate_to_archive', { p_debate_id: debateId })`. If that call returns an error, `showToast('Could not add debate', 'error')` is called and the listener returns. On success, `showToast('Added to archive', 'success')` is called and then `_loadAndRender(container)` is awaited.

### _showEditSheet

`_showEditSheet` is a synchronous function that accepts an `ArchiveEntry` and an `HTMLElement` container and returns nothing. It creates a local alias `esc` for `escapeHTML`. It creates a `div` with class `dba-picker-overlay`, sets its `innerHTML` to an edit sheet containing a text input for `custom_name` (id `dba-edit-name`), a text input for `custom_desc` (id `dba-edit-desc`), and a checkbox for `hide_from_public` (id `dba-edit-hide`). A click listener on the overlay dismisses it when `e.target === overlay`. The overlay is appended to `document.body`. A listener on `#dba-edit-cancel` removes the overlay. A listener on `#dba-edit-save` is async: it reads and trims the values of the three inputs, converting empty strings to `null` for the text fields and reading the boolean state of the checkbox. It disables the save button and sets its text to `'…'`. It then calls `safeRpc('update_archive_entry', { p_entry_id: entry.entry_id, p_custom_name: name, p_custom_desc: desc, p_hide_from_public: hide })`. If the call returns an error, `showToast('Could not save', 'error')` is called, the save button is re-enabled, and its text is restored to `'SAVE'`; the function returns without closing the overlay. On success, the overlay is removed from the DOM, `showToast('Saved', 'success')` is called, and `_loadAndRender(container)` is awaited.

### _toggleHide

`_toggleHide` is an `async` function that accepts an `ArchiveEntry` and an `HTMLElement` container and returns a `Promise<void>`. It computes `newHide` as the boolean negation of `entry.hide_from_public`. It calls `safeRpc('update_archive_entry', { p_entry_id: entry.entry_id, p_hide_from_public: newHide })`. If the call returns an error, it calls `showToast('Could not update', 'error')` and returns. On success it calls `showToast` with the message `'Hidden from public'` when `newHide` is `true` or `'Now visible'` when `newHide` is `false`, both with level `'info'`. It then awaits `_loadAndRender(container)`.

### _removeEntry

`_removeEntry` is an `async` function that accepts an `ArchiveEntry` and an `HTMLElement` container and returns a `Promise<void>`. It immediately calls `confirm('Remove this debate from your archive?')`; if the user cancels, the function returns without doing anything further. If the user confirms, it calls `safeRpc('remove_from_archive', { p_entry_id: entry.entry_id })`. If the call returns an error, it calls `showToast('Could not remove', 'error')` and returns. On success it calls `showToast('Removed from archive', 'info')` and then awaits `_loadAndRender(container)`.

### _loadAndRender

`_loadAndRender` is an `async` function that accepts an `HTMLElement` container and returns a `Promise<void>`. It calls `safeRpc<ArchiveEntry[]>('get_my_debate_archive', {})` with an empty parameters object. If the call returns a truthy `error`, it sets `container.innerHTML` to a `div` containing the text `'Could not load archive.'` and returns. On success, it assigns `(data ?? []) as ArchiveEntry[]` to the module-level `_entries`, replacing whatever was there before. It then calls `_renderTable(container)`. This function is the only place in the module that writes to `_entries` (aside from the two exported functions that reset it).

### loadDebateArchive

`loadDebateArchive` is an exported `async` function that accepts an `HTMLElement` container and an optional boolean `isOwner` (defaulting to `false`) and returns a `Promise<void>`. It calls `_injectCSS()`, which ensures the stylesheet is in the document. It writes `isOwner` to the module-level `_isOwner`, and resets `_entries` to `[]`, `_filterCat` to `'all'`, `_filterResult` to `'all'`, and `_filterSearch` to `''`. It sets `container.innerHTML` to a loading indicator. If `isOwner` is `true`, it awaits `_loadAndRender(container)`, which fetches the current user's archive via `get_my_debate_archive` and renders it. If `isOwner` is `false`, it sets `container.innerHTML` to an unavailability message and returns without making any network call.

### loadPublicDebateArchive

`loadPublicDebateArchive` is an exported `async` function that accepts an `HTMLElement` container and a `string` `userId` and returns a `Promise<void>`. It calls `_injectCSS()`. It writes `false` to `_isOwner`, and resets `_entries` to `[]`, `_filterCat` to `'all'`, `_filterResult` to `'all'`, and `_filterSearch` to `''`. It sets `container.innerHTML` to a loading indicator. It then calls `safeRpc<ArchiveEntry[]>('get_public_debate_archive', { p_user_id: userId })`. If the call returns a truthy `error`, it sets `container.innerHTML` to an unavailability message and returns. On success it assigns `(data ?? []) as ArchiveEntry[]` to `_entries` and calls `_renderTable(container)` directly — unlike `loadDebateArchive`, it does not route through `_loadAndRender` and uses a different RPC that accepts a `p_user_id` parameter rather than inferring the user from session context.
