# Stage 2 Outputs — arena-private-lobby.ts

## Agent 01

### createAndWaitPrivateLobby

This is an async function. It receives up to six parameters: `mode`, `topic`, `visibility` (one of `'private'`, `'group'`, or `'code'`), and optional `invitedUserId`, `invitedUserName`, and `groupId`. On entry it calls `set_view('privateLobbyWaiting')` and `pushArenaState('privateLobbyWaiting')` to update module-level view state and push a browser history entry. If the module-level `screenEl` reference is non-null, its `innerHTML` is cleared. It then creates a `div` element, sets its class and id, injects a fixed HTML skeleton (a spinning ring, title reading "CREATING LOBBY...", status text, a code-display slot, and a cancel button), and appends it to `screenEl`. A click listener is attached to the cancel button that calls `cancelPrivateLobby()` as a fire-and-forget void call.

The function then calls `isPlaceholder()`. If that returns true, it schedules a `setTimeout` for 3000 ms that calls `onPrivateLobbyMatched` with a synthetic debate object using a `'placeholder-'` prefixed id, the provided topic or a random entry from `AI_TOPICS` via `randomFrom`, and a fixed opponent, then returns early — the `try/catch` block below is not reached. In the non-placeholder path the function awaits `safeRpc<PrivateLobbyResult>('create_private_lobby', {...})`, passing `mode`, `topic || null`, and the module-level state variables `selectedCategory`, `selectedRanked`, `selectedRuleset`, `selectedRounds`, plus `visibility`, `invitedUserId || null`, `groupId || null`. If `safeRpc` returns an `error`, it is thrown immediately. Otherwise the `data` value is treated as a `PrivateLobbyResult[]` array and the first element is unwrapped. `set_privateLobbyDebateId(result.debate_id)` is called to write the debate id into module-level state. Three DOM elements are then looked up by id (`arena-private-title`, `arena-private-status`, `arena-private-code-display`). The function branches on `visibility`: if `'code'` and `result.join_code` is truthy, title and status text are updated and the code-display slot receives a block of HTML showing the join code — `escapeHTML` wraps `result.join_code` before it enters `innerHTML` — plus a "Copy Challenge Link" button whose click listener calls `navigator.clipboard.writeText` and calls `showToast` on success or falls back to `showToast(link)` on failure. If `visibility` is `'private'`, title and status text are updated using `escapeHTML(invitedUserName || 'them')`. If `visibility` is `'group'`, title and status are set to group-waiting strings. After the branching, `startPrivateLobbyPoll(result.debate_id, mode, topic)` is called synchronously. The `catch` block logs the error via `console.error`, calls `showToast(friendlyError(err) || 'Failed to create lobby')`, then dynamically imports `arena-lobby.ts` and awaits it before calling `renderLobby()` to return the user to the lobby screen.

### startPrivateLobbyPoll

This is a synchronous function that receives `debateId`, `mode`, and `topic` as strings. It reads the module-level `privateLobbyPollTimer`. If that value is truthy, it clears the existing interval with `clearInterval`. A local `elapsed` counter is initialized to `0` and a constant `TIMEOUT_SEC` is set to `600`. The function calls `setInterval` with an async callback and a 3000 ms interval, and immediately passes the resulting timer id to `set_privateLobbyPollTimer` to write it into module-level state. The function then returns; all subsequent work happens inside the interval callback.

Each time the interval fires, `elapsed` is incremented by 3. The callback first reads the module-level `view` variable: if it is not `'privateLobbyWaiting'`, it clears the interval, calls `set_privateLobbyPollTimer(null)`, and returns early — this is the externally-triggered exit path. If `elapsed` has reached or exceeded `TIMEOUT_SEC` (600 s), the interval is cleared, `set_privateLobbyPollTimer(null)` is called, `cancelPrivateLobby()` is called as a fire-and-forget void, `showToast('Lobby expired — no one joined')` is called, and the callback returns. Otherwise the callback enters a `try` block and awaits `safeRpc<CheckPrivateLobbyResult>('check_private_lobby', { p_debate_id: debateId })`. If `error` is truthy or `data` is falsy, the callback returns without action, deferring to the next tick. If `result.status === 'matched'` and both `result.player_b_ready` and `result.opponent_id` are truthy, the interval is cleared, `set_privateLobbyPollTimer(null)` is called, and `onPrivateLobbyMatched` is called synchronously with an object built from `debateId`, `topic || randomFrom(AI_TOPICS)`, a hardcoded role of `'a'`, and opponent fields from the result. If `result.status === 'cancelled'`, the interval is cleared, `set_privateLobbyPollTimer(null)` is called, `showToast('Lobby cancelled')` is called, and `arena-lobby.ts` is dynamically imported before `renderLobby()` is called. The `catch` block swallows all errors silently, allowing the next tick to retry.

### onPrivateLobbyMatched

This is a synchronous function. It receives a single object parameter with fields `debate_id`, `topic`, optional `role`, `opponent_name`, `opponent_elo`, `opponent_id`, optional `total_rounds`, and optional `language`. It reads two module-level state variables: `selectedMode` and `selectedRanked`, `selectedRuleset`. It constructs a `CurrentDebate` object by mapping the parameter fields and module-level state directly: `id` from `data.debate_id`, `topic` from `data.topic`, `role` from `data.role ?? 'a'`, `mode` from `selectedMode ?? 'text'`, `round` hardcoded to `1`, `totalRounds` from `data.total_rounds ?? DEBATE.defaultRounds`, opponent fields from the parameter, `ranked` from `selectedRanked`, `ruleset` from `selectedRuleset`, `language` from `data.language ?? 'en'`, and `messages` as an empty array. It then calls `showMatchFound(debateData)` — imported from `arena-match.ts` — passing the assembled object. It writes no module-level state and returns no value.

### cancelPrivateLobby

This is an async function that takes no parameters. It first reads the module-level `privateLobbyPollTimer`: if truthy, it calls `clearInterval` on it and immediately calls `set_privateLobbyPollTimer(null)` in the same statement. It then reads module-level `privateLobbyDebateId` and calls `isPlaceholder()`. If `privateLobbyDebateId` is truthy and `isPlaceholder()` returns false, it calls `safeRpc('cancel_private_lobby', { p_debate_id: privateLobbyDebateId })` as a fire-and-forget call — the returned promise has `.catch` attached to log a warning via `console.warn`, but the function does not await it. Regardless of that branch, `set_privateLobbyDebateId(null)` is then called unconditionally to clear the stored debate id. The function then awaits a dynamic import of `arena-lobby.ts` and calls `renderLobby()` to navigate the user back to the lobby. The function returns `void` after `renderLobby()` completes.

On the non-placeholder path the function awaits `safeRpc<PrivateLobbyResult>('create_private_lobby', {...})`, passing `mode`, the `topic` (or null), and the module-level state variables `selectedCategory`, `selectedRanked`, `selectedRuleset`, `selectedRounds`, plus `visibility`, `invitedUserId` (or null), and `groupId` (or null). If `safeRpc` returns an `error` field, the function throws it immediately. Otherwise it unwraps the first row from the returned array into `result` and calls `set_privateLobbyDebateId(result.debate_id)` to write the debate id into module-level state. It then reads `arena-private-title`, `arena-private-status`, and `arena-private-code-display` from the DOM. Depending on `visibility`: when `'code'` and `result.join_code` is present, it sets the title to `'SHARE THIS CODE'`, the status to a waiting message, injects a styled join-code display and a copy-link button into the code display slot, and attaches a click listener that writes a `https://themoderator.app/challenge?code=...` URL to the clipboard via `navigator.clipboard.writeText`, calling `showToast('Challenge link copied!')` on success or `showToast(link)` on failure; when `'private'`, it sets the title to `'CHALLENGE SENT'` and the status to a message incorporating `invitedUserName` (escaped via `escapeHTML`); when `'group'`, it sets the title to `'GROUP LOBBY OPEN'` and the status to a generic waiting message. After all DOM updates it calls `startPrivateLobbyPoll(result.debate_id, mode, topic)` synchronously. If anything in the try block throws, the catch logs the error with `console.error`, calls `showToast(friendlyError(err) || 'Failed to create lobby')`, then dynamically imports `./arena-lobby.ts` and awaits the import before calling `renderLobby()` to return the user to the lobby screen.

### startPrivateLobbyPoll

`startPrivateLobbyPoll` is a synchronous function that accepts `debateId`, `mode`, and `topic`. It reads the module-level `privateLobbyPollTimer`; if that value is truthy it calls `clearInterval` on it to stop any pre-existing poll. It initializes a local counter `elapsed` to 0 and a constant `TIMEOUT_SEC` of 600. It then calls `setInterval` with a 3000 ms interval and stores the resulting timer handle by calling `set_privateLobbyPollTimer`, writing it into module-level state.

Each tick of the interval increments `elapsed` by 3, then checks `view` (module-level state) against `'privateLobbyWaiting'`. If `view` has changed away from that value, the interval clears itself via `clearInterval(privateLobbyPollTimer!)` and calls `set_privateLobbyPollTimer(null)`, then returns. If `elapsed` has reached or exceeded `TIMEOUT_SEC` (600 seconds), the interval similarly clears itself, calls `set_privateLobbyPollTimer(null)`, fires `cancelPrivateLobby()` as a void fire-and-forget call, calls `showToast('Lobby expired — no one joined')`, and returns. Otherwise the tick enters a try block and awaits `safeRpc<CheckPrivateLobbyResult>('check_private_lobby', { p_debate_id: debateId })`. If the call returns an error or no data, the tick returns silently and will retry on the next interval. If `result.status` is `'matched'` and both `result.player_b_ready` and `result.opponent_id` are truthy, the interval clears itself, calls `set_privateLobbyPollTimer(null)`, and then calls `onPrivateLobbyMatched` with an object assembled from `debateId`, `topic` (falling back to `randomFrom(AI_TOPICS)` if falsy), a hardcoded role of `'a'`, `result.opponent_name` (defaulting to `'Opponent'`), `result.opponent_elo` (defaulting to 1200), `result.opponent_id`, `result.total_rounds`, and `result.language`. If `result.status` is `'cancelled'` instead, the interval clears itself, calls `set_privateLobbyPollTimer(null)`, calls `showToast('Lobby cancelled')`, then dynamically imports `./arena-lobby.ts` and awaits the import before calling `renderLobby()`. Any exception thrown inside the try block is silently swallowed by the empty catch, and the poll retries on the next tick.

### onPrivateLobbyMatched

`onPrivateLobbyMatched` is a synchronous function. It accepts an inline object type with fields `debate_id`, `topic`, optional `role`, `opponent_name`, `opponent_elo`, `opponent_id`, and optional `total_rounds` and `language`. It reads the module-level state variables `selectedMode`, `selectedRanked`, `selectedRuleset`, and the imported constant `DEBATE.defaultRounds`. From these inputs it constructs a `CurrentDebate` object: `id` from `data.debate_id`; `topic` from `data.topic`; `role` from `data.role` defaulting to `'a'` via nullish coalescing; `mode` from `selectedMode` defaulting to `'text'`; `round` hardcoded to 1; `totalRounds` from `data.total_rounds` defaulting to `DEBATE.defaultRounds`; `opponentName`, `opponentId`, `opponentElo` from their respective data fields; `ranked` from `selectedRanked`; `ruleset` from `selectedRuleset`; `language` from `data.language` defaulting to `'en'`; and `messages` as an empty array. It then calls `showMatchFound(debateData)` with this assembled object. There are no branches, no early returns, no error paths, and no async operations.

### cancelPrivateLobby

`cancelPrivateLobby` is an async function that takes no parameters. It first reads module-level `privateLobbyPollTimer`; if truthy it calls `clearInterval` on it and then `set_privateLobbyPollTimer(null)` in the same statement. It then reads module-level `privateLobbyDebateId` and checks `isPlaceholder()`. If `privateLobbyDebateId` is truthy and `isPlaceholder()` returns false, it calls `safeRpc('cancel_private_lobby', { p_debate_id: privateLobbyDebateId })` as a fire-and-forget call — the returned promise has a `.catch` attached that logs a warning via `console.warn('[Arena] cancel_private_lobby failed:', e)`, but the function does not await it. Regardless of the `privateLobbyDebateId` branch outcome, the function calls `set_privateLobbyDebateId(null)` to clear that module-level state. It then dynamically imports `./arena-lobby.ts`, awaiting the import, and calls `renderLobby()` on the result. There are no try/catch blocks and no early returns; the RPC failure is the only error path, and it is handled only by the `.catch` on the fire-and-forget call.
