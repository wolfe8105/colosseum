# Stage 2 Outputs — src/rivals-presence.ts

## Agent 01

### _injectCSS

`_injectCSS` takes no parameters. It first calls `document.getElementById('rival-presence-css')` and returns immediately if that element already exists, preventing duplicate style injection across multiple calls. If no such element is found, it calls `document.createElement('style')`, assigns `'rival-presence-css'` to the element's `id`, and assigns a multi-rule CSS string to `style.textContent`. The string defines two keyframe animations (`rivalSlideIn`, `rivalSlideOut`) and style rules for `#rival-alert-popup` and its child classes (`.rap-icon`, `.rap-title`, `.rap-name`, `.rap-sub`, `.rap-actions`, `.rap-challenge`, `.rap-dismiss`), including the `.dismissing` modifier class that triggers the slide-out animation. The function then calls `document.head.appendChild(style)` to insert the element. It writes nothing to module-level state and returns `void`.

### _dismissPopup

`_dismissPopup` takes no parameters. It calls `document.getElementById('rival-alert-popup')` and returns immediately if no element is found. If the popup exists, it calls `popup.classList.add('dismissing')`, which triggers the CSS `rivalSlideOut` animation. It then calls `setTimeout` with a 300ms delay. Inside that callback, `popup.remove()` is called to detach the element from the DOM, and the module-level variable `alertActive` is set to `false`. The callback then checks `alertQueue.length`; if the array is non-empty, it schedules a second `setTimeout` for 600ms that calls `_showNext`. This creates a sequenced gap between the end of the dismiss animation and the appearance of the next popup. The function is synchronous; both `setTimeout` calls are fire-and-forget.

### _showNext

`_showNext` takes no parameters. It calls `alertQueue.shift()` to dequeue the first `PresencePayload` from the module-level `alertQueue` array. If the result is `undefined` (the queue was empty), it sets `alertActive` to `false` and returns. Otherwise it sets `alertActive` to `true` and calls `_injectCSS()`. It then calls `document.getElementById('rival-alert-popup')` and removes that element if one already exists in the DOM. It derives `displayName` from `payload.display_name`, falling back to `payload.username`, then to the string `'YOUR RIVAL'`. It applies `toUpperCase()` and strips `<` and `>` characters via `replace(/[<>]/g, '')` to produce `safeName`. A new `div` element is created with `id = 'rival-alert-popup'` and its `innerHTML` is set to a template containing `safeName` (not passed through `escapeHTML()` — only `<` and `>` are stripped). The popup is appended to `document.body`.

A `setTimeout` is created for 8000ms that calls `_dismissPopup` for auto-dismiss, and its return value is stored in `timer`. An event listener is attached to `#rap-dismiss-btn` via `getElementById`; clicking it calls `clearTimeout(timer)` and then `_dismissPopup()`. An event listener is attached to `#rap-challenge-btn`; clicking it calls `clearTimeout(timer)`, calls `_dismissPopup()`, and then — if `payload.user_id` is truthy — calls `import('./auth.ts')` as a dynamic import. The `.then()` callback calls `showUserProfile(payload.user_id, payload.username ?? undefined)` from the resolved module. If the import rejects, `.catch()` calls `console.warn`. The challenge-button branch is fully fire-and-forget; `_showNext` itself is synchronous.

### _queueAlert

`_queueAlert` takes a single `PresencePayload` parameter. It calls `alertQueue.push(payload)` to append the payload to the module-level `alertQueue` array. It then reads the module-level `alertActive` boolean; if it is `false`, it calls `_showNext()` immediately. If `alertActive` is `true`, the function returns without calling `_showNext`, leaving the newly pushed payload to be dequeued later by the `_dismissPopup` timer chain. The function is synchronous, takes no branches beyond the `alertActive` check, and returns `void`.

### _buildRivalSet

`_buildRivalSet` is `async` and takes no parameters. Inside a `try` block, it `await`s the result of `getMyRivals()`, which is cast through `unknown` to `RivalEntry[]` and stored as `rivals`. It then calls `rivalSet.clear()` on the module-level `rivalSet` Set, and iterates over `rivals` with a `for...of` loop. For each entry, if `r.status` is not `'pending'` and `r.rival_id` is truthy, it calls `rivalSet.add(r.rival_id)`. Entries with `status === 'pending'` or a falsy `rival_id` are silently skipped. If `getMyRivals()` throws or rejects, the `catch` block calls `console.warn` with the error and `rivalSet` is left in whatever state it was in after `rivalSet.clear()` — meaning it will be empty after a failed call. The function returns `Promise<void>`.

### _startPresence

`_startPresence` is `async` and takes no parameters. It calls `getSupabaseClient()` and `getCurrentUser()`, storing results in `supabase` and `user`. If either is falsy, it returns immediately. If the module-level `presenceChannel` is non-null, it calls `supabase.removeChannel(presenceChannel)` and sets `presenceChannel` to `null`. It then `await`s `supabase.realtime.setAuth()` inside a `try/catch` that silently swallows any error. It calls `supabase.channel('global-online', ...)` with `config.private: true` and `config.presence.key` set to `user.id`, assigning the result to `presenceChannel`.

Two presence event listeners are registered on `presenceChannel` via `.on()`. The `'join'` listener iterates over `newPresences`: for each entry cast to `PresencePayload`, it skips entries with no `user_id` or a `user_id` matching the current user, skips entries not in `rivalSet`, and skips entries already in `onlineRivals`. If all checks pass, it adds the `user_id` to `onlineRivals` and calls `_queueAlert(payload)`. The `'leave'` listener iterates over `leftPresences` and calls `onlineRivals.delete(payload.user_id)` for each entry with a truthy `user_id`, allowing that rival to trigger a fresh alert if they rejoin.

`presenceChannel.subscribe()` is called with an async callback receiving `status` and an optional `err`. When `status === 'SUBSCRIBED'`, the callback dynamically imports `'./auth.ts'` (not awaited at the `import()` level — the callback is `async` and awaits the resolved module inline), calls `getCurrentProfile()` from the resolved module, then `await`s `presenceChannel!.track(...)` with the current user's `id`, `username`, and `display_name`. When `status === 'CHANNEL_ERROR'`, the callback calls `console.warn` with the error message and returns without retrying. `_startPresence` itself returns `Promise<void>` and does not `await` the subscribe callback.

### init

`init` is `async` and exported. It reads `FEATURES.rivals` and returns immediately if it is falsy. It calls `getIsPlaceholderMode()` and returns if that is truthy. It calls `getCurrentUser()` and returns if the result is falsy. It reads the module-level `initialized` boolean and returns if it is already `true`. If all four guards pass, it sets `initialized` to `true`, then `await`s `_buildRivalSet()`, then `await`s `_startPresence()`. The two awaits are sequential. If `_buildRivalSet` throws, the error propagates out of `init` uncaught and `_startPresence` is never called. The function returns `Promise<void>`.

### destroy

`destroy` is synchronous and exported. It calls `getSupabaseClient()` and stores the result. If both `supabase` and the module-level `presenceChannel` are truthy, it calls `supabase.removeChannel(presenceChannel)` and sets `presenceChannel` to `null`. It then calls `rivalSet.clear()`, `onlineRivals.clear()`, assigns an empty array to `alertQueue`, and sets `alertActive` to `false`. It sets `initialized` to `false`, resetting the guard that `init` checks. Finally, it calls `document.getElementById('rival-alert-popup')` and, if an element is found, calls `popup.remove()` — removing any visible popup synchronously without triggering the dismiss animation. The function returns `void`.

There are no try/catch blocks in `init` itself; any unhandled rejections from the two awaited calls would propagate to the caller.

### destroy

`destroy` is a synchronous exported function that takes no parameters. It calls `getSupabaseClient()` and, if both the client and module-level `presenceChannel` are truthy, calls `supabase.removeChannel(presenceChannel)` and sets `presenceChannel` to `null`. It then calls `rivalSet.clear()`, `onlineRivals.clear()`, sets `alertQueue` to a new empty array, sets `alertActive` to `false`, and sets `initialized` to `false`. Finally it reads the DOM for `#rival-alert-popup` and, if the element is found, calls `popup.remove()`. The function returns `void`. No error path or try/catch block exists.

## Agent 05

### _injectCSS

`_injectCSS` takes no parameters. It first checks the DOM for an element with the id `rival-presence-css`; if that element already exists, it returns immediately without doing anything. Otherwise it creates a `<style>` element, assigns it that id, and sets its `textContent` to a multi-rule CSS string. That string defines two `@keyframes` animations (`rivalSlideIn` and `rivalSlideOut`) and a set of rules scoped to `#rival-alert-popup` and its child class selectors, covering positioning, colors using CSS custom properties (`--mod-*`), border, padding, shadow, font references, and the `dismissing` class that switches the active animation. The finished `<style>` element is appended to `document.head`. The function writes nothing to module-level state and returns `void`.

### _dismissPopup

`_dismissPopup` takes no parameters. It queries the DOM for `#rival-alert-popup`; if no element is found it returns immediately. If the element exists it adds the CSS class `dismissing` to it, which triggers the `rivalSlideOut` keyframe animation. It then schedules a `setTimeout` callback for 300 ms. When that callback fires it calls `popup.remove()` to detach the element from the DOM and sets the module-level variable `alertActive` to `false`. It then checks whether `alertQueue` has any entries; if it does, it schedules a second `setTimeout` for 600 ms that calls `_showNext`, deferring display of the next queued alert until after the gap. The function is synchronous, returns `void`, and writes nothing immediately to module-level state beyond scheduling the callbacks.

### _showNext

`_showNext` takes no parameters. It calls `alertQueue.shift()` to dequeue the first `PresencePayload` from the module-level `alertQueue` array. If the result is `undefined` (queue was empty), it sets `alertActive` to `false` and returns immediately. Otherwise it sets `alertActive` to `true` and calls `_injectCSS` to insert the stylesheet if not already present.

It then removes any pre-existing `#rival-alert-popup` element from the DOM. It computes a display name from `payload.display_name`, falling back to `payload.username`, then to the literal string `'YOUR RIVAL'`. It uppercases that string and strips `<` and `>` characters to produce `safeName`. It creates a `<div>` with id `rival-alert-popup`, sets its `innerHTML` to a template containing the icon, title, `safeName`, subtitle, and two buttons (`rap-challenge-btn` and `rap-dismiss-btn`), and appends the div to `document.body`.

A `setTimeout` for 8 000 ms is stored in `timer` and calls `_dismissPopup` when it fires — this is the auto-dismiss path. An event listener is attached to `rap-dismiss-btn`: when clicked it clears `timer` and calls `_dismissPopup`. An event listener is attached to `rap-challenge-btn`: when clicked it clears `timer`, calls `_dismissPopup`, and then — if `payload.user_id` is truthy — dynamically imports `./auth.ts` and calls `showUserProfile(payload.user_id, payload.username ?? undefined)`. If that import rejects, the error is logged with `console.warn` and swallowed. The function is synchronous (the dynamic import is fire-and-forget inside a click handler), returns `void`, and writes to `alertActive` and the DOM.

### _queueAlert

`_queueAlert` takes one parameter, `payload`, of type `PresencePayload`. It pushes `payload` onto the end of the module-level `alertQueue` array. It then reads `alertActive`: if it is `false`, it calls `_showNext` immediately, which dequeues and displays the alert just added. If `alertActive` is `true` a popup is already visible and the payload remains in the queue for later display. The function is synchronous, returns `void`, and writes only to `alertQueue` (and indirectly to module-level state through `_showNext`).

### _buildRivalSet

`_buildRivalSet` is an `async` function that takes no parameters. It calls `await getMyRivals()`, which is an external async call imported from `auth.ts`. The returned value is cast to `RivalEntry[]`. Inside a `try` block, it calls `rivalSet.clear()` to empty the module-level `rivalSet`, then iterates over every entry `r` in the results. For each entry whose `status` is not `'pending'` and whose `rival_id` is truthy, it adds `r.rival_id` to `rivalSet`. If `getMyRivals()` throws, the `catch` block logs a warning via `console.warn` and swallows the error, leaving `rivalSet` in whatever state it was before (cleared by the `rivalSet.clear()` call that precedes the loop, meaning an error after `clear()` leaves the set empty). The function returns `Promise<void>` and writes only to the module-level `rivalSet`.

### _startPresence

`_startPresence` is an `async` function that takes no parameters. It reads `getSupabaseClient()` and `getCurrentUser()` from the `auth` import. If either returns a falsy value it returns immediately. If the module-level `presenceChannel` is already set, it calls `supabase.removeChannel(presenceChannel)` to unsubscribe the old channel, then sets `presenceChannel` to `null`.

It then calls `await supabase.realtime.setAuth()` inside a `try/catch`; if that throws the error is silently swallowed. Next it calls `supabase.channel('global-online', ...)` with `config.private: true` and `presence.key` set to `user.id`, assigning the result to the module-level `presenceChannel`.

It registers a `presence` `join` event handler on the channel. When a join event fires, the handler loops over `newPresences`. For each presence it casts the entry to `PresencePayload` and skips it if `payload.user_id` is falsy, if it equals the current user's id, if `rivalSet` does not contain `payload.user_id`, or if `onlineRivals` already contains `payload.user_id`. If all checks pass it adds `payload.user_id` to `onlineRivals` and calls `_queueAlert(payload)`. It registers a `presence` `leave` event handler: for each departed presence it deletes `payload.user_id` from `onlineRivals`, allowing that rival to trigger a fresh alert the next time they join.

It then calls `presenceChannel.subscribe(...)` with an `async` callback. When the status is `'SUBSCRIBED'`, the callback dynamically imports `./auth.ts`, reads `getCurrentProfile()`, and calls `await presenceChannel!.track(...)` with an object containing `user_id`, `username`, and `display_name`. When the status is `'CHANNEL_ERROR'`, it logs `err?.message` via `console.warn`. The function itself returns `Promise<void>` and writes to `presenceChannel` and, indirectly through the event handlers, to `onlineRivals` and `alertQueue`.

### init

`init` is an exported `async` function that takes no parameters. It reads the `FEATURES.rivals` flag from `config.ts`; if that flag is falsy it returns immediately. It then calls `getIsPlaceholderMode()`; if that returns truthy it returns immediately. It reads `getCurrentUser()`; if no user is present it returns immediately. It reads the module-level `initialized` flag; if it is `true` it returns immediately, making subsequent calls within the same session no-ops. Otherwise it sets `initialized` to `true`, then calls `await _buildRivalSet()` to populate `rivalSet`, then calls `await _startPresence()` to subscribe to the realtime channel. The function returns `Promise<void>` and writes to `initialized` as well as the state written by its callees.

### destroy

`destroy` is a synchronous exported function that takes no parameters. It reads `getSupabaseClient()`; if a client is available and the module-level `presenceChannel` is set, it calls `supabase.removeChannel(presenceChannel)` to unsubscribe the realtime channel and sets `presenceChannel` to `null`. It then calls `rivalSet.clear()`, `onlineRivals.clear()`, assigns an empty array to `alertQueue`, sets `alertActive` to `false`, and sets `initialized` to `false`, resetting all module-level state to its initial values. Finally it queries the DOM for `#rival-alert-popup` and, if found, calls `popup.remove()` to detach it immediately without the slide-out animation (bypassing `_dismissPopup`). The function returns `void`.
