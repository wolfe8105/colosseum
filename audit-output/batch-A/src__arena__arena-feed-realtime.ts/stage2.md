# Stage 2 Outputs — arena-feed-realtime.ts

## Agent 01

### subscribeRealtime

`subscribeRealtime` accepts a single parameter `debateId` of type `string` and returns `void`. It begins by calling `getSupabaseClient()` to obtain the Supabase client instance; if the client is falsy or `isPlaceholder()` returns `true` (indicating the arena is in a placeholder/non-live state), the function returns immediately without doing anything. If both checks pass, it calls `client.channel(`feed:${debateId}`, { config: { private: true } })` to create a private Supabase Realtime channel scoped to the given debate. It then chains three `.on()` listeners onto that channel before calling `.subscribe()`. The first listener is a `postgres_changes` listener watching for `INSERT` events on the `debate_feed_events` table in the `public` schema, filtered to rows where `debate_id` equals the provided `debateId`; when triggered, its callback receives `payload: { new: FeedEvent }` and immediately calls `appendFeedEvent(payload.new)` to append the new event to the feed UI. The second listener is a `broadcast` listener on the `'heartbeat'` event; its callback receives `payload: { payload?: { role?: string; ts?: number } }`, extracts `role` from the nested payload, and if `role` is truthy, writes `Date.now()` into `lastSeen[role]` — a mutable dictionary read from `arena-feed-state.ts` — recording the timestamp of the most recent heartbeat from that participant role. The third listener is a `broadcast` listener on the `'goodbye'` event; its callback extracts `role` from `payload.payload` and, if `role` is truthy, calls `handleParticipantGone(role)` to trigger disconnection handling for that participant. After chaining all three listeners and calling `.subscribe()`, the resulting channel object is passed to `set_feedRealtimeChannel(channel)` to store it in shared arena state, and then `startHeartbeat()` is called to begin the outbound heartbeat loop for the local participant. There is no explicit error handling; any failure in channel construction or subscription is unhandled and would propagate or fail silently.

### unsubscribeRealtime

`unsubscribeRealtime` takes no parameters and returns `void`. It begins by calling `getSupabaseClient()` to retrieve the Supabase client. It then evaluates a single branching condition: if both `client` is truthy and `feedRealtimeChannel` (read from shared arena state in `arena-state.ts`) is truthy, it calls `client.removeChannel(feedRealtimeChannel)` to detach and clean up the active Realtime channel subscription, then calls `set_feedRealtimeChannel(null)` to clear the stored channel reference in arena state. If either `client` is falsy or `feedRealtimeChannel` is null/undefined, the entire body is skipped and the function returns without side effects. Notably, `unsubscribeRealtime` does not call `stopHeartbeat()` or `sendGoodbye()` — those are separate concerns handled elsewhere; this function is strictly responsible for tearing down the Supabase channel and nulling the state reference. There are no error paths or return values.
