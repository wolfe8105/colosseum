# Stage 2 Outputs — src/pages/home.feed.ts

## Agent 01

### fetchLiveDebates

`fetchLiveDebates` is an async function that takes no parameters and returns a `Promise<LiveDebate[]>`. It begins by calling `getSupabaseClient()` to retrieve the Supabase client instance and `getIsPlaceholderMode()` to check a flag indicating whether the app is running in placeholder/demo mode. If either the client is falsy or placeholder mode is active, the function returns an empty array immediately without touching the network. Inside a `try` block, it awaits a PostgREST query against the `arena_debates` table, selecting the debate ID, topic, category, status, mode, spectator count, round tracking fields, the two debater UUIDs, and joined profile rows for each debater via two foreign-key relations (`debater_a_profile` and `debater_b_profile`). The query filters to rows whose `status` is one of `'live'`, `'round_break'`, or `'voting'`, orders by `created_at` descending, and limits results to five rows. If the response carries an error object or `data` is falsy, the function returns an empty array. Otherwise it maps the raw rows into `LiveDebate` objects: `spectator_count` defaults to `0`, `current_round` defaults to `1`, `max_rounds` defaults to `5`, debater UUIDs are coerced to `null` if absent, and each debater's display name falls back first to `username`, then to the hardcoded string `'Debater A'` or `'Debater B'`. The mapped array is returned. The `catch` block logs the error to `console.error` and returns an empty array, so all network or parsing failures produce the same empty-array result as the early-exit paths.

### renderFeed

`renderFeed` is an async exported function that takes no parameters and returns `Promise<void>`. It begins by reading the DOM for the element with id `'screen-home'`; if that element does not exist, it returns immediately. It then looks up an element with id `'home-feed-container'`. If that element is absent, it creates a new `<div>` with that id, sets its inline style to `width:100%`, and appends it to `homeScreen`. Regardless of whether the container was just created or already existed, the function writes a centered loading spinner into the container's `innerHTML`. It then awaits two calls in sequence: first, `fetchLiveDebates()` — described above — which performs the Supabase network query and returns an array of `LiveDebate` objects; second, `ModeratorAsync.fetchTakes()`, which is awaited before any rendering proceeds (its internal behavior is defined in `src/async.ts` and is not detailed here). After both awaits complete, the function builds an HTML string by iterating over the `liveDebates` array. For each debate, it concatenates a card block containing: a category label passed through `escapeHTML`, both debater names passed through `escapeHTML`, the result of `bountyDot()` called with each debater's ID (which returns an HTML fragment — its exact output is defined in `src/bounties.ts`), the topic passed through `escapeHTML`, a meta line showing spectator count and round info as plain numeric values interpolated directly into the template, and a "Watch Live" button whose `data-debate-id` attribute is `escapeHTML`-wrapped. No branching occurs if `liveDebates` is empty — the loop simply produces no card HTML, and the resulting `html` string contains only the empty-string accumulation. After the loop, a `<div id="hot-takes-feed"></div>` placeholder is appended to `html`. The entire `html` string is then written to `feedEl.innerHTML`, replacing the loading spinner. Finally, `ModeratorAsync.loadHotTakes('all')` is called synchronously (not awaited), making it a fire-and-forget call that populates the `hot-takes-feed` div asynchronously after `renderFeed` returns.
